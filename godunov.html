Методы типа Годунова

HLLC-солвер, реконструкция MUSCL и WENO, переход от 1D к 2D коду для уравнений Эйлера.

§1 Введение и цель лекции

В предыдущей лекции (FLIC) мы решали уравнения Эйлера методом расщепления: лагранжев этап + эйлеров перенос. Давление на границе ячеек аппроксимировалось средним арифметическим, а перенос — донорской схемой. Это давало первый порядок точности и значительную числовую диффузию.

Метод Годунова (1959) предлагает принципиально другой подход: на каждой грани ячеек решается задача Римана — задача о распаде произвольного разрыва. Решение этой задачи даёт точные значения потоков через грань. Это даёт более точное воспроизведение разрывов, хотя сам метод Годунова всё ещё первого порядка.

Путь развития: Годунов (1-й порядок, точный Риман) → MUSCL (2-й порядок, линейная реконструкция + limiter) → PPM/WENO (3-5-й порядок, многоточечные стенсилы). При этом римановский солвер можно заменить приближённым — HLL, HLLC, Roe — для ускорения без потери качества.

Архитектура кода

Ключевое преимущество годуновских методов — модульность. Код разбивается на независимые блоки:

1. Граничные условия // ghost cells 2. Реконструкция // MUSCL / WENO → UL, UR на гранях 3. Римановский солвер // HLLC(UL, UR) → F̂ на грани 4. Обновление // U^{n+1} = U^n - Δt/Δx (F̂_{i+1/2} - F̂_{i-1/2}) 5. CFL → шаг по времени // Δt = CFL · Δx / max(|u|+a)

Каждый блок можно менять независимо: заменить MUSCL на WENO, HLLC на Roe, добавить направление $y$ — остальные блоки не меняются. Это фундаментальное отличие от FLIC, где лагранжев и эйлеров этапы тесно связаны.

§2 Конечно-объёмная формулировка

Исходная точка — интегральная форма закона сохранения:

$$\frac{d}{dt}\int_\Omega U\,d\Omega + \oint_{\partial\Omega} F \cdot n\,dS = 0$$

Физический смысл: скорость изменения интеграла от $U$ по объёму $\Omega$ равна (с минусом) полному потоку $F$ через границу $\partial\Omega$. Это справедливо для любого объёма, включая ячейку разностной сетки.

Дискретизация на декартовой сетке

Считаем $U_{i}$ средним значением по ячейке $i$. Потоки через левую и правую грани: $\hat{F}_{i-1/2}$ и $\hat{F}_{i+1/2}$. Тогда:

$$U_i^{n+1} = U_i^n - \frac{\Delta t}{\Delta x}\left(\hat{F}_{i+1/2} - \hat{F}_{i-1/2}\right)$$

Ключевой вопрос: как вычислить $\hat{F}_{i+1/2}$? На грани встречаются два значения — $U_i$ слева и $U_{i+1}$ справа. В общем случае они различны, и на грани формируется разрыв. Метод Годунова решает задачу Римана для этого разрыва.

Замечание. Формулировка не зависит от типа сетки — она работает на структурированных, неструктурированных, и даже адаптивных сетках. Меняется только способ вычисления геометрии ячеек и граней.

§3 Задача Римана

Задача Римана — это начально-краевая задача с кусочно-постоянными начальными данными:

$$U(x, 0) = \begin{cases} U_L & x < 0 \\ U_R & x > 0 \end{cases}$$

Решение содержит три волны, разделяющие четыре области постоянного состояния:

ВолнаСкоростьТипЧто разрешено разрывать
1-волна (левая)$S_L \approx u_L - a_L$Ударная или разрежениеВсе: $\rho$, $p$, $u$
2-волна (контакт)$S_* \approx u_*$Контактный разрывТолько $\rho$; $p$ и $u$ непрерывны
3-волна (правая)$S_R \approx u_R + a_R$Ударная или разрежениеВсе: $\rho$, $p$, $u$

Точное решение (метод Ньютона для $p^*$) мы уже реализовали в лекции по FLIC. Но оно дорогое — итеративный процесс на каждой грани, на каждом шаге. Для практики используют приближённые солверы.

§4 Солвер HLL (Harten, Lax, van Leer)

Простейший приближённый солвер. Модель: всего две волны ($S_L$ и $S_R$), между ними — одно усреднённое состояние $U^*$. Контактный разрыв не разрешается.

$$\hat{F}^{HLL} = \begin{cases} F_L & \text{если } S_L \geq 0 \\ \frac{S_R F_L - S_L F_R + S_L S_R (U_R - U_L)}{S_R - S_L} & \text{если } S_L < 0 < S_R \\ F_R & \text{если } S_R \leq 0 \end{cases}$$

Оценки скоростей волн

$$S_L = \min(u_L - a_L, \; u_R - a_R), \qquad S_R = \max(u_L + a_L, \; u_R + a_R)$$
Недостаток HLL: контактный разрыв размазывается сильно, потому что модель его не содержит. Для задач с контактными поверхностями (многофазные потоки, интерфейсы) HLL неприемлем.

§5 Солвер HLLC (HLL-Contact)

Улучшение HLL: добавлена третья волна — контактный разрыв со скоростью $S_*$. Модель: три волны ($S_L$, $S_*$, $S_R$), четыре состояния.

5.1 Скорость контактного разрыва

$$S_* = \frac{p_R - p_L + \rho_L u_L(S_L - u_L) - \rho_R u_R(S_R - u_R)}{\rho_L(S_L - u_L) - \rho_R(S_R - u_R)}$$

5.2 Промежуточные состояния

Между $S_L$ и $S_*$ (слева от контакта):

$$U_L^* = \rho_L \frac{S_L - u_L}{S_L - S_*} \begin{pmatrix} 1 \\ S_* \\ E_L/\rho_L + (S_* - u_L)\left[S_* + \frac{p_L}{\rho_L(S_L - u_L)}\right] \end{pmatrix}$$

Аналогично $U_R^*$ (между $S_*$ и $S_R$) — та же формула с заменой $L \to R$.

5.3 Потоковая функция HLLC

$$\hat{F}^{HLLC} = \begin{cases} F_L & \text{если } S_L \geq 0 \\ F_L + S_L(U_L^* - U_L) & \text{если } S_L < 0 \leq S_* \\ F_R + S_R(U_R^* - U_R) & \text{если } S_* < 0 < S_R \\ F_R & \text{если } S_R \leq 0 \end{cases}$$
Почему HLLC, а не точный Риман? HLLC даёт результат за $O(1)$ операций (без итераций), сохраняет контактный разрыв, и на практике неотличим от точного решения. Разница видна только на экзотических тестах (задача Леблана, сильные разрежения). Для курсовых задач HLLC — оптимальный выбор.

§6 Реконструкция MUSCL

Метод Годунова с постоянной реконструкцией (значение в ячейке = const) даёт 1-й порядок. Для 2-го порядка нужна линейная реконструкция: вместо ступенчатого профиля используем наклонный, с ограничителем для подавления осцилляций вблизи разрывов.

6.1 Наклоны и ограничители

Для каждой переменной $q$ в ячейке $i$ вычисляем наклон:

$$\sigma_i = \text{limiter}(\Delta_{i-1/2}, \; \Delta_{i+1/2}), \qquad \Delta_{i-1/2} = q_i - q_{i-1}, \quad \Delta_{i+1/2} = q_{i+1} - q_i$$

Ограничитель minmod (наиболее диффузионный, но устойчивый)

$$\text{minmod}(a, b) = \begin{cases} \text{sign}(a) \min(|a|, |b|) & \text{если } ab > 0 \\ 0 & \text{если } ab \leq 0 \end{cases}$$

Ограничитель van Leer (менее диффузионный)

$$\text{vanLeer}(a, b) = \begin{cases} \frac{2ab}{a + b} & \text{если } ab > 0 \\ 0 & \text{если } ab \leq 0 \end{cases}$$

6.2 Реконструированные состояния на гранях

$$q_{i+1/2}^L = q_i + \frac{\sigma_i}{2}\Delta x, \qquad q_{i+1/2}^R = q_{i+1} - \frac{\sigma_{i+1}}{2}\Delta x$$

Эти значения $q^L$, $q^R$ подаются на вход HLLC-солвера вместо постоянных $q_i$, $q_{i+1}$.

Замечание. Реконструкция выполняется для примитивных переменных ($\rho$, $u$, $p$), а не для консервативных ($\rho$, $\rho u$, $\rho E$). Реконструкция по консервативным переменным может давать нефизические состояния ($p < 0$) вблизи разрывов.

§7 WENO5 (Weighted Essentially Non-Oscillatory)

Для 5-го порядка используются 3 подстенсила по 3 точки каждый. Нелинейные веса подавляют вклад стенсилов, содержащих разрыв:

$$q_{i+1/2}^- = \sum_{k=0}^{2} \omega_k \hat{q}_k, \qquad \omega_k = \frac{\alpha_k}{\sum \alpha_j}, \quad \alpha_k = \frac{d_k}{(\varepsilon + \beta_k)^2}$$

где $d_k$ — идеальные веса, $\beta_k$ — индикаторы гладкости, $\varepsilon = 10^{-6}$.

Сравнение: MUSCL — 2 точки, 2-й порядок, простая реализация. WENO5 — 5 точек, 5-й порядок, сложная реализация. На практике MUSCL достаточен для большинства инженерных задач; WENO5 нужен для высокоточных расчётов (турбулентность, акустика).

§8 Интегрирование по времени

8.1 Эйлер (1-й порядок)

$$U^{n+1} = U^n + \Delta t \cdot L(U^n)$$

8.2 SSP-RK2 (2-й порядок, TVD)

$$U^{(1)} = U^n + \Delta t \cdot L(U^n)$$ $$U^{n+1} = \frac{1}{2}U^n + \frac{1}{2}\left[U^{(1)} + \Delta t \cdot L(U^{(1)})\right]$$

SSP-RK2 сохраняет TVD-свойство ограничителей (монотонность) и даёт 2-й порядок по времени — необходим при использовании MUSCL (иначе пространственный 2-й порядок «убивается» временным 1-м).

§9 Переход от 1D к 2D

В 2D вектор консервативных переменных содержит 4 компоненты:

$$U = (\rho, \; \rho u, \; \rho v, \; \rho E)^T$$

и появляется второй потоковый оператор $G$ (по $y$):

$$U_{i,j}^{n+1} = U_{i,j}^n - \frac{\Delta t}{\Delta x}\left(\hat{F}_{i+1/2,j} - \hat{F}_{i-1/2,j}\right) - \frac{\Delta t}{\Delta y}\left(\hat{G}_{i,j+1/2} - \hat{G}_{i,j-1/2}\right)$$

Что меняется

Компонент1D2D
Переменные3 ($\rho, \rho u, E$)4 ($\rho, \rho u, \rho v, E$)
Потоки1 ($F$)2 ($F, G$)
Реконструкция1 направление2 направления
Решатель Римана1 вызов/грань1 вызов/грань (всё ещё 1D!)
Ключевое наблюдение: решатель Римана в 2D — это всё ещё 1D задача, решаемая вдоль нормали к грани. На $x$-грани используем $(u, v)$, на $y$-грани — $(v, u)$ (скорости «поворачиваются»). Код HLLC не меняется!

§10 CFL в 2D

$$\boxed{\Delta t = \text{CFL} \cdot \frac{1}{\frac{\lambda_x}{\Delta x} + \frac{\lambda_y}{\Delta y}}}, \qquad \lambda_x = \max_{i,j}(|u_{i,j}| + a_{i,j}), \quad \lambda_y = \max_{i,j}(|v_{i,j}| + a_{i,j})$$
Типовая ошибка: использование 1D формулы $\Delta t = \text{CFL} \cdot \Delta x / \lambda_x$ в 2D коде → нарушение CFL по $y$ → blow-up.

§11 Strang splitting для split-методов

Если код построен как split (раздельные $x$- и $y$-обновления):

$$U^{n+1} = L_x(\tfrac{\Delta t}{2}) \circ L_y(\Delta t) \circ L_x(\tfrac{\Delta t}{2}) \; U^n$$

Это даёт 2-й порядок по времени. Unsplit-метод (формула из §9) не требует расщепления, но сложнее в реализации.

§12 Тесты корректности

Экструдированный 1D. Задача Сода, протянутая вдоль $y$ (начальные данные не зависят от $y$). 2D решение должно совпасть с 1D. Проверяет: правильность $y$-потоков и граничных условий.
Консервативность. Замкнутая область (все стенки — зеркальные). Суммарные масса, импульс и энергия должны сохраняться до $\varepsilon \sim 10^{-14}$.
Порядок сходимости (isentropic vortex). Гладкое точное решение. Запускаем на сетках $50^2$, $100^2$, $200^2$, $400^2$. Ошибка $L_1(\rho)$ vs $\Delta x$ в log-log: наклон $\approx 2$ для MUSCL.
Диагональный перенос. Проверяет корректность работы обоих направлений одновременно.
Double Mach Reflection. Сложная ударная задача (Woodward & Colella). Проверяет устойчивость на сильных разрывах.

§13 Типовые ошибки

1. Неверный знак $y$-потока. Ошибка в определении направления потока $G$. Симптом: экструдированный 1D не совпадает; решение «зеркалит» по $y$.
2. Ошибки индексации граней. Путаница между $i+1/2$ и $i-1/2$. Проверяйте знак: $U^{n+1} = U^n - \frac{\Delta t}{\Delta x}(\hat{F}_{i+1/2} - \hat{F}_{i-1/2})$ — именно разность, не сумма.
3. 1D CFL в 2D. Самая частая ошибка → blow-up. Формула 2D: $\Delta t = \text{CFL}/(\lambda_x/\Delta x + \lambda_y/\Delta y)$.
4. Угловые ghost-ячейки. В 2D нужно заполнять и угловые ghost-ячейки, не только боковые.
