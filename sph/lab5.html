Мини-проект: исследование на выбор

Цель: выбрать один из трёх вариантов и выполнить самостоятельное исследование с использованием солвера, написанного в лабораторных 1-4. Отчёт — файл Maple с кодом, результатами и выводами. Объём работы — соответствует одной лабораторной (1.5-2 часа).

Все варианты — расширение уже работающего кода лабораторной 4, без необходимости писать солвер заново.

Вариант 1. Детальный анализ сходимости

ВАРИАНТ 1 (базовый)

Порядок сходимости SPH на задаче Сода

Запустить задачу Сода на серии из 5 сеток: \(N = 100, 200, 400, 800, 1600\). Параметры: \(h = 2\Delta x\), \(\alpha = 1\), \(\beta = 2\), \(t_{\text{end}} = 0.2\).

Для каждого N вычислить \(L_1\)-норму ошибки плотности:

\[ L_1 = \frac{1}{N}\sum_{i=1}^{N}|\rho^{\text{SPH}}_i - \rho^{\text{exact}}(x_i)|. \]

Точное решение — кусочно-постоянное, как в задании 4.2 (лекция 4).

Построить \(\log L_1\) vs \(\log N\). Определить порядок сходимости по углу наклона прямой (линейная регрессия через 5 точек, Statistics[LinearFit]):

\[ L_1 \sim N^{-p} \quad\Longleftrightarrow\quad \log L_1 = -p\log N + \mathrm{const}. \]

Ожидание: \(p \approx 1\) — SPH с искусственной вязкостью на решении с разрывами даёт первый порядок сходимости. Это хуже чем у WENO-3/5 на гладких решениях, но теоретически ожидаемо: на любом разрыве (в том числе на контактном) любая TVD-схема падает до 1-го порядка (теорема Годунова).

# --- Вариант 1: Сходимость SPH ---
# Обернуть весь солвер в процедуру, возвращающую L1-ошибку

run_sod_L1 := proc(N_local)
  local dx_l, h_l, gamma_l, k_l, alpha_l, beta_l, eta_l, t_local,
        t_end_l, dt_l, n_steps, x_l, v_l, rho_l, P_l, c_l, m_l, a_l,
        i, j, rho_ex, L1_sum;

  dx_l := 1.0/N_local;
  h_l := 2.0*dx_l;
  gamma_l := 1.4; k_l := 1.0;
  alpha_l := 1.0; beta_l := 2.0;
  eta_l := 0.01*h_l;

  # ...инициализация массивов x_l, v_l, rho_l и т.д.
  # ...компонент compute_density_l, compute_accel_visc_l, leapfrog_l
  # ...главный цикл while t < t_end_l
  ...

  # L1-норма ошибки плотности
  L1_sum := 0;
  for i from 1 to N_local do
    rho_ex := rho_exact(x_l[i]);  # функция из лаб 4
    L1_sum := L1_sum + abs(rho_l[i] - rho_ex);
  end do;
  return L1_sum / N_local;
end proc:

# Запуск серии (внимание: N=1600 может занять 30+ мин)
Ns := [100, 200, 400, 800, 1600];
L1s := Array(1..5):
for k from 1 to 5 do
  printf("Running N = %d ...\n", Ns[k]);
  L1s[k] := run_sod_L1(Ns[k]);
  printf("  L1 = %.4e\n", L1s[k]);
end do:

# Линейная регрессия в log-log
with(Statistics):
xd := [seq(log10(Ns[k]), k=1..5)]:
yd := [seq(log10(L1s[k]), k=1..5)]:
fit := LinearFit([1, x], xd, yd, x);
printf("Порядок сходимости: p ≈ %.3f\n", -coeff(fit, x, 1));

with(plots):
p_data := plot([seq([xd[k], yd[k]], k=1..5)], style=point, symbol=circle,
               symbolsize=12, color=red):
p_fit := plot(fit, x=min(op(xd))..max(op(xd)), color=blue, thickness=1):
display(p_data, p_fit, labels=["log₁₀(N)", "log₁₀(L1)"],
        title="Сходимость SPH на задаче Сода");
Ожидаемое замечание. В отчёте стоит обсудить: почему порядок не 2-й, хотя SPH-ядро имеет второй порядок аппроксимации (теорема 1 из лекции 1)? Ответ: на разрывах точность любой TVD-схемы падает до 1-го, а в L1-норме именно разрывы дают основной вклад в ошибку (гладкие участки дают \(O(N^{-2})\), но они усреднены с разрывом в \(O(N^{-1})\)).

Вариант 2. Сравнение сглаживающих ядер

ВАРИАНТ 2 (средний)

Три ядра на одной задаче

Помимо кубического B-сплайна (4) реализовать ещё два ядра:

Гауссово ядро (обрезанное):

\[ W_G(q) = \frac{1}{h\sqrt{\pi}}\,e^{-q^2}, \qquad \text{обрезать при } q > 3. \tag{5.5} \]

Квинтический B-сплайн:

\[ W_5(q) = \frac{\sigma_5}{h}\Bigl[(3-q)_+^5 - 6(2-q)_+^5 + 15(1-q)_+^5\Bigr],\quad \sigma_5 = \frac{1}{120}. \tag{5.6} \]

Здесь \(x_+ = \max(x, 0)\) — «срезанный» снизу аргумент.

Запустить задачу Сода (\(N = 400\), \(h = 2\Delta x\) для всех трёх ядер). Сравнить:

  • \(L_1\)-ошибки плотности;
  • визуальное качество профилей \(\rho(x)\);
  • время расчёта (квинтический сплайн имеет больший носитель \(|q| < 3\) против \(|q| < 2\) у кубического — больше соседей у каждой частицы).

Ожидание: квинтический сплайн даёт наилучшее качество (меньше шума, резче контакт), гауссов — среднее, кубический — как раньше. Цена: квинтический \(\sim 1.5\) раза дольше (больше соседей).

# --- Вариант 2: Три ядра ---

# Гауссово ядро
W_gauss := proc(q) local qq;
  qq := abs(q);
  if qq > 3 then return 0
  else return exp(-qq^2)/sqrt(Pi)
  end if;
end proc:

dW_gauss := proc(q) local qq, sgn;
  qq := abs(q); sgn := sign(q);
  if qq > 3 then return 0
  else return -2*qq*exp(-qq^2)/sqrt(Pi) * sgn
  end if;
end proc:

# Квинтический B-сплайн
W_quintic := proc(q) local qq, s5;
  qq := abs(q); s5 := 1/120;
  if qq >= 3 then return 0
  elif qq >= 2 then return s5 * (3-qq)^5
  elif qq >= 1 then return s5 * ((3-qq)^5 - 6*(2-qq)^5)
  else return s5 * ((3-qq)^5 - 6*(2-qq)^5 + 15*(1-qq)^5)
  end if;
end proc:

dW_quintic := proc(q) local qq, sgn, s5;
  qq := abs(q); sgn := sign(q); s5 := 1/120;
  if qq >= 3 then return 0

  elif qq >= 1 then return s5 * (-5*(3-qq)^4 + 30*(2-qq)^4) * sgn
  else return s5 * (-5*(3-qq)^4 + 30*(2-qq)^4 - 75*(1-qq)^4) * sgn
  end if;
end proc:

Вариант 3. Другие задачи Римана

ВАРИАНТ 3 (продвинутый)

Тесты Лакса и 123-задача Einfeldt

(а) Задача Лакса. \(\gamma = 1.4\), \(t_{\text{end}} = 0.16\), \(k = P_L/\rho_L^\gamma\):

\(\rho\) \(P\) \(u\)
\(x < 0.5\)0.4453.5280.698
\(x \geq 0.5\)0.50.5710

Особенность: левое состояние имеет ненулевую начальную скорость \(u_L = 0.698\). Это усложняет задачу — частицы сдвигаются с самого начала.

(б) 123-задача Einfeldt. \(\gamma = 1.4\), \(t_{\text{end}} = 0.15\), \(k = P_L/\rho_L^\gamma\):

\(\rho\) \(P\) \(u\)
\(x < 0.5\)10.4−2
\(x \geq 0.5\)10.4+2

Особенность: две встречные волны разрежения, и между ними плотность падает до очень малых значений \(\rho \to 0\). Это критический тест на положительность плотности — некорректные схемы могут дать отрицательную плотность и упасть. SPH с симметричной формой (2.4) и формулой плотности суммированием (2.2) неявно гарантирует \(\rho > 0\), но результирующее давление может быть очень малым.

Для каждой задачи:

  • Модифицировать инициализацию (лаб 2) под новые начальные условия.
  • Запустить до указанного \(t_{\text{end}}\).
  • Построить \(\rho(x), P(x), v(x)\).
  • Сравнить с точным решением — можно построить через любой точный солвер задачи Римана (например, использовать JavaScript-интерактив из §17 основного курса как эталон).
Ожидаемое поведение на 123-задаче. Из-за численной вязкости SPH \(\rho\) в центре (\(x = 0.5\)) не упадёт строго до нуля, а будет \(\rho_{\min} \approx 0.02-0.1\) в зависимости от N. Давление аналогично остаётся положительным. Это нефизично, но устойчиво. Настоящий вакуум (\(\rho = 0\)) в SPH не воспроизводим по построению метода — там нет частиц, и нет никакой информации.

Требования к отчёту

Один файл Maple (lab5_sph_project_<фамилия>.mw), объёмом 3-5 страниц:

  1. Титульный блок: ФИО, группа, выбранный вариант.
  2. Цель (1-2 предложения).
  3. Код: модификации основного солвера (не весь код — только то что изменилось).
  4. Результаты: 2-4 графика + численная таблица (L1-ошибки, порядок сходимости, время расчёта и т.д.).
  5. Обсуждение (1-2 абзаца): соответствуют ли результаты теоретическим ожиданиям, какие особенности поведения метода вы наблюдали, где метод работает плохо, где хорошо.
  6. Выводы (1 абзац): что узнали нового, что было неожиданным.

Критерии оценивания (максимум 10 баллов):

  • +4 балла — корректная реализация (воспроизводимые результаты).
  • +3 балла — качество анализа (графики информативны, есть численные сравнения).
  • +2 балла — обсуждение и выводы (понимание причин наблюдаемого поведения).
  • +1 балл — дополнительные исследования сверх обязательных (например, в варианте 2 — добавить 4-е ядро; в варианте 3 — решить и задачу Лакса, и 123).
