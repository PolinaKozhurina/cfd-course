1. Уравнения Эйлера

Уравнения Эйлера описывают движение невязкого сжимаемого газа без теплопроводности. Это гиперболическая система законов сохранения. В 2D консервативной форме:

$$\frac{\partial \mathbf{U}}{\partial t} + \frac{\partial \mathbf{F}}{\partial x} + \frac{\partial \mathbf{G}}{\partial y} = 0$$

Вектор консервативных переменных — четыре компоненты:

$$\mathbf{U} = \begin{pmatrix}\rho \\ \rho u \\ \rho v \\ E\end{pmatrix} \qquad \mathbf{F}(\mathbf{U}) = \begin{pmatrix}\rho u \\ \rho u^2 + p \\ \rho u v \\ (E+p)u\end{pmatrix} \qquad \mathbf{G}(\mathbf{U}) = \begin{pmatrix}\rho v \\ \rho u v \\ \rho v^2 + p \\ (E+p)v\end{pmatrix}$$
ПеременнаяФизический смыслСвязь с примитивными
U[0] = ρПлотностьПримитивная
U[1] = ρuИмпульс по x\(u = U[1]/U[0]\)
U[2] = ρvИмпульс по y\(v = U[2]/U[0]\)
U[3] = EПолная энергия/объём\(E = \frac{p}{\gamma-1}+\frac{1}{2}\rho(u^2+v^2)\)

Уравнение состояния и примитивные переменные

Система не замкнута без связи между давлением и консервативными переменными. Для идеального газа:

$$p = (\gamma - 1)\left(E - \frac{1}{2}\rho(u^2+v^2)\right) = (\gamma-1)\left(U[3] - \frac{U[1]^2+U[2]^2}{2\,U[0]}\right)$$ $$\gamma = 1.4 \;\text{(воздух)}, \qquad a = \sqrt{\frac{\gamma p}{\rho}} \;\text{(скорость звука)}$$ 
flow.hpp// Давление из консервативных переменных
inline double pressure(const Vec4& U, double g=1.4) {
    return (g-1)*(U[3] - 0.5*(U[1]*U[1]+U[2]*U[2])/U[0]);
}
// Скорость звука
inline double sound_speed(const Vec4& U, double g=1.4) {
    return sqrt(g * max(pressure(U,g), 1e-12) / U[0]);
}

Характеристическая структура

Система Эйлера гиперболическая — у неё \(n+2\) вещественных собственных значений (\(n=2\) — размерность пространства):

$$\lambda_1 = u_n - a, \quad \lambda_2 = \lambda_3 = u_n, \quad \lambda_4 = u_n + a$$ $$u_n = u\cdot n_x + v\cdot n_y \;\text{— нормальная скорость}$$

Это означает: информация распространяется вдоль характеристик с тремя скоростями. Именно это определяет структуру HLLC-решателя.

2. Конечно-объёмный метод на неструктурированной сетке

От дифференциальной к дискретной форме

Интегрируем уравнение Эйлера по произвольной ячейке \(\Omega_i\) с границей \(\partial\Omega_i\):

$$\int\!\!\int_{\Omega_i} \frac{\partial \mathbf{U}}{\partial t}\,d\Omega + \int\!\!\int_{\Omega_i}\left(\frac{\partial \mathbf{F}}{\partial x}+\frac{\partial \mathbf{G}}{\partial y}\right)d\Omega = 0$$

Применяем теорему Гаусса–Остроградского к объёмному интегралу от дивергенции:

$$\frac{d}{dt}\int\!\!\int_{\Omega_i}\mathbf{U}\,d\Omega \;=\; -\oint_{\partial\Omega_i}(\mathbf{F}n_x + \mathbf{G}n_y)\,dS \;=\; -\oint_{\partial\Omega_i}\mathcal{F}(\mathbf{U},\mathbf{n})\,dS$$

После дискретизации: средняя по ячейке величина \(\mathbf{U}_i\), площадь \(V_i\), сумма по граням \(f\):

$$V_i\,\frac{d\mathbf{U}_i}{dt} = -\sum_{f \in \partial\Omega_i} \mathcal{F}(\mathbf{U}_L,\mathbf{U}_R,\mathbf{n}_f)\cdot L_f$$ основное уравнение КОМ
Ключевое свойство: это соотношение верно для любой формы ячейки — треугольника, четырёхугольника, полигона. Форма зашита в нормаль nf и длину грани Lf. Это делает метод применимым к произвольным неструктурированным сеткам.

Главный цикл — по граням, не по ячейкам

На структурированной сетке: два цикла for(i) for(j), обходим все ячейки, явно обращаемся к соседям через индексы. На неструктурированной — один цикл по граням:

Структурированная сетка

// два цикла, фиксированная нормаль
for (int i=0; i<Nx; ++i)
  for (int j=0; j<Ny; ++j) {
    // правая грань: нормаль (1,0)
    flux = hllc(U[i][j], U[i+1][j], 1,0);
    res[i][j]   -= flux;
    res[i+1][j] += flux;
  }

Неструктурированная сетка

// один цикл, нормаль из Face
for (const auto& f : mesh.faces) {
  flux = hllc(U[f.left], U[f.right],
              f.nx, f.ny); // ← отличие!
  cells[f.left ].res -= flux*f.length;
  cells[f.right].res += flux*f.length;
}
HLLC-решатель не меняется при переходе на неструктурированную сетку. Единственное отличие: нормаль (nx, ny) берётся из объекта Face, а не фиксирована как (1,0) или (0,1).

Рис. 1. Face соединяет две ячейки. Поток F вычисляется одним вызовом HLLC и распределяется обеим ячейкам с противоположными знаками.

Что такое left и right — наглядно

Каждая грань хранит два индекса в массиве cells[]. Нормаль всегда ориентирована от left к right. Поток из left вытекает (минус), в right — втекает (плюс). Если right = -1 — справа нет ячейки, генерируем ghost.

    UR = cells[f.right].U;        // реальный сосед
}
Vec4 flux = hllc(UL, UR, f.nx, f.ny);
cells[f.left ].res -= flux * f.length;
if (f.right >= 0)
    cells[f.right].res += flux * f.length;

Самая важная функция построения сетки. Из набора треугольников строит все грани с топологией и ориентацией нормалей. Вызывается автоматически всеми генераторами.

Словарь рёбер

Ключ = {min(a,b), max(a,b)} — не зависит от направления обхода треугольника. Значение = список ячеек-владельцев.

    }

2

Геометрия грани

Нормаль к ребру AB в 2D: повернуть вектор AB на 90° и нормировать.

$$dx = B_x - A_x,\quad dy = B_y - A_y,\quad L = \sqrt{dx^2+dy^2}$$ $$n_x = dy/L,\quad n_y = -dx/L,\quad m_x=(A_x+B_x)/2,\quad m_y=(A_y+B_y)/2$$
3

Ориентация нормали — критически важно!

Нормаль должна смотреть от left к right. Проверяем скалярным произведением с вектором между центроидами:

$$\mathbf{n}\cdot(\mathbf{c}_R - \mathbf{c}_L) > 0 \;\Rightarrow\; \text{нормаль правильная}$$ $$\text{иначе:}\quad n_x \leftarrow -n_x,\;\; n_y \leftarrow -n_y$$ 
if (owners.size() == 2) {
    f.left  = owners[0]; f.right = owners[1];
    double dcx = cells[f.right].cx - cells[f.left].cx;
    double dcy = cells[f.right].cy - cells[f.left].cy;
    if (f.nx*dcx + f.ny*dcy < 0) { f.nx=-f.nx; f.ny=-f.ny; }
} else {
    f.left = owners[0]; f.right = -1; // граничная
    double dot = f.nx*(f.mx-cells[f.left].cx)
               + f.ny*(f.my-cells[f.left].cy);
    if (dot < 0) { f.nx=-f.nx; f.ny=-f.ny; } // наружу
}
Если нормаль перевёрнута — знаки потоков для левой и правой ячеек меняются местами. Компилятор не предупредит, но физический результат неверен.
4

Регистрируем грань в face_ids ячеек

5. Геометрия ячеек

Площадь и центроид треугольника

$$A = \frac{1}{2}\big|(x_1-x_0)(y_2-y_0) - (x_2-x_0)(y_1-y_0)\big|$$ $$c_x = \frac{x_0+x_1+x_2}{3}, \qquad c_y = \frac{y_0+y_1+y_2}{3}$$

Общая формула Гаусса (для любого полигона)

$$A = \frac{1}{2}\left|\sum_{k=0}^{n-1}(x_k y_{k+1} - x_{k+1} y_k)\right|$$ $$c_x = \frac{1}{6A}\sum_{k=0}^{n-1}(x_k+x_{k+1})(x_k y_{k+1}-x_{k+1}y_k)$$

Рис. Геометрия ячейки (слева) и грани (справа). Нормаль ориентируется так, чтобы n*(cx_R - cx_L) > 0.

static void cell_geometry(Mesh& m) {
    for (auto& c : m.cells) {
        auto& n = c.node_ids;  int sz = n.size();
        double A=0, cx=0, cy=0;
        for (int k=0; k<sz; ++k) {
            auto& A_ = m.nodes[n[k]];
            auto& B_ = m.nodes[n[(k+1)%sz]];
            double cr = A_.x*B_.y - B_.x*A_.y;
            A += cr;
            cx += (A_.x+B_.x)*cr;
            cy += (A_.y+B_.y)*cr;
        }
        A *= 0.5;
        c.vol = abs(A);
        c.cx  = cx / (6*A);   // не abs(A) — сохраняем знак!
        c.cy  = cy / (6*A);
    }
}

6.1 Декартова сетка --mesh-type channel

Самый простой способ создать сетку — разбить прямоугольник на равные клетки, каждую клетку разрезать диагональю на два треугольника. Получается Nx×Ny×2 треугольников. Но нам нужна дыра — цилиндр. Решение в лоб: треугольники, центроид которых попал внутрь цилиндра, просто не добавляем в массив ячеек. Граница тела при этом получается ступенчатой — «лесенка» из прямоугольных кусков. Это грубо, зато строится за секунду и не требует никакой топологической сложности.

Когда это подходит: задачи без сложной геометрии тела, первоначальная отладка кода, задачи в каналах и трубах где тело — лишь препятствие для потока, а не объект аэродинамического исследования. Когда это не подходит: расчёт подъёмной силы и сопротивления, где точность контура критична. Из-за ступенчатой стенки нормаль к телу постоянно скачет на 45°, и давление интегрируется с большой погрешностью.

Алгоритм удаления

Проверяем каждый кандидат-треугольник: если его центроид (среднее трёх вершин) лежит внутри круга — выбрасываем. Это работает без явного построения контура — не нужно знать, какие именно ячейки граничат с цилиндром. Граничные грани появятся автоматически при build_faces(): рёбра с только одним владельцем — это и есть граница, включая ступенчатую стенку тела.

$$\text{Удалить } \triangle ABC \;\Leftrightarrow\; (c_x-x_{cyl})^2+(c_y-y_{cyl})^2 < R^2$$ $$c_x = (A_x+B_x+C_x)/3, \quad c_y = (A_y+B_y+C_y)/3$$

};

// Удаление ячеек внутри цилиндра
auto inside = [&](int a, int b, int c) {

Назначение BC по точным координатам узлов

После build_faces() каждая граничная грань имеет right=-1, но мы не знаем, к какой физической границе она относится. Для декартовой сетки это легко: узлы на левой стенке имеют ровно x=xmin, на правой — x=xmax и т.д. Сравниваем с эпсилон-допуском (не на точное равенство — числа с плавающей точкой). Всё что не совпало с рамкой домена — это поверхность тела.

6.2 Триангуляция Делоне --mesh-type delaunay

Декартова сетка с дырой хорошо строится, но плохо аппроксимирует тело. Следующий шаг — разместить точки не на регулярной решётке, а умно: гуще там, где нужна точность, реже там, где поток однороден. И затем триангулировать их так, чтобы получившиеся треугольники были как можно более равностороними — вытянутые треугольники дают большую числовую диффузию и ухудшают сходимость.

Триангуляция Делоне — это алгоритмически оптимальный способ соединить набор точек в треугольники. Из всех возможных триангуляций одного и того же набора точек Делоне максимизирует минимальный угол — то есть делает треугольники наименее вытянутыми. Это напрямую улучшает точность аппроксимации на сетке.

Определение

Формально: триангуляция называется триангуляцией Делоне, если для каждого треугольника описанная окружность не содержит ни одной другой вершины набора точек. На рисунке ниже: слева точка D попала внутрь описанного круга треугольника ABC — это нарушение Делоне, которое нужно исправить флипом диагонали. Справа — правильная конфигурация.

Критерий — детерминант 3×3

$$\det\begin{pmatrix} A_x-D_x & A_y-D_y & (A_x-D_x)^2+(A_y-D_y)^2 \\ B_x-D_x & B_y-D_y & (B_x-D_x)^2+(B_y-D_y)^2 \\ C_x-D_x & C_y-D_y & (C_x-D_x)^2+(C_y-D_y)^2 \end{pmatrix} > 0 \;\Leftrightarrow\; D \text{ вне круга } \triangle ABC$$

Рис. 2. Слева — нарушение: D попала внутрь описанного круга. Справа — правильная Делоне: все круги пусты.

Критерий выражается через детерминант 3×3. Если он положителен — точка D вне круга, нарушения нет. Если отрицателен — нарушение, нужен flip. Детерминант можно вычислить точно целочисленной арифметикой (если координаты целые) или с двойной точностью для вещественных. Для CFD вещественной точности хватает — погрешность 10⁻¹² несущественна.

static bool in_circumcircle(
    double ax,double ay, double bx,double by,
    double cx,double cy, double dx,double dy)
{
    double ax_=ax-dx, ay_=ay-dy;
    double bx_=bx-dx, by_=by-dy;
    double cx_=cx-dx, cy_=cy-dy;
    return (ax_*(by_*(cx_*cx_+cy_*cy_) - cy_*(bx_*bx_+by_*by_))
           -ay_*(bx_*(cx_*cx_+cy_*cy_) - cx_*(bx_*bx_+by_*by_))
           +(ax_*ax_+ay_*ay_)*(bx_*cy_-by_*cx_)) > 1e-12;
}

6.3 Алгоритм Боуэра–Уотсона

Хорошо, Делоне оптимальна — но как её построить? Можно перебирать все возможные триангуляции, но это экспоненциально долго. Боуэр (1981) и Уотсон (1981) независимо придумали элегантный инкрементальный алгоритм: вставляем точки по одной, и после каждой добавленной точки локально восстанавливаем свойство Делоне. Ключевое слово — локально: изменения затрагивают только треугольники вокруг новой точки, не весь граф.

Интуиция: когда мы вставляем точку P, она «попадает» в какой-то треугольник, который «не ожидал» такой точки в своей описанной окружности. Этот треугольник теперь нарушает Делоне. Мы его удаляем. Но его соседи тоже могут нарушать — проверяем их. В итоге удаляется некоторая «дыра» из треугольников, и новая точка соединяется со всеми рёбрами на границе этой дыры. Результат — снова триангуляция Делоне. Сложность O(n log n) в среднем, O(n²) в худшем.

1

Инициализация — супертреугольник

Алгоритм должен с чего-то начать — с одного треугольника, который содержит все будущие точки. Мы создаём огромный треугольник вокруг всего набора. Он будет удалён в конце вместе со всеми треугольниками, у которых есть его вершины. Размер: в 3 раза больше диапазона данных — гарантия того, что ни одна рабочая точка не окажется за ним.

double span = max(xmax-xmin, ymax-ymin) * 3;
nodes.push_back({cx,       cy+span});    // N
nodes.push_back({cx-span, cy-span});    // N+1
nodes.push_back({cx+span, cy-span});    // N+2
triangles = {{N, N+1, N+2}};
2

Для каждой точки P — найти «плохие» треугольники и пробить дыру

Для каждой новой точки нужно найти все треугольники, чья описанная окружность её содержит — это «плохие» треугольники, которые нарушают Делоне из-за P. Их удаляем. Получается «дыра» — многоугольник. Рёбра дыры — те, что принадлежали ровно одному плохому треугольнику (общие рёбра двух плохих треугольников исчезли). P соединяется с каждым ребром дыры — получаем новые треугольники.

for (int pi=0; pi<N; ++pi) {
  double px=pts[pi][0], py=pts[pi][1];

  // 2a. Находим «плохие» треугольники
  bad.clear(); good.clear();
  for (auto& t : tris) {
    if (in_circumcircle(pts[t.a], pts[t.b], pts[t.c], px,py))
      bad.push_back(t);
    else
      good.push_back(t);
  }
  // 2b. Граничные рёбра дыры
  // (принадлежат ровно одному плохому треугольнику)
  auto hole = boundary_edges(bad);

  // 2c. Новые треугольники
  tris = good;
  for (auto& e : hole)
    tris.push_back({e.first, e.second, pi});
}
3

Находим граничные рёбра дыры

Ребро «граничное», если оно входит в bad нечётное число раз (т.е. не является общим для двух плохих треугольников):

// Собираем все рёбра плохих треугольников
for (auto& t : bad) {
  edges.push_back({t.a, t.b});
  edges.push_back({t.b, t.c});
  edges.push_back({t.c, t.a});

// Оставляем только те, у которых нет обратного
for (auto& e : edges) {

4

Очистка от супертреугольника

// Удаляем треугольники, содержащие вершины суперт-ка
for (auto& t : tris)
  if (t.a < N && t.b < N && t.c < N)
    final_tris.push_back(t);

После удаления супертреугольника остаётся валидная триангуляция Делоне для всех рабочих точек. Теперь — удалить треугольники внутри тела. Это нельзя сделать по координатам (как в декартовой сетке), потому что граница нечёткая. Вместо этого смотрим: если треугольник содержит хотя бы одну вершину из [cyl_start, cyl_end) — он граничный, оставляем. Если все три вершины нумеруются после граничных точек цилиндра — он внутри, удаляем. Или проще: flood-fill от центра цилиндра (раздел 6.4).

Три группы точек и сгущение

Качество сетки Делоне полностью определяется тем, куда мы поставили точки. Нужно явно задать три группы: точки на границе тела (задают контур), точки на внешней границе домена (задают BC), и внутренние точки (задают плотность сетки внутри). Внутренние точки генерируются функцией размера h(r) — чем ближе к телу, тем меньше h, тем плотнее точки.

N_cyl — цилиндр

for (i=0; i<N_cyl; ++i) {
  double th=2*PI*i/N_cyl;
  pts.push_back({
    cyl_x+R*cos(th),
    cyl_y+R*sin(th)
  });
}
// idx [0..N_cyl) → Wall BC

N_box — bbox

// 4 стороны равномерно
for(i=0;i<=Nb;++i)
  pts.push_back({
    xmin+dx*i, ymin
  });
// аналогично top/right/left
// → Inflow/Outflow/Wall

N_inner — внутренние

$$h(r)=h_{near}+(h_{far}-h_{near})\!\min\!\left(1,\frac{r-R}{r_{zone}}\right)$$

Точка принимается если до всех соседей расстояние ≥ 0.6·h(r)

Сгущение в следе (wake refinement)

За телом в направлении потока формируется вихревой след. В нём давление и скорость изменяются быстро, и если сетка там крупная — вихри численно рассеются. Добавляем зону сгущения: полосу за цилиндром шириной 3R и длиной wake_len. Внутри этой зоны h уменьшается вдвое от h_near. Это небольшое усложнение кода даёт заметно лучшее сохранение вихревой дорожки Кармана.

// Зона следа: x > cyl_x, |y-cyl_y| < 3R
double dxw = x - cyl_x;
double dyw = abs(y - cyl_y);
if (dxw>0 && dxw<wake_len && dyw<R*3.0) {
    double wt = min(1.0, dxw/(wake_len*0.3));
    double hw = h_near*2.0 + (h_far-h_near*2.0)*wt;
    h = min(h, max(hw, h_near));
}

6.4 Constrained Delaunay и алгоритм Руппера

Чистый Боуэр–Уотсон знает только точки — он ничего не знает про геометрию тела. Если вы поставите точки равномерно по окружности цилиндра, треугольник после триангуляции может соединить две точки с разных сторон тела, пройдя насквозь через металл. Граница тела окажется «размытой» — часть рёбер совпадёт с контуром, часть пойдёт по хордам внутри.

Constrained Delaunay Triangulation (CDT) решает эту проблему: пользователь задаёт список «ограниченных» рёбер — сегментов, которые обязаны присутствовать в итоговой сетке как явные рёбра треугольников. После Боуэра–Уотсона каждое такое ребро либо уже есть в сетке (повезло), либо его нужно принудительно вставить, разрезав мешающие треугольники.

Принудительное ребро — алгоритм Walk&Flip

Идея простая: ребро PQ мешает нескольким существующим треугольникам. Мы проходим от P к Q вдоль треугольников и там, где диагональ четырёхугольника «поперёк» нашего ребра — делаем flip. После каждого flip диагональ поворачивается на 90° и либо ребро PQ уже появилось, либо мешающих треугольников стало меньше. Алгоритм сходится за конечное число шагов.

1

Шаг 1 — найти все треугольники, пересекаемые ребром PQ

Проходим от P к Q: на каждом шаге выбираем грань, через которую вектор PQ пересекает текущий треугольник.

// Список треугольников, пересечённых ребром constraint [p,q]
vector<int> find_crossing(int p, int q) {
    vector<int> crossed;
    int cur = find_tri_containing(pts[p]);
    while (cur != -1) {
        Triangle& t = tris[cur];
        // грань, через которую PQ выходит из t
        int next = exit_edge(t, p, q);
        if (reaches_q(next, q)) break;
        crossed.push_back(cur);
        cur = adj[cur][next];
    }
    return crossed;
}
2

Шаг 2 — восстановить ребро через последовательные flip

Каждый flip заменяет диагональ четырёхугольника на противоположную. Повторяем до тех пор, пока ребро PQ не станет явным ребром сетки.

while (!edge_exists(p, q)) {
    // Найти пересекающийся четырёхугольник ABCD
    auto [a, b, c, d] = find_crossing_quad(p, q);
    // Флип: убрать диагональ AB, поставить CD
    if (convex(a,b,c,d))
        flip_edge(a, b, c, d);
    // иначе — откладываем, переходим к следующей паре
}
3

Шаг 3 — пометить треугольники внутри тела как «hole»

Запустить flood-fill от точки внутри каждого тела. Все достигнутые треугольники — «inside» и удаляются.

void flood_fill_holes() {
    for (auto& hole_pt : hole_seeds) {
        int seed = find_tri_containing(hole_pt);
        queue<int> q; q.push(seed);
        while (!q.empty()) {
            int t = q.front(); q.pop();
            if (inside[t]) continue;
            inside[t] = true;
            for (int nb : adj[t])
                if (nb>=0 && !constrained_edge(t,nb))
                    q.push(nb);
        }
    }
}

Flood-fill работает как заливка в Paint: стартуем из точки гарантированно внутри дыры (например, центр цилиндра), распространяемся во все соседние треугольники через незащищённые рёбра, останавливаемся на constrained-рёбрах (граница тела). Все закрашенные треугольники — «внутри тела» и выбрасываются. Оставшиеся — наша расчётная область.

Алгоритм Руппера — гарантия качества треугольников

CDT убирает плохую топологию, но не гарантирует качество углов. После принудительного вставления рёбер у границы тела могут появиться «иглы» — треугольники с углами 2–5°. Такие ячейки катастрофически ухудшают CFL-условие: на тонкий треугольник нужен очень маленький шаг по времени, и вся схема замедляется. Алгоритм Руппера (1995) это исправляет: он итеративно находит «плохой» треугольник и вставляет новую точку прямо в центр его описанной окружности — туда, откуда треугольник «хотел бы» быть правильным. После вставки Боуэр–Уотсон перетриангулирует окрестность, и бывший плохой треугольник исчезает.

Единственный нюанс: если circumcenter нового треугольника попадает слишком близко к existing constraint-edge, то вместо него вставляется середина этого ребра — иначе алгоритм может зациклиться.

$$\text{Треугольник «плохой»} \;\Leftrightarrow\; \frac{R}{h_{\min}} > B \approx 1.0$$ $$\text{где } R = \text{радиус описанной окружности},\quad h_{\min} = \text{минимальная сторона}$$ $$\text{Новая точка} = \text{центр описанной окружности плохого треугольника}$$ 
void ruppert_refine(double B = 1.0) {
    auto bad = [&](Tri t) {
        double R = circumradius(t);
        double h = min_edge(t);
        return R / h > B;
    };
    while (true) {
        auto it = find_if(tris, bad);
        if (it == tris.end()) break;
        Point cc = circumcenter(*it);
        // Если circumcenter энкроачивает на constraint-edge — вставить midpoint
        if (encroaches_on_constrained_edge(cc))
            cc = midpoint_of_encroached_edge(cc);
        bowyer_insert(cc);
    }
}
B=1.0 → все углы ≥ 30°. B=0.7 → все углы ≥ 40°. Для CFD достаточно B=1.0–1.2. Меньше B → меньше ячеек, хуже геометрия.

Поле размеров h(x,y) — адаптивное сгущение

Самый мощный инструмент при построении сетки — это не алгоритм триангуляции, а функция целевого размера ячейки. Нам не нужна равномерная сетка по всей области: у тела, где градиенты давления и скорости максимальны, нужна мелкая сетка; вдали от тела — крупная. Разница в размере может быть в 20–50 раз.

Идея: перед генерацией внутренних точек определяем функцию h(x,y) — целевой размер ячейки в каждой точке пространства. При сэмплировании новой точки-кандидата мы её принимаем только если она находится на расстоянии не меньше h(x,y) от всех уже принятых точек. Таким образом плотность точек автоматически следует за h. Функцию удобно строить через smoothstep — кубическую интерполяцию без скачков производной на границах зон:

$$h(r) = h_{\text{near}} + (h_{\text{far}} - h_{\text{near}})\cdot \text{smoothstep}\!\left(\frac{r - R_{\text{body}}}{R_{\text{far}} - R_{\text{body}}}\right)$$ $$\text{smoothstep}(t) = t^2(3-2t), \quad t = \text{clamp}(t, 0, 1)$$

Poisson Disk Sampling

Метод Bridson (2007) — генерирует Пуассоновы случайные точки с гарантированным расстоянием ≥ h(x,y) между любыми двумя. O(n) время.

// Active list — кандидаты
for (int k=0; k<K; ++k) {  // K=30 попыток
  double r = h(p) * (1 + rng());
  double th = 2*PI*rng();
  Point q = {p.x+r*cos(th),
              p.y+r*sin(th)};
  if (no_nearby_pts(q, h(q)))
    accept(q);
}

Halton Sequence (квазислучайная)

Детерминированная последовательность с низким расхождением. Лучше покрывает область, чем псевдослучайная. База 2 для X, база 3 для Y.

double halton(int i, int base) {
  double f=1, r=0;
  while (i > 0) {
    f /= base;
    r += f * (i % base);
    i /= base;
  }
  return r;
}
// pts[i] = { halton(i,2), halton(i,3) }

Выбор метода генерации точек влияет на равномерность покрытия и воспроизводимость. Случайная равномерная выборка даёт кластеры и «дыры» — некоторые регионы оказываются переполнены точками, другие пусты. Halton и Poisson disk решают это по-разному: первый через математическую структуру последовательности (низкое discrepancy), второй через явный контроль расстояния. В production-коде применяется Poisson disk — он единственный, кто напрямую принимает h(x,y) как параметр.

Constrained Delaunay с отверстием — полный код

Вот как все шаги собираются вместе. Порядок важен: сначала граничные точки, потом внутренние (иначе Poisson disk может поставить точку внутри тела), затем Боуэр–Уотсон, CDT, Руппер, flood-fill и BC. Менять местами нельзя.

Mesh build_delaunay_with_hole(
    double xmin, double xmax, double ymin, double ymax,
    double cyl_x, double cyl_y, double R,
    int N_cyl=32, int Nb=20,
    double h_near=0.1, double h_far=0.5)
{
    // ── 1. Граничные точки ─────────────────────────────
    vector<array<double,2>> pts;

    // Цилиндр (cos-spacing для равномерности)
    for (int i=0; i<N_cyl; ++i) {
        double th = 2*M_PI*i/N_cyl;
        pts.push_back({cyl_x+R*cos(th), cyl_y+R*sin(th)});
    }
    int cyl_start=0, cyl_end=N_cyl;  // индексы точек цилиндра

    // Ограничивающий прямоугольник
    auto add_edge = [&](double x0,double y0, double x1,double y1) {
        for (int k=0; k<=Nb; ++k) {
            double t = (double)k/Nb;
            pts.push_back({x0+t*(x1-x0), y0+t*(y1-y0)});
        }
    };
    add_edge(xmin,ymin, xmax,ymin);  // нижняя
    add_edge(xmax,ymin, xmax,ymax);  // правая
    add_edge(xmax,ymax, xmin,ymax);  // верхняя
    add_edge(xmin,ymax, xmin,ymin);  // левая

    // ── 2. Внутренние точки (Poisson disk с h(r)) ─────
    auto h = [&](double x, double y) {
        double r = hypot(x-cyl_x, y-cyl_y) - R;
        double t = clamp(r/(5*R), 0.0, 1.0);
        return h_near + (h_far-h_near) * t*t*(3-2*t);  // smoothstep
    };
    poisson_disk_sample(pts, xmin,xmax,ymin,ymax, h, cyl_x,cyl_y,R);

    // ── 3. Боуэр–Уотсон ───────────────────────────────
    Triangulation T = bowyer_watson(pts);

    // ── 4. CDT: добавляем ребра цилиндра как constrained
    for (int i=0; i<N_cyl; ++i)
        T.constrain_edge(i, (i+1)%N_cyl);  // Walk&Flip

    // ── 5. Ruppert refinement ──────────────────────────
    T.ruppert_refine(/*B=*/1.0);

    // ── 6. Удаляем треугольники внутри цилиндра ───────
    T.flood_fill_holes({cyl_x, cyl_y});  // seed внутри дыры

    // ── 7. Назначаем BC ───────────────────────────────
    T.assign_bc_auto(xmin, xmax, ymin, ymax,
                    cyl_x, cyl_y, R,
                    cyl_start, cyl_end);

    return T.to_mesh();
}

6.5 Gmsh — профессиональный генератор --mesh-type gmsh

Gmsh — открытый инструмент (LGPL), который делает всё то, что мы писали вручную, только надёжнее, быстрее и с поддержкой сложной геометрии. Он используется в SU2, OpenFOAM, FEniCS и сотнях академических кодов. Принцип работы: вы описываете геометрию в .geo-файле или через Python API, задаёте поля размеров и граничные условия, вызываете mesh.generate(2) — и получаете готовый .msh-файл.

Главное отличие от нашего самодельного генератора — Gmsh понимает геометрию как объект: кривые, поверхности, тела. Не просто набор точек и рёбер, а именно геометрические примитивы. Это позволяет автоматически находить границы, делать булевы операции (вычесть тело из домена), импортировать CAD-файлы и задавать BC по имени объекта, а не по координатам.

Иерархия геометрических объектов — Point, Curve, Surface

В Gmsh геометрия строится снизу вверх, четырьмя уровнями. Каждый уровень ссылается на предыдущий. Это не просто набор точек и рёбер — это топологическая модель, где каждый объект знает своих «детей» и «родителей».

dim=0 — Point (точка)

Атомарный элемент. Задаётся координатами (x, y, z) и характеристической длиной lc — подсказкой генератору, насколько мелкой должна быть сетка вблизи этой точки. lc — это не жёсткое ограничение, а пожелание. Если рядом есть поле размеров — оно победит.

# Точка в начале координат, целевой размер 0.05
p1 = gmsh.model.geo.addPoint(0, 0, 0, 0.05)
# p1 = целочисленный тег, например 1

# Точка без явного lc (берётся из поля размеров)
p2 = gmsh.model.geo.addPoint(1, 0, 0)

В .geo файле: Point(1) = {0, 0, 0, 0.05};

dim=1 — Curve (кривая)

Соединяет точки. Бывает прямой Line, дугой CircleArc, сплайном Spline или BSpline. Каждая кривая имеет направление: от start-точки к end-точке. Это важно при создании петель — ориентация влияет на то, какая сторона «внутри».

# Прямая линия от p1 до p2
l1 = gmsh.model.geo.addLine(p1, p2)

# Дуга окружности: start, center, end
# Обходит дугу против часовой стрелки
arc = gmsh.model.geo.addCircleArc(p2, p_center, p3)

# Сплайн через список точек
spline = gmsh.model.geo.addSpline([p1, p2, p3, p4])

В .geo: Line(1) = {1, 2}; / Circle(2) = {2, 0, 3};

CurveLoop — замкнутый контур

Список кривых, образующих замкнутый контур. Это не геометрический объект сам по себе, а топологический «контейнер» для создания поверхностей. Кривые должны идти друг за другом (конец одной = начало следующей). Если нужно перевернуть кривую — ставим минус перед тегом.

# Прямоугольник из 4 линий
# Линии должны образовать замкнутый обход
loop = gmsh.model.geo.addCurveLoop([l1, l2, l3, l4])

# Если l3 идёт «не туда» — переворачиваем:
loop = gmsh.model.geo.addCurveLoop([l1, l2, -l3, l4])
# -l3 означает: использовать l3 в обратном направлении

В .geo: Curve Loop(1) = {1, 2, -3, 4};

dim=2 — Surface (поверхность)

Создаётся из одного или нескольких CurveLoop. Первый loop — внешняя граница. Все последующие — дыры (отверстия). Именно здесь задаётся «что триангулировать» — область между внешним контуром и вырезанными телами.

# Простой прямоугольник
outer_loop = gmsh.model.geo.addCurveLoop([l1,l2,l3,l4])
surf = gmsh.model.geo.addPlaneSurface([outer_loop])

# Прямоугольник с дырой — цилиндр вырезан
inner_loop = gmsh.model.geo.addCurveLoop([arc1,arc2,arc3,arc4])
surf = gmsh.model.geo.addPlaneSurface([outer_loop, inner_loop])
# Правило: inner loop должен обходить дыру ПО ЧАСОВОЙ!

В .geo: Plane Surface(1) = {1, 2};

Physical Groups — как задаются граничные условия

Это самая важная часть для CFD. Физические группы — это именованные наборы геометрических объектов одной размерности. Именно они определяют, какие рёбра сетки получат пометку «inflow», «wall», «outflow». Без физических групп Gmsh экспортирует всю сетку без какой-либо BC-информации — наш читатель ничего не поймёт.

Ключевой момент: физическая группа создаётся из геометрических объектов (кривых, поверхностей), а не из узлов или рёбер сетки. Когда Gmsh генерирует сетку, он автоматически помечает все граничные рёбра-элементы, лежащие на кривых из данной физгруппы. Это означает, что если добавить кривую в физгруппу — все рёбра сетки на этой кривой получат её тег.

Принцип работы Physical Groups# Допустим, у нас есть кривые с тегами:
# l_left=1 (левая стенка), l_right=2 (правая), l_top=3, l_bot=4, arc_cyl=5

# Создаём физгруппу dim=1 (кривые) с тегом 10, называем "inflow"
gmsh.model.addPhysicalGroup(1, [l_left],  10)
gmsh.model.setPhysicalName(1, 10, "inflow")

gmsh.model.addPhysicalGroup(1, [l_right], 11)
gmsh.model.setPhysicalName(1, 11, "outflow")

# Стенки трубы и поверхность тела — в одну группу
gmsh.model.addPhysicalGroup(1, [l_top, l_bot, arc_cyl], 12)
gmsh.model.setPhysicalName(1, 12, "wall")

# Обязательно! Без этого Gmsh не экспортирует элементы поверхности
gmsh.model.addPhysicalGroup(2, [surf], 1)
gmsh.model.setPhysicalName(2, 1, "fluid")
Если не добавить физгруппу dim=2 для поверхности (fluid) — Gmsh вообще не экспортирует треугольники в .msh. Это частая ошибка: файл есть, но в нём только граничные рёбра и узлы.

Полный цикл: от геометрии до тегов в сетке

Чтобы было понятно как связаны геометрия и сетка — вот что происходит пошагово от создания точки до получения BC в решателе:

1

Создаём геометрию

Точки → кривые → петли → поверхности. На этом этапе сетки ещё нет, только геометрические объекты с тегами. Теги — просто целые числа, мы их сами назначаем или позволяем Gmsh назначить автоматически (функции возвращают тег).

2

synchronize()

Передаём геометрию из ядра (geo или occ) в модель Gmsh. До sync() геометрия существует только внутри ядра и невидима для остальных частей Gmsh. После sync() можно назначать физгруппы, поля размеров, запускать генерацию. Без synchronize() ничего не работает — это одна из самых частых ошибок.

3

Назначаем физгруппы

Теперь можно добавлять физические группы. Gmsh знает все кривые и поверхности, можно запросить их теги через getEntities(dim) или getBoundary(). Для OCC особенно удобно: после cut() теги кривых меняются, и лучше не хардкодить их, а находить по центру масс или через запрос.

4

mesh.generate(2) и экспорт

Генератор строит сетку. Каждое граничное ребро получает тег физгруппы, которой принадлежит его кривая. Треугольники получают тег физгруппы поверхности. В .msh-файле эти теги записываются как первый тег каждого элемента.

Понимание формата файла критично для написания читателя. Файл состоит из секций, каждая начинается с $Имя и заканчивается на $EndИмя. Разберём реальный минимальный пример — прямоугольник 1×1 с двумя физгруппами:

Полный .geo скрипт — цилиндр в канале

Файл .geo — это нативный язык описания геометрии Gmsh. Он проще Python API, не требует установки пакетов и хорошо подходит для простых задач. Синтаксис C-подобный, есть переменные, циклы, функции. Разберём полный рабочий скрипт для задачи «цилиндр в прямоугольном канале» — той же, что мы строили вручную.

В скрипте виден весь путь: сначала параметры (чтобы легко менять R, размер домена, размер сетки), потом точки, потом линии и дуги, потом петли и поверхности, потом физгруппы и поля. Каждый шаг опирается на теги, созданные на предыдущем.

channel_cylinder.geo// ══════════════════════════════════════════════════

// ══════════════════════════════════════════════════
Lx = 4.0;      // длина канала
Ly = 2.0;      // высота канала
cx = 1.5;      // центр цилиндра x

h_far  = 0.15;   // размер сетки вдали

// ══════════════════════════════════════════════════
// Точки внешнего прямоугольника (4 угла)
// Формат: Point(тег) = {x, y, z, lc};
// ══════════════════════════════════════════════════
Point(1) = {0,  0,  0, h_far};   // нижний левый
Point(2) = {Lx, 0,  0, h_far};   // нижний правый
Point(3) = {Lx, Ly, 0, h_far};   // верхний правый
Point(4) = {0,  Ly, 0, h_far};   // верхний левый

// Точки цилиндра: центр + 4 точки на окружности
// (нужны для создания дуг — CircleArc требует центр)
// ══════════════════════════════════════════════════
Point(5) = {cx,   cy,   0, h_wall};  // центр — НЕ станет узлом сетки!
Point(6) = {cx+R, cy,   0, h_wall};  // право
Point(7) = {cx,   cy+R, 0, h_wall};  // верх
Point(8) = {cx-R, cy,   0, h_wall};  // лево

// ══════════════════════════════════════════════════

Line(2) = {2, 3};   // правая = outflow
Line(3) = {3, 4};   // верхняя стенка
Line(4) = {4, 1};   // левая = inflow

Circle(7) = {8, 5, 9};   // лево → низ
Circle(8) = {9, 5, 6};   // низ → право

// Внешний контур: CCW (против часовой)
// Внутренний (дыра): CW (по часовой) → отрицательные теги
// ══════════════════════════════════════════════════

Curve Loop(2) = {5, 6, 7, 8};       // цилиндр CCW
// Gmsh сам понимает что 2 — дыра, т.к. он внутри 1

// ══════════════════════════════════════════════════
// Плоская поверхность: первый loop — граница, остальные — дыры
// ══════════════════════════════════════════════════
Plane Surface(1) = {1, 2};
// Поверхность 1 = прямоугольник с дырой-цилиндром

// ══════════════════════════════════════════════════
// Физические группы — теги для BC в .msh
// Physical Curve(имя, тег) = {список кривых};
// ══════════════════════════════════════════════════
Physical Curve("inflow",  10) = {4};          // левая стенка
Physical Curve("outflow", 11) = {2};          // правая стенка
Physical Curve("wall",    12) = {1, 3, 5, 6, 7, 8}; // низ+верх+цилиндр
Physical Surface("fluid",  1)  = {1};          // вся расчётная область

// ══════════════════════════════════════════════════
// Поле сгущения — Distance от кривых тела + Threshold
// ══════════════════════════════════════════════════
Field[1] = Distance;
Field[1].CurvesList = {5, 6, 7, 8};   // дуги цилиндра
Field[1].Sampling = 200;              // точек для вычисления расстояния

Field[2] = Threshold;
Field[2].InField  = 1;
Field[2].SizeMin  = h_wall;    // мелко у цилиндра
Field[2].SizeMax  = h_far;     // крупно вдали
Field[2].DistMin  = 0;         // от поверхности...
Field[2].DistMax  = 1.0;       // ...до 1.0 от поверхности

Background Field = 2;

// ══════════════════════════════════════════════════
// Настройки генератора
// ══════════════════════════════════════════════════
Mesh.Algorithm = 5;           // Delaunay (Frontal)
Mesh.CharacteristicLengthMin = 0.01;
Mesh.CharacteristicLengthMax = 0.2;
Mesh.MshFileVersion = 2.2;    // важно! наш читатель понимает только v2

Пройдёмся по тому, что тут видно. Секция $PhysicalNames даёт нам таблицу tag → имя. Она необязательна — если вы не звали setPhysicalName(), секции не будет. Тогда в решателе придётся хардкодить: «тег 10 = inflow». С именами удобнее: читатель делает if name == "inflow" → BC::Inflow.

Секция $Nodes — просто список координат. Нумерация с 1. Индексы не обязаны быть непрерывными (Gmsh может пропускать), поэтому в читателе строим словарь id → индекс. Координата z всегда 0 для 2D задач, но поле присутствует.

В $Elements смешаны все типы. Элементы типа 1 — рёбра на границах, от них нам нужен только тег физгруппы (первый тег). Элементы типа 2 — треугольники, их читаем как ячейки. Обратите внимание: второй тег (geom) для рёбер — номер кривой (4, 5, 6...), для треугольников — номер поверхности (1). Нас интересует только первый тег.

Полный читатель .msh v2 — пошагово

Теперь напишем читатель, который понимает именно этот формат. Три фазы: имена групп → узлы → элементы. Каждая фаза ищет свою секцию, парсит, закрывает.

Читаем файл в три прохода: сначала имена физгрупп (необязательная секция), потом узлы, потом элементы. Граничные рёбра сохраняем в словарь edge_phys, треугольники — в cells. После build_faces() сопоставляем каждой граничной грани её физгруппу, а физгруппу — типу BC.

Gmsh по умолчанию пишет формат .msh v4 (с 2018 года). Наш читатель работает с v2. Явно задайте: gmsh.option.setNumber("Mesh.MshFileVersion", 2.2)
mesh_io.hppMesh load_gmsh(const std::string& fn,
              const map<int,Face::BC>& tag_bc) {
    ifstream f(fn);  Mesh m; string line;

    // Пропустить $MeshFormat ... $EndMeshFormat
    while (getline(f, line) && line != "$Nodes") {}
    int npts; f >> npts;
    m.nodes.resize(npts);
    for (int i=0; i<npts; ++i) {
        int id; f >> id >> m.nodes[i-1].x >> m.nodes[i-1].y;
        double z; f >> z;
    }

    while (getline(f, line) && line != "$Elements") {}
    int nelem; f >> nelem;

    map<pair<int,int>,int> edge_phys;
    for (int i=0; i<nelem; ++i) {
        int id, type, ntags; f >> id >> type >> ntags;
        int phys=0;
        for(int t=0; t<ntags; ++t) { int v; f>>v; if(t==0) phys=v; }

        if (type == 1) {               // линия (граничное ребро)
            int n0,n1; f >> n0 >> n1; n0--; n1--;
            edge_phys[{min(n0,n1), max(n0,n1)}] = phys;
        } else if (type == 2) {        // треугольник
            int a,b,c; f >> a >> b >> c;
            m.cells.push_back({{a-1,b-1,c-1}});
        }
    }
    build_faces(m);
    // Назначаем BC из тегов Gmsh
    for (auto& face : m.faces) {
        if (!face.is_boundary()) continue;
        auto key = make_pair(min(face.n0,face.n1), max(face.n0,face.n1));
        auto it = edge_phys.find(key);
        if (it != edge_phys.end()) {
            auto jt = tag_bc.find(it->second);
            if (jt != tag_bc.end()) face.bc = jt->second;
        }
    }
    cell_geometry(m);
    return m;
}

6.6 Построение сетки для профиля NACA — подробно

Крыло — геометрически самый сложный случай для сеткостроения. Проблем несколько: передняя кромка имеет очень малый радиус (∼0.01–0.02 хорды) и требует экстремально мелкой сетки; задняя кромка — острый угол, где сходятся верхний и нижний потоки; позади профиля тянется след (wake) — зона низкого давления, которую тоже нужно хорошо разрешить. Обычная изотропная триангуляция с этим плохо справляется.

Стандартное решение — O-сетка (O-mesh): концентрические замкнутые кольца вокруг профиля, как годовые кольца дерева. Первое кольцо вплотную к поверхности, следующие расходятся наружу с геометрической прогрессией. Форма колец постепенно переходит от формы профиля к окружности (farfield). Это даёт три преимущества: ячейки вытянуты вдоль обтекания (минимум ложной диффузии), размер нарастает плавно (не нарушается CFL), граница тела задана точно — узлы лежат прямо на аналитически вычисленном контуре.

Для профиля NACA координаты вычисляются по формуле. Семейство NACA 4-digit: цифры M, P, XX задают максимальный камбер, его положение и максимальную толщину в процентах хорды.

Шаг 1 — Координаты профиля NACA

Семейство NACA 4-digit. Параметры: максимальный камбер M (%), позиция камбера P (×10% хорды), толщина XX (%).

$$y_t(x) = 5T\left[0.2969\sqrt{x} - 0.1260x - 0.3516x^2 + 0.2843x^3 - 0.1015x^4\right]$$ $$y_c(x) = \begin{cases} \dfrac{M}{P^2}(2Px - x^2) & x \leq P \\ \dfrac{M}{(1-P)^2}\!\left((1-2P)+2Px-x^2\right) & x > P \end{cases}$$ $$\theta(x) = \arctan\!\left(\frac{dy_c}{dx}\right)$$ $$x_u = x - y_t\sin\theta,\quad y_u = y_c + y_t\cos\theta$$ $$x_l = x + y_t\sin\theta,\quad y_l = y_c - y_t\cos\theta$$

Шаг 2 — Косинусное сгущение точек на профиле

Если расставить N точек равномерно по хорде (x = 0, 1/N, 2/N, ...), передняя кромка, где профиль наиболее искривлён, окажется плохо разрешена — там нужны точки вдесятеро чаще, чем в середине хорды. Решение: параметрически отображаем равномерный угол на хорду. Косинусное преобразование сжимает точки к x=0 и x=1, где кривизна максимальна:

$$x_k = \frac{1 - \cos(\pi k / N)}{2}, \quad k = 0, 1, \ldots, N$$ $$\text{Это отображение } [0,N] \to [0,1] \text{ с концентрацией у } x=0 \text{ и } x=1$$ 
// N точек на полуконтуре (верхняя или нижняя поверхность)
vector<Point> naca_points(int N, double M, double P, double T,
                           double chord, double le_x, double le_y)
{
    vector<Point> upper, lower;
    for (int k=0; k<=N; ++k) {
        double xn = (1 - cos(M_PI*k/N)) * 0.5;  // [0..1]

        double yt = 5*T*(0.2969*sqrt(xn) - 0.1260*xn
                      - 0.3516*xn*xn + 0.2843*pow(xn,3)
                      - 0.1015*pow(xn,4));

        double yc, dyc;
        if (xn <= P) {
            yc  = M/(P*P) * (2*P*xn - xn*xn);
            dyc = M/(P*P) * 2 * (P - xn);
        } else {
            yc  = M/((1-P)*(1-P)) * (1-2*P+2*P*xn-xn*xn);
            dyc = M/((1-P)*(1-P)) * 2 * (P - xn);
        }
        double theta = atan(dyc);
        upper.push_back({le_x + chord*(xn-yt*sin(theta)),
                         le_y + chord*(yc+yt*cos(theta))});
        lower.push_back({le_x + chord*(xn+yt*sin(theta)),
                         le_y + chord*(yc-yt*cos(theta))});
    }
    // O-mesh: LE → upper → TE → lower reversed → LE
    vector<Point> surface = upper;
    for (int k=N-1; k>=1; --k) surface.push_back(lower[k]);
    return surface;  // 2N точек, замкнутый контур
}

Шаг 3 — Радиальные слои с геометрической прогрессией

Теперь от каждой точки поверхности «стреляем» наружу по нормали, создавая последовательные слои. Расстояние между слоями нарастает геометрически: первый слой тонкий (нужна точность), дальние — крупные (экономия ячеек). Нормаль к контуру в дискретных точках вычисляется как среднее нормалей соседних рёбер — это сглаживает острые углы на задней кромке. На выходе получаем двумерный массив layers[layer][point] — «сетка в параметрическом пространстве», которую потом нарезаем на треугольники.

$$d_k = d_1 \cdot r^{k-1}, \quad D_{\text{total}} = d_1 \cdot \frac{r^{N_l} - 1}{r - 1}$$ $$d_1 \approx 0.01c \text{ (первый слой)}, \quad r \approx 1.2\text{–}1.3, \quad N_l = 10\text{–}20 \text{ слоёв}$$ 
Point normal_offset(int i, const vector<Point>& surf, double d) {
    int n = surf.size();
    // Нормаль = среднее нормалей соседних рёбер, нормированное
    Point p  = surf[i];
    Point pm = surf[(i-1+n)%n];
    Point pp = surf[(i+1)%n];
    double nx1 = (p.y-pm.y), ny1 = -(p.x-pm.x);  // нормаль к ребру pm→p
    double nx2 = (pp.y-p.y), ny2 = -(pp.x-p.x);  // нормаль к ребру p→pp
    double nx = nx1+nx2, ny = ny1+ny2;
    double L = hypot(nx, ny);
    return {p.x + d*nx/L, p.y + d*ny/L};  // наружу
}

// Построение слоёв
double d1 = 0.01*chord, ratio = 1.25;
double d_cum = 0;
vector<vector<Point>> layers = {surface};  // слой 0 = профиль
for (int layer=1; layer<=N_layers; ++layer) {
    d_cum += d1 * pow(ratio, layer-1);
    vector<Point> ring;
    for (int i=0; i<surface.size(); ++i)
        ring.push_back(normal_offset(i, surface, d_cum));
    layers.push_back(ring);
}

7. Граничные условия — метод ghost-ячеек

Для граничной грани (right=-1) правой ячейки нет. Генерируем виртуальное «призрачное» состояние U_ghost так, чтобы HLLC возвратил физически верный поток.

7.1 Wall — адиабатическая стенка

Физическое условие: нормальная скорость на стенке = 0. Реализация: зеркальное отражение.

$$u_n = u\cdot n_x + v\cdot n_y$$ $$u_{ghost} = u - 2u_n n_x, \quad v_{ghost} = v - 2u_n n_y$$ $$\rho_{ghost} = \rho, \quad E_{ghost} = E \quad\text{(давление сохраняется)}$$ 
Vec4 bc_wall(const Vec4& UL, double nx, double ny) {
    double rho=UL[0], u=UL[1]/rho, v=UL[2]/rho;
    double un = u*nx + v*ny;
    return {rho,
            rho*(u - 2*un*nx),
            rho*(v - 2*un*ny),
            UL[3]};
}

7.2 Farfield — характеристические BC

Физически корректно при дозвуке. Инварианты Римана разделяют информацию идущую внутрь и наружу домена:

$$R^+ = u_{n,int} + \frac{2a_{int}}{\gamma-1} \quad\longleftarrow\text{ выходит из домена}$$ $$R^- = u_{n,inf} - \frac{2a_{inf}}{\gamma-1} \quad\longleftarrow\text{ приходит снаружи}$$ $$u_{n,ghost} = \frac{R^++R^-}{2}, \qquad a_{ghost} = \frac{\gamma-1}{4}(R^+-R^-)$$

Энтропия берётся из freestream (вход) или из домена (выход):

Vec4 bc_farfield(const Vec4& UL, double nx, double ny) {
    double Rp = unL + 2*aL/(gamma-1);
    double Rm = un_inf - 2*a_inf/(gamma-1);
    double un_g = 0.5*(Rp+Rm);
    double a_g  = max(0.25*(gamma-1)*(Rp-Rm), 1e-6);
    // вход (unL<0): энтропия из freestream
    // выход(unL≥0): энтропия из домена
    double sg = (unL<0) ? p_inf/pow(rho_inf,gamma)
                       : pL/pow(rL,gamma);
    double rg = pow(a_g*a_g/(gamma*sg), 1/(gamma-1));
    // ... собираем U_ghost
}

8. HLLC — приближённый решатель задачи Римана

Задача Римана на грани: слева \(\mathbf{U}_L\), справа \(\mathbf{U}_R\), нормаль \((n_x,n_y)\). Три характеристических волны порождают четыре региона решения.

Рис. 4. Три волны задачи Римана. SL и SR — крайние волны (ударная или волна разрежения), S* — контактный разрыв.

Roe-среднее — оценка скоростей волн

$$u_{Roe}=\frac{\sqrt{\rho_L}u_L+\sqrt{\rho_R}u_R}{\sqrt{\rho_L}+\sqrt{\rho_R}}, \quad H_{Roe}=\frac{\sqrt{\rho_L}H_L+\sqrt{\rho_R}H_R}{\sqrt{\rho_L}+\sqrt{\rho_R}}$$ $$a_{Roe}=\sqrt{(\gamma-1)\!\left(H_{Roe}-\tfrac{1}{2}(u_{Roe}^2+v_{Roe}^2)\right)}$$ $$S_L=\min(u_{nL}-a_L,\;u_{nRoe}-a_{Roe}), \quad S_R=\max(u_{nR}+a_R,\;u_{nRoe}+a_{Roe})$$

Контактная волна S*

$$S^* = \frac{p_R-p_L+\rho_L u_{nL}(S_L-u_{nL})-\rho_R u_{nR}(S_R-u_{nR})}{\rho_L(S_L-u_{nL})-\rho_R(S_R-u_{nR})}$$

Промежуточное состояние

$$\mathbf{U}_K^* = \frac{\rho_K(S_K-u_{nK})}{S_K-S^*}\begin{pmatrix}1\\u_K+(S^*-u_{nK})n_x\\v_K+(S^*-u_{nK})n_y\\E_K/\rho_K+(S^*-u_{nK})\!\left[S^*+\frac{p_K}{\rho_K(S_K-u_{nK})}\right]\end{pmatrix}, \quad K\in\{L,R\}$$ 
Vec4 hllc(const Vec4& UL, const Vec4& UR, double nx, double ny, double g=1.4)
{
    // Примитивные переменные + нормальная скорость
    double unL = uL*nx + vL*ny;  // ← берём из Face!
    double unR = uR*nx + vR*ny;

    // Roe-среднее
    double sqL=sqrt(rL), sqR=sqrt(rR), inv=1/(sqL+sqR);
    double Hr = (sqL*HL + sqR*HR)*inv;
    double ar = sqrt(max(1e-12,(g-1)*(Hr-0.5*(ur*ur+vr*vr))));

    // Скорости волн
    double SL = min(unL-aL, unr-ar);
    double SR = max(unR+aR, unr+ar);

    // Контактная волна
    double Ss = (pR-pL + rL*unL*(SL-unL) - rR*unR*(SR-unR))
              / (rL*(SL-unL) - rR*(SR-unR) + 1e-300);

    // Выбор региона
    if (SL>=0) return Fn(UL,...);
    if (SR<=0) return Fn(UR,...);
    if (Ss>=0) { FL += SL*(Us(UL,SL)-UL); return FL; }
    else        { FR += SR*(Us(UR,SR)-UR); return FR; }
}

9. Схема Годунова (1-й порядок)

Кусочно-постоянное приближение внутри каждой ячейки. Значение на грани = значение в центроиде ближайшей ячейки.

$$\varphi_{face,L} = \varphi_L, \qquad \varphi_{face,R} = \varphi_R$$

Рис. 5. Годунов: скачок на грани = 1.0 (полная разность).

void residuals_godunov(Mesh& m, double g=1.4) {
    m.zero_res();
    for (const auto& f : m.faces) {
        const Vec4& UL = m.cells[f.left].U;
        Vec4 UR = f.is_boundary()
            ? ghost(UL, f.nx, f.ny, f.bc)
            : m.cells[f.right].U;

        Vec4 flux = hllc(UL, UR, f.nx, f.ny, g);
        double L = f.length;

        m.cells[f.left ].res -= flux * L;
        if (f.right>=0) m.cells[f.right].res += flux * L;
    }
}
Знаки: нормаль грани ориентирована от left к right. Для левой ячейки поток вытекает (−), для правой — втекает (+). Это обеспечивает консервативность: сумма потоков по сетке = 0.

10. Схема Годунова–Колгана (2-й порядок)

Кусочно-линейное приближение: линейная экстраполяция от центроида к грани. Ограничитель minmod предотвращает осцилляции у ударных волн.

$$\varphi_{face} = \varphi_C + \frac{1}{2}\,\psi(r)\cdot\Delta_{near}$$ $$\Delta_{near} = \varphi_N - \varphi_C \quad\text{(к соседу)}, \quad \Delta_{far} = \varphi_C - \varphi_{far} \quad\text{(от дальнего)}$$ $$r = \frac{\Delta_{far}}{\Delta_{near}}, \qquad \psi(r) = \max(0,\,\min(1,\,r)) \quad\text{(minmod)}$$

Рис. 6. Колган: ψ_L=0 (r<0 — экстремум), ψ_R=0.6. Скачок уменьшился с 1.0 до 0.7 → меньше числовой диффузии.

Поиск дальнего соседа

На неструктурированной сетке нет индекса i-1. Ищем ячейку с максимальной проекцией на обратное направление к грани:

int far_neighbor(int cell_id, int face_id, const Mesh& m) {
    const Face& f0 = m.faces[face_id];
    double dx0 = f0.mx - m.cells[cell_id].cx;  // вектор к грани
    double dy0 = f0.my - m.cells[cell_id].cy;
    double L0  = sqrt(dx0*dx0 + dy0*dy0) + 1e-14;

    double best = -1e30;  int result = -1;
    for (int fid : m.cells[cell_id].face_ids) {
        if (fid == face_id) continue;
        int nbr = other_cell(fid, cell_id);
        if (nbr < 0) continue;
        double dxk = m.cells[nbr].cx - m.cells[cell_id].cx;
        double dyk = m.cells[nbr].cy - m.cells[cell_id].cy;
        // проекция на обратное направление к грани
        double proj = -(dxk*dx0 + dyk*dy0) / L0;
        if (proj > best) { best=proj; result=nbr; }
    }
    return result; // -1 → граница → ψ=0 автоматически
}

11. Интегрирование по времени

Обновление ячеек

$$\mathbf{U}_i^{n+1} = \mathbf{U}_i^n + \frac{dt}{V_i}\cdot\mathbf{R}_i$$

Делим на площадь ячейки vol, не на dx. На неструктурированной сетке ячейки разного размера — это существенно.

Явный Эйлер (1-й порядок по t)

$$\mathbf{U}^{n+1} = \mathbf{U}^n + \frac{dt}{V}\,R(\mathbf{U}^n)$$

12. Условие CFL

$$dt = \mathrm{CFL}\cdot\min_{\text{ячейки }i,\;\text{грани }f}\frac{h_{i,f}}{|\mathbf{u}_i\cdot\mathbf{n}_f|+a_i}$$ $$h_{i,f} = \frac{V_i}{L_f} \quad\text{— характерный размер ячейки через грань }f$$

На неструктурированной сетке с tanh-сгущением первый слой у цилиндра в 50–100 раз тоньше последнего. Именно эти ячейки ограничивают глобальный dt.

double compute_dt(const Mesh& m, double CFL, double g=1.4) {
    double dt = 1e20;
    for (const auto& f : m.faces)
        for (int s=0; s<2; ++s) {
            int ci = (s==0) ? f.left : f.right;
            if (ci<0) continue;
            const Vec4& U = m.cells[ci].U;
            double rho=U[0], u=U[1]/rho, v=U[2]/rho;
            double p = max((g-1)*(U[3]-0.5*rho*(u*u+v*v)), 1e-12);
            double a = sqrt(g*p/rho);
            double h = m.cells[ci].vol / f.length;         // V/L
            double lam = abs(u*f.nx + v*f.ny) + a;
            dt = min(dt, CFL * h / (lam + 1e-300));
        }
    return dt;
}

14. Задача Вудворда–Колеллы

Классический тест 1984 года для численных схем. Сверхзвуковой поток M=3 ударяется об уступ, порождая систему отражающихся ударных волн.

Физика — временная эволюция

t ≈ 0.2

Косая ударная волна от острого угла уступа распространяется вверх. Угол наклона определяется числом Маха: \(\sin\theta_{shock} = 1/M\).

t ≈ 0.4

Волна достигает верхней стенки. При M=3 происходит нерегулярное отражение (Маховское): отражённая волна наклонена иначе, чем при регулярном.

t ≈ 0.8

Образуется тройная точка — место пересечения трёх ударных волн. За ней — контактный разрыв (слой смешения).

t ≈ 2–4

Квазиустановившаяся картина: система пересекающихся скачков, «Маховский диск» у стенки, вихрь у угла уступа.

NACA 4-digit — формула профиля

Профиль задаётся четырьмя цифрами: MPXX. Например, NACA 2412: M=2% (камбер), P=4 (позиция 0.4c), XX=12% (толщина).

Распределение толщины:

$$y_t = 5\,t\!\left(0.2969\sqrt{\xi} - 0.1260\,\xi - 0.3516\,\xi^2 + 0.2843\,\xi^3 - 0.1015\,\xi^4\right), \quad \xi = x/c$$

Камберная линия и её наклон:

$$y_c = \begin{cases}\dfrac{M}{P^2}(2P\xi - \xi^2) & \xi \le P \\[8pt] \dfrac{M}{(1-P)^2}(1 - 2P + 2P\xi - \xi^2) & \xi > P\end{cases}$$

Координаты верхней и нижней поверхностей (учёт угла наклона камберной линии):

$$\theta = \arctan\!\left(\frac{dy_c}{d\xi}\right)$$ $$x_U = x - y_t\sin\theta,\quad y_U = y_c + y_t\cos\theta$$ $$x_L = x + y_t\sin\theta,\quad y_L = y_c - y_t\cos\theta$$

Косинусоидальное распределение точек на профиле

Равномерное распределение параметра x даёт слишком редкие точки у кромок. Косинусоидальное сгущает их туда где нужно:

$$x_i = \frac{c}{2}\!\left(1 - \cos\frac{\pi\,i}{N}\right), \quad i = 0,1,\ldots,N$$

При \(i=0\): \(x=0\) (передняя кромка). При \(i=N\): \(x=c\) (задняя кромка). Точки у кромок сгущены в \(\pi/2\) раз сильнее, чем у середины.

15.2 Профиль NACA через Gmsh

Gmsh даёт настоящую качественную сетку вокруг профиля: O-слой вплотную к поверхности (structured boundary layer), переход к неструктурированной Делоне в дальнем поле. Это стандарт в промышленных CFD-кодах.

Подход 1 — координаты напрямую из Python

Генерируем координаты профиля скриптом, вставляем в .geo:

import numpy as np

def naca4(code, N=200):
    M, P = int(code[0])/100, int(code[1])/10
    T = int(code[2:])/100
    beta = np.linspace(0, np.pi, N+1)
    x = 0.5 * (1 - np.cos(beta))          # косинус-сгущение
    yt = 5*T*(0.2969*np.sqrt(x) - 0.1260*x
             - 0.3516*x**2 + 0.2843*x**3 - 0.1015*x**4)
    yc = np.where(x<P,
        M/P**2 * (2*P*x - x**2),
        M/(1-P)**2 * (1 - 2*P + 2*P*x - x**2))
    dyc = np.where(x<P,
        2*M/P**2*(P-x), 2*M/(1-P)**2*(P-x))
    theta = np.arctan(dyc)
    xu = x - yt*np.sin(theta);  yu = yc + yt*np.cos(theta)
    xl = x + yt*np.sin(theta);  yl = yc - yt*np.cos(theta)
    # Полный обход: верх (от хвоста к носу) + низ (от носа к хвосту)
    xs = np.concatenate([xu[::-1], xl[1:]])
    ys = np.concatenate([yu[::-1], yl[1:]])
    return xs, ys

xs, ys = naca4("2412", N=150)

# Генерируем .geo файл

17. Инициализация поля — init.hpp

Вызывается один раз перед главным циклом — строго после назначения BC. Три обязательных шага:

  1. Построить сетку (build_*_mesh())
  2. Назначить BC граничным граням (обычно внутри генератора)
  3. Заполнить начальное поле (один из методов ниже)
init.hpp// 1. Равномерный поток везде — самый надёжный старт
void init_freestream(Mesh& m) {
    Vec4 Ui = FLOW.Uinf();
    for (auto& c : m.cells) c.U = Ui;
}

// 2. Потенциальное течение вокруг цилиндра — быстрее сходится при M<<1
// u = U0(1 - R²/r² cos2θ), v = -U0 R²/r² sin2θ, p из теоремы Бернулли
void init_potential(Mesh& m, double R, double g=1.4) {
    double U0 = std::sqrt(FLOW.u_inf*FLOW.u_inf + FLOW.v_inf*FLOW.v_inf);
    for (auto& c : m.cells) {
        double r2=c.cx*c.cx+c.cy*c.cy, R2r=R*R/r2;
        double u = U0*(1-R2r*(c.cx*c.cx-c.cy*c.cy)/r2);
        double v = -U0*R2r*2*c.cx*c.cy/r2;
        double p = std::max(FLOW.p_inf + 0.5*FLOW.rho_inf*(U0*U0-u*u-v*v), 1e-6);
        c.U = {FLOW.rho_inf, FLOW.rho_inf*u, FLOW.rho_inf*v,
               p/(g-1) + 0.5*FLOW.rho_inf*(u*u+v*v)};
    }
}

// 3. Задача Вудворда-Колеллы: равномерный сверхзвук M=3
void init_step(Mesh& m, double Mach=3.0, double g=1.4) {
    double rho=1.4, p=1.0;
    double a = std::sqrt(g*p/rho);   // a=1 при rho=1.4, p=1
    double u = Mach * a;
    double E = p/(g-1) + 0.5*rho*u*u;
    Vec4 U0 = {rho, rho*u, 0.0, E};
    for (auto& c : m.cells) c.U = U0;
    FLOW.Mach=Mach; FLOW.init();   // Inflow BC обновляется
}

// 4. Рестарт: int32 N + N×4 double
void load_restart(Mesh& m, const std::string& fn) {
    std::ifstream f(fn, std::ios::binary);
    int nc; f.read((char*)&nc, 4);
    if (nc != m.nc()) throw std::runtime_error("size mismatch");
    for (auto& c : m.cells) f.read((char*)c.U.data(), 32);
}

    write_snap(mesh, 0);

            std::cout << "t="<<t<<" Cd="<<ac.Cd<<" Cl="<<ac.Cl<<"\n";
        }
    }

    write_vtk(mesh, "results/out_final.vtk");
    save_restart(mesh, "results/restart.bin");
    write_cp(mesh, "results/surface_cp.dat");
}

20. Аэродинамические коэффициенты

$$F_x = -\sum_{\text{Wall}} p\cdot n_x\cdot L_f, \qquad F_y = -\sum_{\text{Wall}} p\cdot n_y\cdot L_f$$ $$C_d = \frac{F_x}{q_\infty D_{ref}}, \quad C_l = \frac{F_y}{q_\infty D_{ref}}, \quad q_\infty = \tfrac{1}{2}\rho_\infty V_\infty^2$$ 
schemes.hppstruct Aero { double Cd=0, Cl=0; };

Aero compute_aero(const Mesh& m, double rho_inf,
                   double V_inf, double D_ref, double g=1.4) {
    double Fx=0, Fy=0, q=0.5*rho_inf*V_inf*V_inf;
    for (const auto& f : m.faces) {
        if (f.bc != Face::BC::Wall) continue;
        const Vec4& U = m.cells[f.left].U;
        double rho=U[0], u=U[1]/rho, v=U[2]/rho;
        double p=(g-1)*(U[3]-0.5*rho*(u*u+v*v));
        // Минус: нормаль НАРУЖУ от ячейки → сила на тело
        Fx -= p * f.nx * f.length;
        Fy -= p * f.ny * f.length;
    }
    return {Fx/(q*D_ref), Fy/(q*D_ref)};
}
Уравнения Эйлера — невязкий газ. Реальный Cd профиля крыла — вязкий эффект пограничного слоя (требует Навье–Стокса). Cl при малых углах атаки Эйлер предсказывает правильно. Для NACA 2412, aoa=5°: теоретический Cl ≈ 0.62.
