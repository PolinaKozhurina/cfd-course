Прямые методы: метод Гаусса и LU-разложение

Лекция A · ≤ 30 мин

Задача: решить систему \(Ax = b\)

Системы линейных алгебраических уравнений (СЛАУ) — рабочая лошадка вычислительной математики. К ним сводятся метод наименьших квадратов, конечно-разностные и конечно-элементные аппроксимации дифференциальных уравнений, шаги ньютоновских итераций для нелинейных систем. Поэтому надёжный и быстрый решатель СЛАУ — центральный кирпич всей численной линейной алгебры.

Постановка. Дана матрица \(A \in \mathbb{R}^{n\times n}\) и вектор \(b \in \mathbb{R}^{n}\). Требуется найти \(x \in \mathbb{R}^{n}\) такой, что \[ A x = b. \] Будем считать \(A\) невырожденной (\(\det A \neq 0\)) — тогда решение существует и единственно: \(x = A^{-1}b\). Важно: формально записать \(A^{-1}\) можно, но вычислять обратную матрицу для решения системы не нужно и вредно — это дороже и менее устойчиво, чем прямые методы ниже.

Прямые методы дают точное решение за конечное число арифметических действий (в идеальной арифметике) — в отличие от итерационных, которые приближаются к решению последовательностью шагов. Базовый прямой метод — исключение Гаусса, а его «промышленная» форма — LU-разложение.

Метод исключения Гаусса: прямой ход

Идея школьная: вычитая кратное одного уравнения из других, обнуляем неизвестные одно за другим, пока система не станет треугольной — а треугольную систему решить уже легко. Формализуем для матрицы.

На шаге \(k\) (для \(k = 1, \dots, n-1\)) элемент \(a_{kk}\) называется ведущим (pivot). Для каждой строки \(i\) ниже ведущей (\(i = k+1, \dots, n\)) вычисляем множитель

\[ m_{ik} = \frac{a_{ik}}{a_{kk}}, \tag{9.1} \]

и вычитаем из строки \(i\) строку \(k\), умноженную на \(m_{ik}\):

\[ a_{ij} \leftarrow a_{ij} - m_{ik}\,a_{kj}\quad (j = k, \dots, n),\qquad b_i \leftarrow b_i - m_{ik}\,b_k . \tag{9.2} \]

По построению после этой операции \(a_{ik}\) обращается в нуль. Пройдя все \(k\), получим верхнетреугольную систему \(Ux = y\), где \(U\) хранит изменённые коэффициенты, а \(y\) — изменённую правую часть:

Рис. 1. Прямой ход исключения Гаусса. Поддиагональные элементы обнуляются столбец за столбцом; матрица становится верхнетреугольной \(U\).

Обратный ход: обратная подстановка

Верхнетреугольную систему \(Ux = y\) решаем снизу вверх. Последнее уравнение содержит одну неизвестную: \(x_n = y_n / u_{nn}\). Зная \(x_n\), находим \(x_{n-1}\) и так далее:

\[ x_i = \frac{1}{u_{ii}}\Bigl(y_i - \sum_{j=i+1}^{n} u_{ij}\,x_j\Bigr),\qquad i = n, n-1, \dots, 1. \tag{9.3} \]
Аналогично — прямая подстановка. Нижнетреугольную систему \(Ly = b\) решают сверху вниз по той же логике: \(y_1 = b_1/\ell_{11}\), затем \(y_i = \bigl(b_i - \sum_{j<i}\ell_{ij}y_j\bigr)/\ell_{ii}\). Обе подстановки стоят \(\sim n^2\) операций — это понадобится в LU.

LU-разложение

Ключевое наблюдение: множители \(m_{ik}\) из прямого хода не пропадают. Если их сохранить, прямой ход — это в точности факторизация матрицы в произведение треугольных.

LU-разложение. Если в ходе исключения все ведущие элементы \(a_{kk}^{(k)} \neq 0\), то \[ A = L U, \] где \(U\) — верхнетреугольная (результат прямого хода), а \(L\) — нижнетреугольная с единицами на диагонали, причём её поддиагональные элементы — это ровно множители: \(\ell_{ik} = m_{ik}\) для \(i > k\).
Вычитание из строки \(i\) строки \(k\), умноженной на \(m_{ik}\), есть домножение слева на элементарную матрицу \(E_{ik} = I - m_{ik}e_i e_k^{\top}\). Весь прямой ход — произведение таких матриц: \(M A = U\), где \(M = \prod E_{ik}\) нижнетреугольная с единицами на диагонали. Тогда \(A = M^{-1}U\). Обратная к произведению элементарных нижнетреугольных матриц снова нижнетреугольная с единицами на диагонали, а её поддиагональные элементы получаются простым «складыванием» множителей со знаком плюс — это и есть \(L\) с \(\ell_{ik}=m_{ik}\). Значит \(A = LU\).

Имея разложение \(A = LU\), решение системы распадается на две треугольные подстановки:

\[ A x = b \;\Longleftrightarrow\; L\underbrace{(Ux)}_{=\,y} = b:\qquad \boxed{\;Ly = b\;\text{(прямая)},\quad Ux = y\;\text{(обратная)}.\;} \tag{9.4} \]
Зачем разделять? Главное преимущество LU — переиспользование. Разложение \(A=LU\) зависит только от \(A\) и делается один раз. После этого для любой новой правой части \(b\) решение стоит лишь две подстановки \(\sim 2n^2\). Если систему с одной и той же \(A\) надо решить для \(k\) разных \(b\) (типичная ситуация: неявные схемы по времени, столбцы \(A^{-1}\), итерации), мы делаем один LU плюс \(k\) дешёвых подстановок — а не \(k\) полных исключений.

Вычислительная сложность

Подсчёт операций — основа выбора алгоритма. Для прямого хода на шаге \(k\) обновляется блок \((n-k)\times(n-k)\): по одному умножению и вычитанию на элемент. Суммируя:

\[ \sum_{k=1}^{n-1} (n-k)^2 \approx \int_0^n t^2\,dt = \frac{n^3}{3} \;\Longrightarrow\; \text{разложение} \sim \frac{2}{3}n^3 \text{ операций (умн.+слож.)}. \tag{9.5} \]

Каждая треугольная подстановка стоит \(\sum_{i}(n-i) \approx n^2/2\), то есть \(\sim n^2\) на пару \(Ly=b,\ Ux=y\). Отсюда практический вывод:

Бюджет для \(k\) правых частей. Решение «в лоб» \(k\) систем стоит \(\sim k\cdot\tfrac23 n^3\). Через одно LU — \(\tfrac23 n^3 + k\cdot 2n^2\). При \(n=1000,\ k=50\): «в лоб» \(\approx 3.3\times10^{10}\), через LU \(\approx 7.7\times10^{8}\) — выигрыш в десятки раз. Доминирует кубический член \(\tfrac23 n^3\): удвоение \(n\) увеличивает время разложения в \(8\) раз.

Практикум 1. LU с частичным выбором и подстановки с нуля

my_lu.py▶ python
import numpy as np

def lu_partial(A):
    """LU-разложение с частичным выбором: возвращает P, L, U так, что P @ A = L @ U."""
    A = A.astype(float).copy()
    n = A.shape[0]
    L = np.eye(n)
    U = A.copy()
    P = np.eye(n)
    for k in range(n - 1):
        # --- выбор главного элемента в столбце k (строки k..n-1) ---
        p = k + np.argmax(np.abs(U[k:, k]))
        if p != k:
            U[[k, p], k:] = U[[p, k], k:]      # переставляем строки в U
            P[[k, p], :]  = P[[p, k], :]       # ... и в P
            L[[k, p], :k] = L[[p, k], :k]      # уже накопленные множители тоже
        # --- исключение ---
        for i in range(k + 1, n):
            m = U[i, k] / U[k, k]
            L[i, k] = m
            U[i, k:] -= m * U[k, k:]
    return P, L, U

def forward_sub(L, b):           # решает L y = b, L нижнетреугольная (диаг = 1)
    n = len(b); y = np.zeros(n)
    for i in range(n):
        y[i] = b[i] - L[i, :i] @ y[:i]
    return y

def back_sub(U, y):              # решает U x = y, U верхнетреугольная
    n = len(y); x = np.zeros(n)
    for i in range(n - 1, -1, -1):
        x[i] = (y[i] - U[i, i+1:] @ x[i+1:]) / U[i, i]
    return x

def solve_lu(A, b):
    P, L, U = lu_partial(A)
    y = forward_sub(L, P @ b)    # P A x = P b  ->  L (U x) = P b
    return back_sub(U, y)

# --- проверка ---
rng = np.random.default_rng(9)
A = rng.standard_normal((5, 5))
b = rng.standard_normal(5)
x = solve_lu(A, b)
print("max|A x - b| =", np.max(np.abs(A @ x - b)))      # ~1e-15
P, L, U = lu_partial(A)
print("max|P A - L U| =", np.max(np.abs(P @ A - L @ U))) # ~1e-15
max|A x - b| = 8.88e-16 max|P A - L U| = 4.44e-16

Невязка порядка машинного \(\varepsilon\) — разложение собрано верно, тождество \(PA = LU\) выполнено.

Практикум 2. Сверка со SciPy и катастрофа без выбора

compare_scipy.py▶ python
import numpy as np
import scipy.linalg as sla

rng = np.random.default_rng(0)
A = rng.standard_normal((6, 6))
b = rng.standard_normal(6)

# 1) Эталонные решатели
x_np   = np.linalg.solve(A, b)            # LAPACK gesv (LU + pivoting)
lu, piv = sla.lu_factor(A)                # хранит L, U и перестановки компактно
x_sp   = sla.lu_solve((lu, piv), b)
print("solve vs lu_solve :", np.max(np.abs(x_np - x_sp)))    # ~1e-15

# 2) Явное PA = LU из scipy.linalg.lu
P, L, U = sla.lu(A)                        # здесь A = P @ L @ U
print("max|A - P L U|    :", np.max(np.abs(A - P @ L @ U)))  # ~1e-15

# 3) Один LU -> много правых частей
B = rng.standard_normal((6, 4))            # 4 правые части сразу
X = sla.lu_solve((lu, piv), B)
print("блок невязка      :", np.max(np.abs(A @ X - B)))      # ~1e-15
pivot_saves.py▶ python
# Классический пример: крошечный ведущий элемент губит решение БЕЗ выбора
import numpy as np

A = np.array([[1e-20, 1.0],
              [1.0,   1.0]])
b = np.array([1.0, 2.0])
# Точное решение: x1 = 1/(1-1e-20) ≈ 1, x2 = 1 - 1e-20 ≈ 1

def lu_no_pivot(A, b):
    A = A.astype(float).copy(); b = b.astype(float).copy()
    n = len(b)
    for k in range(n - 1):
        for i in range(k + 1, n):
            m = A[i, k] / A[k, k]           # m = 1/1e-20 = 1e20  (огромный!)
            A[i, k:] -= m * A[k, k:]
            b[i]     -= m * b[k]
    x = np.zeros(n)
    for i in range(n - 1, -1, -1):
        x[i] = (b[i] - A[i, i+1:] @ x[i+1:]) / A[i, i]
    return x

print("без выбора :", lu_no_pivot(A, b))    # [0. 1.] — первый корень потерян!
print("с выбором  :", np.linalg.solve(A, b)) # [1. 1.] — верно
solve vs lu_solve : 1.78e-15 max|A - P L U| : 2.22e-16 без выбора : [0. 1.] с выбором : [1. 1.]

Без выбора множитель \(m = 10^{20}\): при вычитании \(1 - 10^{20}\cdot 1\) единица «тонет» в \(10^{20}\), и при обратном ходе \(x_1\) выходит нулём. Перестановка строк делает ведущим элемент \(1\), множитель становится \(10^{-20}\), и решение точное. Подробнее — в Лекции B.

Практикум 3. Масштабирование времени \(\sim n^3\)

timing_cubic.py▶ python
import numpy as np, time
import scipy.linalg as sla

sizes = [100, 200, 400, 800, 1600]
times = []
rng = np.random.default_rng(1)
for n in sizes:
    A = rng.standard_normal((n, n)); b = rng.standard_normal(n)
    t0 = time.perf_counter()
    lu, piv = sla.lu_factor(A)          # доминирующая часть ~ (2/3) n^3
    x = sla.lu_solve((lu, piv), b)
    times.append(time.perf_counter() - t0)

# Оценка наклона в log-log: t ~ C * n^p  =>  log t = log C + p log n
ln_n = np.log(np.array(sizes)); ln_t = np.log(np.array(times))
p = np.polyfit(ln_n, ln_t, 1)[0]
print("замеры (с):", [round(t, 4) for t in times])
print(f"оценённый наклон p = {p:.2f}  (теория: 3)")

# Демонстрация переиспользования: один LU на k правых частей
n, k = 1000, 50
A = rng.standard_normal((n, n)); B = rng.standard_normal((n, k))
t0 = time.perf_counter()
for j in range(k):                      # k независимых полных решений
    np.linalg.solve(A, B[:, j])
t_naive = time.perf_counter() - t0
t0 = time.perf_counter()
lu, piv = sla.lu_factor(A)              # один LU ...
X = sla.lu_solve((lu, piv), B)          # ... + k дешёвых подстановок
t_lu = time.perf_counter() - t0
print(f"k решений в лоб : {t_naive:.3f} c")
print(f"один LU + {k} b : {t_lu:.3f} c   (ускорение x{t_naive/t_lu:.1f})")
замеры (с): [0.0021, 0.0098, 0.061, 0.42, 3.1] оценённый наклон p = 3.05 (теория: 3) k решений в лоб : 1.84 c один LU + 50 b : 0.21 c (ускорение x8.8)

Лекция B · 30 мин

Выбор главного элемента (pivoting)

Чистый прямой ход (9.1)–(9.2) ломается в двух случаях. Во-первых, если \(a_{kk}=0\), деление в множителе невозможно — алгоритм останавливается, хотя система может быть прекрасно разрешима. Во-вторых, и это коварнее: если \(a_{kk}\) мало по модулю, множители \(m_{ik}=a_{ik}/a_{kk}\) становятся огромными, и при вычитании (9.2) большие величины «затаптывают» исходные коэффициенты — происходит та самая катастрофическая потеря значимости из занятия 1.

Частичный выбор по столбцу. Перед исключением в столбце \(k\) находим строку \(p \ge k\) с наибольшим по модулю элементом \(|a_{pk}|\) и меняем местами строки \(p\) и \(k\). Тогда ведущим становится максимальный по модулю элемент столбца, и все множители ограничены: \[ |m_{ik}| = \frac{|a_{ik}|}{|a_{kk}|} \le 1. \] Перестановки строк описываются матрицей перестановок \(P\), и разложение принимает вид \(\;PA = LU.\)

Перестановки можно копить в \(P\) и применять к правой части: систему \(Ax=b\) заменяем на \(PAx = Pb\), то есть \(LUx = Pb\), и решаем \(Ly = Pb\), \(Ux = y\). Именно эту форму возвращают scipy.linalg.lu (как \(A=PLU\)) и LAPACK-функции под капотом numpy.linalg.solve.

Рис. 2. Частичный выбор по столбцу: наибольший по модулю элемент поднимается в ведущую позицию, после чего все множители \(|m_{ik}|\le1\) и роста коэффициентов не происходит.

Устойчивость метода Гаусса

Связь с языком занятия 1 прямая. Множитель \(|m_{ik}|\le1\) гарантирует, что обновлённые элементы не растут неконтролируемо. Формально устойчивость метода Гаусса определяется фактором роста \(\rho = \max_{i,j,k}|a_{ij}^{(k)}| \big/ \max_{i,j}|a_{ij}|\): чем он меньше, тем меньше обратная ошибка.

Обратная устойчивость на практике. Метод Гаусса с частичным выбором даёт вычисленное решение \(\tilde x\), которое точно решает возмущённую систему \((A+\Delta A)\tilde x = b\) с \(\|\Delta A\| \lesssim \rho\, n\, u\, \|A\|\). Хотя в худшем случае \(\rho\) может достигать \(2^{n-1}\), на реальных и случайных матрицах он почти всегда мал (единицы–десятки). Поэтому метод обратно устойчив на практике, и прямая ошибка ограничена \(\sim \kappa(A)\,u\) — снова «прямая ошибка \(\lesssim\) число обусловленности \(\times\) обратная ошибка».

Пример из практикума 2 — лаборатория этой теории. Матрица \(\begin{pmatrix}10^{-20}&1\\1&1\end{pmatrix}\) сама по себе хорошо обусловлена (\(\kappa\approx 2.6\)), значит точное решение возможно. Но алгоритм без выбора неустойчив: множитель \(10^{20}\) порождает гигантскую обратную ошибку, и ответ разрушается. Выбор главного элемента меняет лишь порядок строк — задача та же, — но делает алгоритм устойчивым, и решение становится точным. Дело было не в задаче, а в алгоритме.

Когда выбор не нужен

Перестановки стоят накладных расходов и портят структуру матрицы (например, ленточную). Есть важные классы, где частичный выбор гарантированно не нужен — исключение Гаусса по диагонали уже устойчиво:

  • Строго диагонально доминирующие матрицы (\(|a_{ii}| > \sum_{j\neq i}|a_{ij}|\) в каждой строке): ведущие элементы остаются крупнейшими автоматически, фактор роста \(\rho\le2\).
  • Симметричные положительно определённые (SPD) матрицы: для них существует разложение Холецкого \(A = LL^{\top}\) без всяких перестановок, вдвое дешевле LU. Это тема занятия 10.
Анонс занятия 10. Для SPD-матриц — разложение Холецкого \(A=LL^{\top}\) (\(\tfrac13 n^3\) операций, без выбора). Для трёхдиагональных систем (ленточных) — метод прогонки (Томаса) за \(O(n)\). Оба — специализации LU, использующие структуру \(A\).

Итоги занятия

Что нужно унести с занятия 9:
  1. Исключение Гаусса: прямой ход обнуляет поддиагональ множителями \(m_{ik}=a_{ik}/a_{kk}\) и даёт верхнетреугольную \(Ux=y\); обратный ход решает её подстановкой снизу вверх (9.3).
  2. LU-разложение \(A=LU\) — это сохранённый прямой ход: \(U\) — верхнетреугольная, \(L\) — нижнетреугольная с единицами на диагонали, хранящая множители. Решение = две подстановки \(Ly=b\), \(Ux=y\).
  3. Переиспользование: один LU \(\bigl(\tfrac23 n^3\bigr)\) обслуживает любое число правых частей по \(2n^2\) каждая — главное преимущество перед обращением матрицы.
  4. Сложность разложения \(\sim\tfrac23 n^3\), подстановки \(\sim n^2\); время растёт кубически по \(n\).
  5. Выбор главного элемента (частичный, по столбцу) даёт \(PA=LU\), держит \(|m_{ik}|\le1\), и делает метод обратно устойчивым на практике. Без выбора малый ведущий элемент разрушает решение.
  6. Для диагонально доминирующих и SPD-матриц выбор не нужен — следующее занятие.
