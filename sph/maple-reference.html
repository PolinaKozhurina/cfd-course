Памятка по Maple для лабораторных SPH

Страница собирает все функции и конструкции Maple, которые понадобятся в лабораторных 1-5, с описанием параметров, примерами использования и типичными ошибками. Рекомендуется держать открытой во время работы.

Предполагается использование Maple в Document Mode (не Worksheet Mode) — команды разделяются точкой с запятой ; (вывод результата) или двоеточием : (без вывода).

Основы языка Maple

restart язык

restart;

Очищает всё рабочее пространство: переменные, процедуры, загруженные пакеты. Всегда начинайте файл с этой команды — иначе отладка превратится в кошмар (переменные из прошлых запусков останутся).

restart;
# Теперь гарантировано что все имена свободны
N := 400;
dx := 1/N;

Присваивание язык

имя := выражение;

В Maple знак одинарного равенства = означает равенство (используется в уравнениях), а для присваивания используется := (двоеточие + равно). Это частая ошибка у новичков — Maple молча воспримет x = 5 как уравнение и не присвоит значение.

h := 2*dx;             # присваивание
eq := x^2 - 4 = 0;     # уравнение (для solve)

Разделители команд язык

; — выполнить и показать результат. : — выполнить и скрыть результат. При создании больших массивов используйте :, иначе Maple выведет тысячи чисел и подвесит интерфейс.

N := 400;         # покажет "N := 400"
x := Array(1..N): # создаст без вывода (быстрее)

evalf / evalhf math

evalf(expr) evalf(expr, digits) evalhf(expr)

Вычислить выражение численно. evalf — с произвольной точностью (по умолчанию 10 знаков); evalhf — в аппаратной двойной точности (быстрее в \(\sim 100\) раз, но точность IEEE 754).

Maple по умолчанию держит выражения символьно: sqrt(2) остаётся sqrt(2), а не превращается в 1.4142... Для численных алгоритмов это гибельно — вычисление замедляется на порядки. В SPH-лабах используйте evalhf везде, где можно.

evalf(Pi);              # 3.141592654
evalf(Pi, 20);          # 20 знаков
evalhf(sqrt(2));        # 1.41421356237310

# Нормировка с самого начала — числовая
sigma1D := 2.0/3.0;       # НЕ 2/3 (символьная дробь!)
h := 2.0*dx;              # 2.0, не 2
Частая ошибка. Написали sigma := 2/3 — Maple держит это как символьную дробь Rational(2,3). В цикле по 400 частицам каждое умножение на sigma превращается в рациональную арифметику произвольной точности. Расчёт на 10-100 минут вместо 30 секунд. Всегда используйте 2.0/3.0 или evalf(2/3).

Процедуры (функции)

proc ... end proc основное

имя := proc(arg1, arg2, ...) local v1, v2, ...; глобальные := ...; <команды>; return <значение>; end proc:

Определение процедуры (функции). Обязательные элементы:

proc(...)
— список аргументов через запятую; может быть пустым proc().
local ...;
— локальные переменные, изолированные от внешнего мира. Всегда объявляйте все временные переменные как local, иначе они засорят глобальное пространство имён.
return ...
— возвращаемое значение. Можно возвращать несколько: return a, b, c; — тогда при вызове r1, r2, r3 := f(...).
end proc
— завершает тело. После следует : (без вывода самой процедуры).
W_kernel := proc(q)
  local qq;
  qq := abs(q);                         # ядро чётное
  if qq >= 2 then return 0
  elif qq >= 1 then return sigma1D * (2 - qq)^3 / 4
  else return sigma1D * (1 - 3*qq^2/2 + 3*qq^3/4)
  end if;
end proc:

# Вызов
W_kernel(0.5);     # вернёт значение
Подводный камень. Процедура «видит» глобальные переменные (sigma1D в примере выше). Если вы переопределите sigma1D где-то ниже по файлу, процедура будет использовать новое значение. Для воспроизводимости — передавайте всё через параметры или делайте local sigma1D := 2.0/3.0 внутри процедуры.

unapply math

unapply(выражение, x) unapply(выражение, x, y, ...)

Превращает выражение в функцию от указанных переменных. Удобно когда выражение получилось сложным после simplify или int, а надо использовать его как функцию.

F := int(sin(x)^2, x);              # F — выражение
Fn := unapply(F, x);                # теперь Fn(1.5) работает
evalf(Fn(1.5));

Массивы и циклы

Array основное

Array(1..N) Array(1..M, 1..N) Array(1..N, init_proc) Array(1..N, datatype=float[8])

Создаёт числовой массив. Это главное отличие от array (старого типа) и table (ассоциативного массива) — Array эффективен для численных циклов.

Параметр datatype=float[8] явно указывает что массив хранит IEEE-754 double; это ускоряет доступ в \(\sim 3\) раза. Без этого Maple допускает произвольные типы, что медленно.

# Способ 1: пустой массив, потом заполняем
x := Array(1..N, datatype=float[8]):
for j from 1 to N do
  x[j] := (j - 0.5) * dx;
end do:

# Способ 2: инициализация сразу
rho := Array(1..N, i -> if x[i] < 0.5 then 1.0 else 0.125 end if):

# Способ 3: из списка (медленнее, но читаемее)
m := Array([seq(rho[j]*dx, j=1..N)]);
Совет. Индексация в Maple по умолчанию с единицы: x[1] — первый элемент, x[N] — последний. Для SPH-задач это естественно. Если нужна нумерация с нуля (для совместимости с другими языками) — укажите диапазон Array(0..N-1).

for ... do ... end do основное

for имя from A to B do ... end do for имя from A to B by step do ... end do for имя in список do ... end do

Основная циклическая конструкция. Можно вкладывать друг в друга (двойной цикл \(O(N^2)\) в SPH).

# Обычный for
for i from 1 to N do
  x[i] := (i - 0.5) * dx;
end do:

# С нестандартным шагом
for i from 2 to N by 2 do   # только чётные индексы
  ...
end do:

# По списку
for alpha in [0, 0.5, 1.0, 2.0] do
  run_sod(alpha);
end do:

# Двойной цикл (для SPH-сумм)
for i from 1 to N do
  sum_ := 0;
  for j from 1 to N do
    sum_ := sum_ + m[j] * W_kernel(abs(x[i]-x[j])/h) / h;
  end do;
  rho[i] := sum_;
end do:

while ... do ... end do основное

while условие do ... end do

Цикл с предусловием. Основное применение в SPH — главный цикл по времени.

t := 0;
n_steps := 0;
while t < t_end do
  compute_density();
  compute_accel_visc();
  dt := compute_dt();
  if t + dt > t_end then dt := t_end - t; end if;
  leapfrog_step(dt);
  t := t + dt;
  n_steps := n_steps + 1;
end do:
Осторожно с бесконечными циклами. Если в теле есть опечатка и условие никогда не становится ложным — Maple «зависает». Спасение: Tools → Stop в меню интерфейса или Ctrl+Break.

seq основное

seq(expr, i=A..B) seq(expr, i in список)

Генерирует последовательность значений. Главное применение — создание точек для графиков и инициализация массивов.

# Список квадратов
[seq(i^2, i=1..5)];      # [1, 4, 9, 16, 25]

# Точки (x, y) для графика
data := [seq([x[i], rho[i]], i=1..N)];
plot(data, style=line, color=blue);

# Условное включение
[seq(if rho[i] > 0.5 then i else NULL end if, i=1..N)];
# NULL в списке автоматически исчезает

add основное

add(expr, i=A..B) add(expr, i in список)

Суммирует выражение по указанному диапазону. Предпочитайте add перед sum для численных сумм: sum пытается найти замкнутую формулу символьно (что часто невозможно и медленно).

# Полная масса
M_total := add(m[i], i=1..N);

# L1-ошибка
L1 := add(abs(rho[i] - rho_exact(x[i])), i=1..N) / N;

# Полный импульс
P_total := add(m[i]*v[i], i=1..N);
Разница add vs sum. 
add(1/k, k=1..100);      # численно: 5.187... (быстро)
sum(1/k, k=1..100);      # символьно: огромная дробь (медленно)
sum(1/k^2, k=1..infinity); # Pi^2/6 (работает только у sum)
В численных циклах по SPH всегда используйте add.

Математические функции

Элементарные функции math

abs, sign, sqrt, exp, ln, log10 sin, cos, tan, arcsin, arccos, arctan min, max

Все стандартные функции работают как ожидается. Важные детали:

  • abs(x) — модуль (для чисел и выражений).
  • sign(x) — знак: −1, 0, или +1. В SPH используется для знака градиента ядра в 1D.
  • sqrt(x) — квадратный корень. Maple знает что sqrt(2) — иррациональное число; используйте evalf(sqrt(2)) для численного значения.
  • ln(x) — натуральный логарифм; log10(x) — десятичный.
  • min(a, b, c, ...) и max(...) — переменное число аргументов.
c_sound[i] := sqrt(gamma_g * P[i] / rho[i]);    # c = sqrt(γP/ρ)
mu := h * vdotr / (dxij^2 + eta^2);
if sign(dxij) < 0 then ... end if;

# Макс по массиву — через цикл или с помощью seq/max
maxVC := max(seq(c_sound[i] + abs(v[i]), i=1..N));

Константы math

Основные математические константы:

Pi
— π (именно с большой буквы). pi — это просто имя.
exp(1)
— число e. Отдельного имени E или e нет.
I
— мнимая единица (i).
infinity
— бесконечность.
sigma2D := 10 / (7*Pi);    # правильно
sigma2D := 10 / (7*pi);    # НЕПРАВИЛЬНО (pi — это просто символ)

evalf(Pi);                  # 3.141592654
evalf(exp(1));             # 2.718281828

piecewise — условные выражения

piecewise math

piecewise(cond1, val1, cond2, val2, ..., condN, valN, else_val)

Создаёт выражение, которое принимает разные значения в зависимости от условий. В отличие от if...then...else, это выражение можно дифференцировать, интегрировать и строить как график — незаменимо для SPH-ядер.

# Кубический B-сплайн как piecewise (для символьной работы)
W_sym := (q) -> piecewise(
  q < 0,  0,                                      # отрицательные (не должны приходить)
  q < 1,  sigma1D*(1 - 3*q^2/2 + 3*q^3/4),       # ядро: 0 ≤ q < 1
  q < 2,  sigma1D*(2-q)^3/4,                      # хвост: 1 ≤ q < 2
  0);                                             # иначе

# Теперь можно:
plot(W_sym(q), q=0..3);                          # — построить
diff(W_sym(q), q);                               # — дифференцировать
int(W_sym(q), q=0..2);                           # — интегрировать

# Точное решение Sod для верификации
rho_exact := (xx) -> piecewise(
  xx < 0.263,  1.0,
  xx < 0.486,  1.0 + (0.4263 - 1.0)*(xx - 0.263)/(0.486 - 0.263),
  xx < 0.685,  0.4263,
  xx < 0.850,  0.2656,
  0.125);
piecewise vs proc с if. Правило: если функцию нужно интегрировать, дифференцировать или строить через plot(f(q), q=0..3) — используйте piecewise. Если функция вызывается только изнутри численных циклов — можно и proc с if, оно быстрее на большое количество вызовов.

if ... then ... elif ... else ... end if язык

if cond1 then ... elif cond2 then ... else ... end if

Обычная императивная условная конструкция (используется внутри процедур). Не путать с piecewise: if возвращает значение только при вызове, не является символьным выражением.

# Внутри процедуры — используем if
W_kernel := proc(q) local qq;
  qq := abs(q);
  if qq >= 2 then return 0
  elif qq >= 1 then return sigma1D*(2-qq)^3/4
  else return sigma1D*(1 - 3*qq^2/2 + 3*qq^3/4)
  end if;
end proc:
Частая ошибка. Maple требует end if (два слова, пробел), а не endif. Аналогично end do, end proc. Это отличается от C-подобных языков.

Символьные вычисления

int math

int(expr, x) — неопределённый int(expr, x=a..b) — определённый int(expr, x=a..b, numeric) — численно

Аналитическое (символьное) интегрирование. Используется в лабораторной 1 для проверки нормировки ядра.

# Неопределённый интеграл
F := int(sin(x), x);               # -cos(x)

# Определённый: проверка нормировки 1D ядра
Wbar := (q) -> piecewise(q < 1, 1 - 3*q^2/2 + 3*q^3/4,
                         q < 2, (2-q)^3/4, 0);

I1 := 2 * int(Wbar(q), q=0..2);     # вернёт 3/2
printf("σ·I = %a\n", (2/3) * I1);  # должно быть 1

# Если аналитика не берётся — численно
J := int(exp(-x^2)*sin(x), x=0..10, numeric);
Вычислительная стоимость. Символьное интегрирование может быть очень медленным или вообще не завершиться. Если Maple думает больше 10 секунд — прерывайте и используйте int(..., numeric).

diff math

diff(expr, x) diff(expr, x$n) — n-я производная diff(expr, x, y) — смешанная

Символьное дифференцирование. Полезно для получения производной ядра, если вы записали ядро через piecewise:

W_sym := (q) -> piecewise(q < 1, sigma1D*(1 - 3*q^2/2 + 3*q^3/4),
                         q < 2, sigma1D*(2-q)^3/4, 0);
dW_sym := diff(W_sym(q), q);

# Вторая производная
d2W := diff(W_sym(q), q$2);

# Сравнить с вручную выведенной формулой:
simplify(dW_sym - dW_manual);              # должно быть 0

simplify math

simplify(expr) simplify(expr, assume=...)

Упрощает выражение, применяя все доступные алгебраические тождества. Очень полезно после int или diff — выход часто содержит избыточные конструкции.

res := int(cos(x)^2, x=0..Pi);        # → 1/2 · Pi
simplify(res);                       # Pi/2

# С предположениями о знаках
simplify(sqrt(x^2), assume=positive);    # x (не |x|)

coeff math

coeff(poly, x) — коэффициент при x¹ coeff(poly, x, n) — при xⁿ coeff(poly, x, 0) — свободный член

Извлекает коэффициент при степени переменной. Используется в лаб 5 для получения угла наклона линейной регрессии.

fit := Statistics[LinearFit]([1, x], xdata, ydata, x);
# fit имеет вид a + b*x
intercept := coeff(fit, x, 0);      # a
slope     := coeff(fit, x, 1);      # b

# p-порядок сходимости
p := -slope;

Графика

with(plots) графика

Перед использованием расширенных команд графики (display, textplot и т.д.) нужно подключить пакет:

with(plots):   # двоеточие — чтобы не выводить длинный список имён

Базовая команда plot доступна без with(plots).

plot графика

plot(expr, x=a..b, options) plot([список точек], options) plot([expr1, expr2, ...], x=a..b, options)

Построить одномерный график. Три типа первого аргумента:

expr
— выражение от x: plot(sin(x), x=0..2*Pi).
[[x1,y1],[x2,y2],...]
— список точек.
[expr1,expr2,...]
— несколько кривых на одном графике.

Основные опции:

plot(data, style=line|point|pointline|patchnogrid,
           color=red|blue|"DarkGreen"|"DarkOrange",
           thickness=N,                       # 1 (тонкая) до 5
           symbol=circle|box|cross|diamond,
           symbolsize=N,                      # 5..20
           legend="подпись",
           labels=["x-ось","y-ось"],
           title="Название",
           view=[xmin..xmax, ymin..ymax]);
# График функции
plot(sin(x), x=0..2*Pi, color=blue, thickness=2);

# Точки SPH + эталон
p_SPH := plot([seq([x[i], rho[i]], i=1..N)],
              style=point, symbol=circle, symbolsize=5,
              color=red, legend="SPH"):
p_ex := plot(rho_exact(xx), xx=0..1, color=black,
             thickness=2, legend="exact"):
display(p_ex, p_SPH, title="ρ(x) при t=0.2");

display графика

display(p1, p2, ..., options) display(Array([p1, p2, ...]))

Объединяет несколько графических объектов в один рисунок. Без display нельзя положить точки и кривую на одни оси.

with(plots):

p1 := plot(sin(x), x=0..2*Pi, color=blue, legend="sin"):
p2 := plot(cos(x), x=0..2*Pi, color=red, legend="cos"):
display(p1, p2);                         # обе кривые на одном графике

# Сетка из нескольких графиков
display(Array([p_rho, p_P, p_v]));        # три графика рядом

# Переопределить опции объединённого
display(p1, p2, title="Сравнение", labels=["x","f"]);

textplot графика

textplot([[x, y, "label"], ...], options)

Добавляет текстовые надписи в точках графика. В SPH лабах — для пометки волн разрежения, контактного разрыва и ударной волны.

labels := textplot([
  [0.37, 0.75, "ВР"],
  [0.70, 0.35, "КР"],
  [0.87, 0.20, "УВ"]
], font=[TIMES, BOLD, 12]):

display(p_rho, labels);          # надписи поверх графика

Статистика и регрессия

Statistics[LinearFit] math

Statistics[LinearFit](базис, xdata, ydata, var)

Линейная регрессия методом наименьших квадратов. Возвращает выражение в виде полинома от var. В лаб 5 используется для определения порядка сходимости.

базис
— список базисных функций, например [1, x] для y = a + b·x.
xdata, ydata
— списки иксов и игреков одинаковой длины.
var
— имя переменной в возвращаемом выражении.
with(Statistics):

# Данные: log(N) vs log(L1-ошибка)
Ns := [100, 200, 400, 800, 1600]:
L1s := [0.012, 0.0065, 0.0034, 0.0017, 0.00086]:
xd := [seq(log10(Ns[k]), k=1..5)]:
yd := [seq(log10(L1s[k]), k=1..5)]:

fit := LinearFit([1, x], xd, yd, x);
# fit имеет вид: a - p*x

slope := coeff(fit, x, 1);
printf("Порядок сходимости: p ≈ %.3f\n", -slope);
# Ожидается p ≈ 1 для SPH на разрывах

Производительность

time язык

time() — в данный момент, сек CPU time(expr) — время выполнения expr

Измерение времени — необходимо для сравнения вариантов (напр. ядра в лаб 5).

# Способ 1: засечь начало и конец
t0 := time();
run_sod_kernel("quintic");
printf("Время: %.2f сек\n", time() - t0);

# Способ 2: одной командой
time(compute_density());

Compile (ускорение) язык

cproc := Compile(procedure)

Компилирует процедуру в нативный код. Может дать ускорение в 10-100 раз для численных циклов, но работает только для процедур с примитивными типами (float, integer) и без if в сложных местах.

Не рекомендую в учебной работе — слишком много ограничений и подводных камней (Compile ругается на разные вещи). Лучше оптимизируйте логику алгоритма.

# Простая процедура, пригодная для компиляции
fast_dot := proc(a::Array(datatype=float[8]),
                  b::Array(datatype=float[8]), n::integer)::float;
  local i, s;
  s := 0.0;
  for i from 1 to n do
    s := s + a[i]*b[i];
  end do;
  return s;
end proc:

fast_dot_c := Compile(fast_dot):    # может не получиться!

printf / sprintf язык

printf("format", args...) sprintf("format", args...)

Форматированный вывод. Синтаксис как в C: %d — целое, %f — с фикс. точкой, %e — научная форма, %.4f — 4 знака, %a — автоматический формат Maple, \n — перевод строки.

printf("N=%d, L1 = %.4e\n", N, L1);
# Вывод: N=400, L1 = 1.2340e-03

printf("rho[x=0.2] = %.4f  (ожид. 1.0)\n", rho[80]);
# Вывод: rho[x=0.2] = 0.9987  (ожид. 1.0)

# sprintf возвращает строку, не выводит
s := sprintf("step %d", n_steps);

Частые ошибки и как их избежать

Символьная арифметика вместо численной

Симптом: расчёт занимает 30+ минут вместо 1-2 минут. В выводах типа rho[200] видите длинные рациональные дроби вместо чисел.

Причина: где-то в инициализации использовали 2/3 вместо 2.0/3.0 или 1/N вместо 1.0/N. Maple поддерживает точную арифметику и весь расчёт идёт в рациональных числах.

Лечение: везде в численной части писать числа с точкой: sigma1D := 2.0/3.0, dx := 1.0/N, h := 2.0*dx. Для проверки — type(rho[1], float) должно возвращать true.

Забыли local — глобальные переменные в процедуре

Симптом: после запуска процедуры переменные в основном коде меняют значения неожиданно.

Причина: без объявления local i, sum_, ... все переменные внутри процедуры — глобальные. Если процедура использует имя i как счётчик цикла, она перезапишет любое i во внешнем коде.

Лечение: всегда первая строка процедуры после proc(...) — local v1, v2, ... со всеми временными переменными. Maple выдаст предупреждение типа Warning, `i` is implicitly declared local — его надо фиксить явным объявлением.

Array vs table vs list

Симптом: медленный доступ к элементам, странные ошибки типа «cannot assign to value».

Причина: в Maple есть три похожих структуры:
  • [a, b, c] — список (list), неизменяемый: L[1] := 5 не работает;
  • table([1=a, 2=b]) — ассоциативный массив, медленный для численных операций;
  • Array(1..N) — числовой массив, эффективный.
Лечение: для SPH-данных всегда используйте Array. Если при инициализации у вас получился список (например, через seq), превратите: a := Array([seq(f(i), i=1..N)]).

end if / end do / end proc

Симптом: ошибка syntax error, unexpected `fi` или end of input.

Причина: Maple исторически поддерживает два стиля: старый (fi, od) и новый (end if, end do). Современный синтаксис — новый, с пробелом.

Лечение: везде пишите end if (два слова), end do, end proc. Никогда endif или enddo — это синтаксическая ошибка.

Перепутаны := и =

Симптом: переменная не получает значение, в выводе появляются «нерезолвнутые» выражения.

Причина: написали x = 5 вместо x := 5. Maple понял первое как уравнение, не присваивание.

Лечение: правило — внутри процедур и для определения значений всегда :=. = используется только в определении уравнений для solve, fsolve, и в опциях функций (color=blue, x=0..1).

Забыли with(plots) перед display

Симптом: Error, unable to evaluate display, хотя plot работает.

Причина: display, textplot, pointplot, animate живут в пакете plots. Без его подключения они не видны.

Лечение: первой строкой секции графики: with(plots):.

Знак sign в градиенте ядра

Симптом: ускорения получаются перевёрнутыми, либо вообще нули. Ударная волна «разворачивается» назад.

Причина: в 1D-градиенте ядра есть член sign(x_i - x_j). Если забыли или поставили неправильный знак, симметричность пары \((i,j)\) пропадёт.

Лечение: формула: 
dWij := dW_kernel(qij) * sign(x[i]-x[j]) / h^2;
# dW_kernel(q) уже содержит знак производной (отрицательный)
# sign(x[i]-x[j]) — знак направления от j к i
Проверка: при малом сближении частиц \(i\) и \(j\) ускорение \(i\) должно быть направлено от \(j\) (отталкивание), то есть a[i] > 0 если \(x_i > x_j\).

Забыли обрезать сумму по радиусу ядра

Симптом: расчёт работает правильно, но в 10 раз медленнее ожидаемого.

Причина: внутри двойного цикла считаете \(W_{ij}\) для всех пар \((i,j)\), хотя ядро 0 при \(q > 2\). Впустую работают большинство итераций.

Лечение: сразу после вычисления \(q_{ij}\) — проверяйте: 
qij := abs(x[i]-x[j])/h;
if qij < 2 then                 # только внутри носителя
  s := s + m[j] * W_kernel(qij)/h;
end if;
Ускорение ~N/5 раз, потому что носитель ядра охватывает ~4-5 частиц.

Divergence при Leap-Frog без полушага

Симптом: после 10-20 шагов по времени решение «разваливается», появляются NaN или огромные значения.

Причина: забыли инициализировать \(v^{1/2}\) через полушаг после того как вычислили \(a^0\). Без этого первый leapfrog_step применяет \(a^0\) к текущим \(v^0\), получая \(v^1\) вместо \(v^{1/2}\), и далее вся фазовая синхронизация сбита.

Лечение: перед главным циклом: 
compute_density();
compute_eos();
compute_accel();          # получили a^0
dt0 := compute_dt();
for i from 1 to N do
  v[i] := v[i] - 0.5 * dt0 * a[i];   # сдвиг на полшага НАЗАД
end do:
# теперь v[i] = v^{-1/2}, и первый leapfrog_step(dt0) даст v^{1/2} правильно
