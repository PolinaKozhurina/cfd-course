Leap-Frog интегратор и первый запуск SPH-солвера

Цель: собрать процедуры из лаб 2 в работающий солвер теста Сода. Получить решение при \(t_{\text{end}} = 0.2\), проверить качественную картину (три волны) и законы сохранения. Осцилляции давления на ударной волне — ожидаемая патология без вязкости, её устранение в лаб 4.

К занятию: схема Leap-Frog (3.2), CFL-условие (3.4), алгоритм шага.

Задание 3.1. Leap-Frog и адаптивный шаг

ЗАДАНИЕ 3.1

Процедуры compute_dt и leapfrog_step

Написать две процедуры:

  • compute_dt() — найти \(\max_i(c_i + |v_i|)\) и вычислить шаг по формуле (3.4);
  • leapfrog_step(dt) — обновить \(v_i\) и \(x_i\) по формулам (3.2).

Для инициализации выполнить полушаг: \(v^{1/2}_i = v^0_i + \tfrac{1}{2}\Delta t^0\cdot a^0_i\), где \(\Delta t^0\) — CFL-шаг на начальных данных.

# --- Задание 3.1: CFL и Leap-Frog ---
# (продолжение кода лаб 2: массивы x, v, rho, P, c_sound, a, m уже существуют)

C0 := 0.3;  # CFL-число (без вязкости)

compute_dt := proc()
  local i, maxVC;
  maxVC := 0;
  for i from 1 to N do
    if c_sound[i] + abs(v[i]) > maxVC then
      maxVC := c_sound[i] + abs(v[i]);
    end if;
  end do;
  return C0 * h / maxVC;
end proc:

leapfrog_step := proc(dt)
  local i;
  # v^{n+1/2} = v^{n-1/2} + dt * a^n
  for i from 1 to N do
    v[i] := v[i] + dt * a[i];
  end do;
  # x^{n+1} = x^n + dt * v^{n+1/2}
  for i from 1 to N do
    x[i] := x[i] + dt * v[i];
  end do;
end proc:

# Инициализация: полушаг
compute_density();
compute_eos();
compute_accel();
dt0 := compute_dt();
printf("dt0 = %.4e\n", dt0);
# Сдвиг на полшага назад (фиктивная v^{-1/2} = v^0 - 0.5·dt0·a^0)
# После этого первый leapfrog_step(dt0) даст v^{1/2} правильно
for i from 1 to N do
  v[i] := v[i] - 0.5 * dt0 * a[i];
end do:

Задание 3.2. Главный цикл

ЗАДАНИЕ 3.2

Цикл по времени до \(t_{\text{end}} = 0.2\)

Написать цикл while t < t_end, в теле которого последовательно:

  1. compute_density
  2. compute_eos (давление и скорость звука)
  3. compute_accel
  4. compute_dt (получить \(\Delta t\))
  5. leapfrog_step(\(\Delta t\)) — обновить v и x
  6. увеличить t := t + \(\Delta t\)

Если \(t + \Delta t > t_{\text{end}}\), уменьшить \(\Delta t\) до \(t_{\text{end}} - t\). Вывести число выполненных шагов и средний \(\Delta t\).

# --- Задание 3.2: Главный цикл ---
t := 0;
t_end := 0.2;
n_steps := 0;
sum_dt := 0;
start_time := time();

while t < t_end do
  compute_density();
  compute_eos();
  compute_accel();
  dt := compute_dt();
  if t + dt > t_end then dt := t_end - t; end if;
  leapfrog_step(dt);
  t := t + dt;
  n_steps := n_steps + 1;
  sum_dt := sum_dt + dt;
  # Диагностика каждые 100 шагов
  if n_steps mod 100 = 0 then
    printf("Step %d: t = %.4f, dt = %.4e\n", n_steps, t, dt);
  end if;
end do:

printf("\nЗавершено за %d шагов, среднее Δt = %.4e\n",
       n_steps, sum_dt/n_steps);
printf("Время расчёта: %.2f сек\n", time() - start_time);
Ожидаемая производительность. При N=400 расчёт до t=0.2 в Maple обычно занимает 1-3 минуты (зависит от версии). Число шагов \(\sim 400-600\), средний \(\Delta t \sim 3\cdot 10^{-4}\). Если расчёт занимает 30+ минут — проблема в циклах (возможно используете символьные \(x[j]\) вместо Array). Ускорение достигается через Compile() процедур или переход на Matrix-нотацию.

Задание 3.4. Законы сохранения

ЗАДАНИЕ 3.4

Проверка сохранения массы и импульса

Вычислить:

  • полную массу \(M(t) = \sum_i m_i\),
  • полный импульс \(\mathcal P(t) = \sum_i m_i v_i\).

Сравнить значения при \(t = 0\) и \(t = 0.2\). Убедиться:

  • Масса сохраняется точно: \(M(0.2) = M(0)\), так как \(m_i = \mathrm{const}\).
  • Импульс сохраняется с точностью до машинного \(\epsilon\) (\(|\Delta \mathcal P/\mathcal P_0| < 10^{-12}\)) благодаря симметричной форме (2.4). В задаче Сода \(\mathcal P_0 = 0\), поэтому \(\mathcal P(t) \approx 0\) всё время.
# --- Задание 3.4: Сохранение массы и импульса ---

M_total := add(m[i], i=1..N);
P_total := add(m[i]*v[i], i=1..N);

printf("\n=== Законы сохранения при t=%.3f ===\n", t);
printf("Полная масса M = %.15f\n", evalf(M_total));
printf("Полный импульс P = %.4e (ожид. ~ 0)\n", evalf(P_total));

# Для сравнения: начальные значения (до цикла — надо сохранить)
# M_0 = 0.5 * 1.0 + 0.5 * 0.125 = 0.5625 (полмассы плотной + полмассы разреженной)
printf("Ожид. M_0 = %.15f\n", 0.5 * 1.0 + 0.5 * 0.125);
printf("Δ M = %.4e\n", evalf(abs(M_total - 0.5625)));
Проверка сохранения. Полная масса должна совпадать с ожидаемой 0.5625 с точностью до машинного эпсилона (\(\Delta M \sim 10^{-15}\)) — массы вообще не меняются со временем. Полный импульс в начале 0 (все скорости нулевые), а в конце должен быть близок к 0 (\(|P| \lesssim 10^{-10}\)) — несовершенство из-за симметрии (частицы у краев не имеют равных по обе стороны соседей), но на порядки меньше любой физической величины.
