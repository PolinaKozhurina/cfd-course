Искусственная вязкость, верификация и сходимость

Цель: добавить в наш SPH-код искусственную вязкость Монагана (α=1, β=2), получить решение задачи Сода без осцилляций, сравнить с эталонным решением Торо, проверить сходимость при уменьшении шага.

К занятию: формулы (4.1)-(4.3), эталонные значения.

Задание 4.1. Полный солвер с вязкостью

ЗАДАНИЕ 4.1

Модифицировать compute_accel

В процедуру compute_accel из лаб 2 добавить вычисление \(\mu_{ij}\) (4.3) и \(\Pi_{ij}\) (4.2), использовать формулу (4.1). Модифицировать compute_dt по (4.5).

Запустить тест Сода при \(\alpha = 1\), \(\beta = 2\), \(t_{\text{end}} = 0.2\).

# --- Задание 4.1: Солвер с искусственной вязкостью ---
# (продолжение кода лаб 3; массивы x, v, rho, P, c_sound, a, m уже существуют)

alpha_visc := 1.0;
beta_visc  := 2.0;
eta_visc   := 0.01 * h;

# Процедура ускорения с вязкостью
compute_accel_visc := proc()
  local i, j, sum_a, dxij, dvij, qij, dWij, vdotr, mu_ij, c_bar, rho_bar, Pi_ij;
  for i from 1 to N do
    sum_a := 0;
    for j from 1 to N do
      if i <> j then
        dxij := x[i] - x[j];
        qij := abs(dxij) / h;
        if qij < 2 then
          dWij := dW_kernel(qij) * sign(dxij) / h^2;
          # Вязкость: включается при сжатии пары
          dvij := v[i] - v[j];
          vdotr := dvij * dxij;
          if vdotr < 0 then
            mu_ij  := h * vdotr / (dxij^2 + eta_visc^2);
            c_bar  := (c_sound[i] + c_sound[j]) / 2;
            rho_bar:= (rho[i] + rho[j]) / 2;
            Pi_ij  := (-alpha_visc*c_bar*mu_ij + beta_visc*mu_ij^2) / rho_bar;
          else
            Pi_ij := 0;
          end if;
          sum_a := sum_a - m[j] * (P[i]/rho[i]^2 + P[j]/rho[j]^2 + Pi_ij) * dWij;
        end if;
      end if;
    end do;
    a[i] := sum_a;
  end do;
end proc:

# Модифицированный CFL с учётом вязкости
compute_dt_visc := proc()
  local i, maxVC;
  maxVC := 0;
  for i from 1 to N do
    if (1+alpha_visc)*c_sound[i] + abs(v[i]) > maxVC then
      maxVC := (1+alpha_visc)*c_sound[i] + abs(v[i]);
    end if;
  end do;
  return 0.2 * h / maxVC;
end proc:

# --- Инициализация (повтор из лаб 2 для полноты) ---
for j from 1 to N do
  x[j] := (j - 0.5) * dx;
  v[j] := 0;
  if x[j] < 0.5 then rho[j] := 1.0; P[j] := 1.0;
  else rho[j] := 0.125; P[j] := 0.1; end if;
  m[j] := rho[j] * dx;
end do:

compute_density();
compute_eos();
compute_accel_visc();
dt0 := compute_dt_visc();
# Сдвиг на полшага назад
for i from 1 to N do v[i] := v[i] - 0.5 * dt0 * a[i]; end do:

# --- Главный цикл ---
t := 0;
t_end := 0.2;
n_steps := 0;
while t < t_end do
  compute_density();
  compute_eos();
  compute_accel_visc();
  dt := compute_dt_visc();
  if t + dt > t_end then dt := t_end - t; end if;
  leapfrog_step(dt);
  t := t + dt;
  n_steps := n_steps + 1;
end do:

printf("Завершено за %d шагов\n", n_steps);
Сравнение с лабораторной 3. Шагов стало больше (в \(\sim 1.5\) раза) — это следствие (4.5). Зато в решении больше нет осцилляций давления.

Задание 4.2. Верификация по эталонному решению

ЗАДАНИЕ 4.2

Построить точное решение кусочно и сравнить

Построить на одном графике \(\rho^{\text{SPH}}(x)\) (точки) и точное решение (сплошная линия). Аналогично для \(P(x)\) и \(v(x)\).

Точное решение (кусочно):

  • \(\rho = 1\) при \(x < 0.263\);
  • линейный переход \(1 \to 0.4263\) при \(0.263 \le x \le 0.486\) (ВР);
  • \(\rho = 0.4263\) при \(0.486 < x < 0.685\);
  • \(\rho = 0.2656\) при \(0.685 \le x < 0.850\);
  • \(\rho = 0.125\) при \(x \ge 0.850\).

Аналогично для \(P\) (с другими значениями) и \(v\) (с другими значениями). Подписать на графиках «ВР», «КР», «УВ».

# --- Задание 4.2: Верификация ---

# Точное решение как piecewise-функции
rho_exact := proc(xx)
  if xx < 0.263 then 1.0
  elif xx <= 0.486 then 1.0 + (0.4263 - 1.0) * (xx - 0.263) / (0.486 - 0.263)
  elif xx < 0.685 then 0.4263
  elif xx < 0.850 then 0.2656
  else 0.125 end if;
end proc:

P_exact := proc(xx)
  if xx < 0.263 then 1.0
  elif xx <= 0.486 then 1.0 + (0.3031 - 1.0)*(xx - 0.263)/(0.486 - 0.263)
  elif xx < 0.850 then 0.3031
  else 0.1 end if;
end proc:

v_exact := proc(xx)
  if xx < 0.263 then 0
  elif xx <= 0.486 then 0.9274 * (xx - 0.263)/(0.486 - 0.263)
  elif xx < 0.850 then 0.9274
  else 0 end if;
end proc:

with(plots):
# Плотность
p_SPH_rho := plot([seq([x[i], rho[i]], i=1..N)], style=point,
                  symbol=circle, symbolsize=5, color=red, legend="SPH"):
p_exact_rho := plot(rho_exact(xx), xx=0..1, color=black, thickness=2,
                    legend="exact (Toro)"):
p_rho_full := display(p_exact_rho, p_SPH_rho,
  title="ρ(x) при t=0.2 (Sod, SPH vs exact)",
  labels=["x","ρ"]):

# То же для P и v
p_SPH_P := plot([seq([x[i], P[i]], i=1..N)], style=point,
                symbol=circle, symbolsize=5, color=red):
p_exact_P := plot(P_exact(xx), xx=0..1, color=black, thickness=2):
p_P_full := display(p_exact_P, p_SPH_P, title="P(x) при t=0.2",
                    labels=["x","P"]):

p_SPH_v := plot([seq([x[i], v[i]], i=1..N)], style=point,
                symbol=circle, symbolsize=5, color=red):
p_exact_v := plot(v_exact(xx), xx=0..1, color=black, thickness=2):
p_v_full := display(p_exact_v, p_SPH_v, title="v(x) при t=0.2",
                    labels=["x","v"]):

# Вывод всех трёх
display(Array([p_rho_full, p_P_full]));
display(p_v_full);

# Аннотации ВР/КР/УВ можно добавить через textplot:
labels_plot := textplot([
  [0.37, 0.75, "ВР", color=blue],
  [0.70, 0.35, "КР", color="DarkGreen"],
  [0.87, 0.20, "УВ", color=red]
], font=[TIMES,BOLD,12]):
display(p_rho_full, labels_plot);
Ожидаемое качество. Численные точки должны лежать строго на эталонной ломаной всюду, кроме: (а) ближайших 3-5 частиц к ударному фронту (размывание вязкостью), (б) пары частиц на контактном разрыве (SPH не может дать резкий скачок). Если есть отклонения более \(5\%\) на плато — ошибка в реализации вязкости.

Задание 4.3. Анализ сходимости

ЗАДАНИЕ 4.3

L1-норма ошибки при N = 100, 200, 400, 800

Вычислить L1-норму ошибки плотности:

\[ L_1 = \frac{1}{N}\sum_{i=1}^{N} |\rho^{\text{SPH}}_i - \rho^{\text{exact}}(x_i)|. \]

Повторить расчёт при \(N = 100, 200, 400, 800\) (при \(h = 2\Delta x\)). Построить \(\log L_1\) vs \(\log \Delta x\). Определить наклон — порядок сходимости. Ожидание: \(\sim 1\) (SPH с вязкостью на разрывном решении).

# --- Задание 4.3: Сходимость ---

run_sod := proc(Nx)
  local dx_l, h_l, t_l, dt_l, ns_l, L1, i, x_l, v_l, rho_l, P_l, c_l, m_l, a_l;
  # ... повторный запуск всего солвера для заданного Nx ...
  # (опущен для краткости; скопируйте весь код лаб 2-3-4 и уберите глобальные
  # переменные, передавая Nx как параметр)
  ...
  L1 := add(abs(rho_l[i] - rho_exact(x_l[i])), i=1..Nx) / Nx;
  return L1;
end proc:

# Запуск серии
Ns := [100, 200, 400, 800];
L1s := Array(1..4):
for k from 1 to 4 do
  L1s[k] := run_sod(Ns[k]);
  printf("N = %d, L1 = %.4e\n", Ns[k], L1s[k]);
end do:

# log-log график
pts := [seq([log10(1.0/Ns[k]), log10(L1s[k])], k=1..4)]:
p_conv := plot(pts, style=pointline, symbol=circle, symbolsize=10,
               color=blue, thickness=2, labels=["log(Δx)", "log(L1)"],
               title="Сходимость L1-ошибки плотности"):
# Наклон (линейная регрессия через 4 точки)
slope := evalf((log10(L1s[4]) - log10(L1s[1])) / (log10(1.0/Ns[4]) - log10(1.0/Ns[1])));
printf("Измеренный порядок сходимости: p ≈ %.3f\n", slope);

display(p_conv);
Практическая рекомендация. В Maple обёртка всего солвера в одну процедуру может быть громоздкой. Проще скопировать тело кода 4 раза и вручную менять \(N\) (или использовать unapply для ограниченного запуска). На \(N = 800\) один запуск может занимать 10+ минут — запаситесь временем или уменьшите время расчёта до \(t_{\text{end}} = 0.1\).

# --- Задание 4.4: Влияние α ---

count_front_particles := proc(rho_arr, x_arr, Nx)
  local i, count;
  count := 0;
  for i from 1 to Nx do
    if x_arr[i] > 0.7 and rho_arr[i] > 0.15 and rho_arr[i] < 0.40 then
      count := count + 1;
    end if;
  end do;
  return count;
end proc:

Ожидаемые значения.
  • \(\alpha = 0\): ширина \(\sim 5\)-\(7\) частиц, но видны осцилляции \(\pm 5\%\);
  • \(\alpha = 0.5\): ширина \(\sim 8\)-\(10\), осцилляции ослаблены;
  • \(\alpha = 2.0\): ширина \(\sim 14\)-\(18\), осцилляций нет, но фронт слишком размыт — точность падает.
