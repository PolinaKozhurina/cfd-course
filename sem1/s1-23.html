Метод Годунова–Колгана

Кусочно-постоянная (Годунов) vs кусочно-линейная (Колган) реконструкция. Наклоны определяют \(u_L, u_R\) на границах.

Мотивация

Классическая схема Годунова основана на кусочно-постоянной аппроксимации решения внутри ячеек расчётной сетки. Такой подход обеспечивает монотонность и устойчивость схемы, однако приводит лишь к первому порядку точности по пространству и значительной численной диффузии в областях гладкого решения.

Метод Годунова–Колгана предназначен для повышения порядка аппроксимации за счёт более точного представления решения внутри ячейки, при сохранении базовой структуры метода Годунова и использования решателя задачи Римана на границах ячеек.

Линейная реконструкция решения внутри ячейки

В методе Годунова–Колгана решение внутри каждой ячейки аппроксимируется линейной функцией:

\[ u(x, t_n)\big|_{x \in [x_{i-1/2}, x_{i+1/2}]} = u_i^n + \sigma_i(x - x_i), \tag{23.1} \]

где \(u_i^n\) — среднее значение в \(i\)-й ячейке, \(x_i\) — её центр, а \(\sigma_i\) — наклон линейной реконструкции. Такое представление обеспечивает второй порядок аппроксимации в областях гладкого решения при соответствующем выборе наклонов \(\sigma_i\).

Определение левых и правых состояний

Из линейной реконструкции определяются левое и правое состояния на границе ячейки:

\[ u_{i+1/2}^L = u_i^n + \frac{\Delta x}{2}\sigma_i, \qquad u_{i+1/2}^R = u_{i+1}^n - \frac{\Delta x}{2}\sigma_{i+1}. \tag{23.2} \]

Выбор наклонов

Наклон \(\sigma_i\) может быть определён как центральная разность:

\[ \sigma_i = \frac{u_{i+1}^n - u_{i-1}^n}{2\Delta x}. \]

Однако такой выбор не гарантирует монотонности. Для её обеспечения наклон ограничивается (лимитируется), что рассматривается в следующем параграфе.

Численный поток

Численный поток вычисляется по формуле

\[ F_{i+1/2} = F\bigl(u_{i+1/2}^L,\; u_{i+1/2}^R\bigr), \tag{23.3} \]

где \(F\) — численный поток, определяемый выбранным решателем задачи Римана (точным или приближённым). Таким образом, метод Годунова–Колгана сохраняет ключевую идею схемы Годунова — использование локальных задач Римана, но повышает точность за счёт более точной аппроксимации данных на границе ячеек.

Свойства

Метод Годунова–Колгана является консервативным и при корректном выборе наклонов обеспечивает второй порядок точности на гладких решениях. По сравнению с классической схемой Годунова он обладает существенно меньшей численной диффузией.

Замечание. Линейная реконструкция сама по себе не гарантирует монотонности, что требует дальнейшего развития метода. Эта проблема послужила основой для построения метода Годунова–Колгана–Родионова, рассматриваемого в следующем параграфе.

Интерактив 1: реконструкция на разных функциях

Ниже — визуализация того, как выглядит реконструкция \((23.1)\) для разных типов данных. Переключайте функцию и тип ограничителя — следите за тремя вещами:

  • Серые ступеньки — исходные данные \(u_i^n\) (средние по ячейке).
  • Цветные наклонные отрезки — реконструкция (23.1) внутри каждой ячейки. Видно, где наклон определён уверенно (гладкая функция), а где обнуляется ограничителем (экстремумы, разрывы).
  • Точки на гранях — значения \(u_L\) (слева от грани) и \(u_R\) (справа), которые подаются в решатель Римана (23.3).
- - точная функция ─ средние u_i по ячейкам / реконструкция с наклоном ● u_L, u_R на гранях

Интерактив 2: с реконструкцией vs без — где 2-й порядок?

Теперь — прямое численное сравнение: одинаковая начальная функция, одинаковый решатель Римана, одинаковое число ячеек. Разница — только в реконструкции. Слева (синий) — Годунов без реконструкции (σ=0, 1-й порядок). Справа (красный) — Годунов-Колган с выбранным ограничителем.

Сверху над каждой ячейкой нарисованы маркеры порядка: ═ зелёный горизонтальный = наклон = 0 (1-й порядок, ступенька), ∕ красный наклонный = наклон ≠ 0 (2-й порядок, линия внутри ячейки). Таким образом видно, где лимитер «откатывает» схему к 1-му порядку.

Шаг 0. В каждой ячейке над графиками — ═ если σ=0 (1-й порядок), ∕ если σ≠0 (2-й порядок).
- - точное — Годунов (σ=0, 1-й порядок) — Годунов-Колган (с реконстр.)
Упражнение. Запустите интерактив 2 на смешанной задаче, затем переключите ограничитель с minmod на «центральная» — видно как на разрыве появляются осцилляции (маркеры остаются красными везде, но решение теряет монотонность). Это и доказывает тезис §8: реконструкция 2-го порядка требует нелинейного ограничителя.
