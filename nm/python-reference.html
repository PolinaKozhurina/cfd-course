2. Массивы NumPy

Массив ndarray — основная структура данных всего научного стека. В отличие от списка Python, он хранит элементы одного типа подряд в памяти, что даёт быстрые векторизованные операции без явных циклов.

arrays_create.py▶ python
import numpy as np

a = np.array([1.0, 2.0, 3.0])        # из списка
np.arange(0, 1, 0.25)                # [0, 0.25, 0.5, 0.75] — шаг задан
np.linspace(0, 1, 5)                 # [0, 0.25, 0.5, 0.75, 1] — 5 точек, концы включены
np.zeros(4)                          # [0. 0. 0. 0.]
np.ones((2, 3))                      # матрица 2x3 из единиц
np.eye(3)                            # единичная матрица 3x3
np.full(3, 7.0)                      # [7. 7. 7.]

A = np.array([[1.0, 2.0],
              [3.0, 4.0]])
A.shape        # (2, 2)  — размеры
A.ndim         # 2       — число осей
A.dtype        # float64 — тип элементов

Для построения сеток и табулирования функций linspace удобнее arange: число точек задаётся явно и концы гарантированно попадают в сетку (у arange правый конец не включается и страдает от ошибок округления шага).

A[1, 2]              # элемент (строка 1, столбец 2)
A[:, 0]              # весь нулевой столбец
A[1, :]              # вся первая строка
A[-1, -1]            # правый нижний угол

Broadcasting — автоматическое согласование форм: массивы разной размерности «растягиваются» до общей формы без копирования данных. Векторизация — замена циклов поэлементными операциями над целыми массивами; она в десятки раз быстрее и читается ближе к математической записи.

vectorize.py▶ python
x = np.linspace(0, 2*np.pi, 1000)

# Векторизация: одна строка вместо цикла
y = np.sin(x) * np.exp(-x/5)         # поэлементно ко всем 1000 точкам

# Broadcasting: столбец (n,1) + строка (m,) -> матрица (n,m)
xi = np.linspace(0, 1, 4).reshape(-1, 1)   # (4,1)
xj = np.linspace(0, 1, 3)                  # (3,)
D = xi - xj                                # (4,3) — матрица разностей

# Анти-паттерн (медленно): тот же y явным циклом
y_slow = np.empty_like(x)
for i in range(x.size):
    y_slow[i] = np.sin(x[i]) * np.exp(-x[i]/5)

3. Линейная алгебра

Функции есть в двух модулях: numpy.linalg (базовый набор) и scipy.linalg (шире, включает lu_factor/lu_solve, разложения с дополнительными опциями). Для решения одной системы \(Ax=b\) — np.linalg.solve; никогда не считайте inv(A) @ b (медленнее и менее точно).

solve.py▶ python
import numpy as np

A = np.array([[3.0, 1.0],
              [1.0, 2.0]])
b = np.array([9.0, 8.0])

x = np.linalg.solve(A, b)            # решение Ax = b
print(x)                             # [2. 3.]
print(np.allclose(A @ x, b))         # True — проверка

Если систему с одной матрицей \(A\) надо решить для многих правых частей, разложите \(A\) один раз LU-факторизацией и переиспользуйте.

lu.py▶ python
from scipy.linalg import lu, lu_factor, lu_solve, cholesky

# Явное PA = LU (для изучения метода Гаусса)
P, L, U = lu(A)                      # A = P @ L @ U

# Практичный путь: факторизовать один раз, решать многократно
lu_piv = lu_factor(A)               # (LU, pivots)
x1 = lu_solve(lu_piv, b)
x2 = lu_solve(lu_piv, np.array([1.0, 0.0]))   # другая правая часть — без повторного разложения

# Симметричная положительно определённая матрица: Холецкий (вдвое дешевле)
S = np.array([[4.0, 1.0], [1.0, 3.0]])
Lc = cholesky(S, lower=True)        # S = Lc @ Lc.T

Нормы и обусловленность. np.linalg.norm по умолчанию даёт евклидову норму вектора (\(\ell_2\)) или фробениусову норму матрицы. Параметр ord выбирает другую норму. np.linalg.cond — число обусловленности матрицы.

norms.py▶ python
v = np.array([3.0, -4.0])
np.linalg.norm(v)             # 5.0      — l2 (по умолчанию)
np.linalg.norm(v, 1)          # 7.0      — сумма модулей
np.linalg.norm(v, np.inf)     # 4.0      — максимум модуля

M = np.array([[1.0, 2.0], [0.0, 2.0]])
np.linalg.norm(M, 2)          # спектральная норма (макс. синг. число)
np.linalg.norm(M, 1)          # макс. сумма по столбцам
np.linalg.norm(M, np.inf)     # макс. сумма по строкам
np.linalg.norm(M, 'fro')      # фробениусова

np.linalg.cond(M)             # число обусловленности (в норме 2)
np.linalg.cond(M, np.inf)     # в норме inf

# Относительная невязка как проверка качества решения:
r = np.linalg.norm(A @ x - b) / np.linalg.norm(b)

Собственные значения и разложения для задач устойчивости, МНК и анализа:

# Собственные значения / векторы
w = np.linalg.eigvals(A)             # только собственные значения
w, V = np.linalg.eig(A)             # значения w и векторы (столбцы V)
w = np.linalg.eigvalsh(S)           # для симметричных — быстрее и точнее

# Сингулярное разложение  A = U @ diag(s) @ Vh

# QR-разложение  A = Q @ R

# Линейный МНК: min ||A x - b|| для прямоугольной A

4. Интерполяция

Модуль scipy.interpolate строит функцию, проходящую через заданные узлы. Для гладких данных предпочтительна кубическая сплайн-интерполяция; полином Лагранжа высокой степени страдает от феномена Рунге и численно неустойчив.

interp_scipy.py▶ python
import numpy as np
from scipy.interpolate import (lagrange, BarycentricInterpolator,
                               CubicSpline, interp1d)

xi = np.linspace(0, 2*np.pi, 7)
yi = np.sin(xi)
xx = np.linspace(0, 2*np.pi, 200)

# Полином Лагранжа (только для малого числа узлов!)
p = lagrange(xi, yi)                 # numpy.poly1d
yy = p(xx)

# Барицентрическая форма — устойчивее при оценке в точках
B = BarycentricInterpolator(xi, yi)
yy = B(xx)

# Кубический сплайн — рабочая лошадка для гладких данных
cs = CubicSpline(xi, yi, bc_type='natural')
yy  = cs(xx)
dyy = cs(xx, 1)                      # производная сплайна
I   = cs.integrate(0, np.pi)         # интеграл сплайна

# Универсальный интерполятор (линейный / кубический)
f = interp1d(xi, yi, kind='cubic', fill_value='extrapolate')
yy = f(xx)

Для приближения многочленами и работы с их корнями/производными удобен пакет numpy.polynomial — современная замена старому np.polyfit/poly1d. Базис Чебышёва численно устойчив на отрезке.

polynomial.py▶ python
from numpy.polynomial import Polynomial as P, Chebyshev as C

# Аппроксимация по МНК многочленом степени 5
xd = np.linspace(-1, 1, 50)
yd = np.cos(3*xd) + 0.01*np.random.randn(50)
p = P.fit(xd, yd, deg=5)             # устойчивый МНК-фит

p(0.3)                               # значение в точке
p.roots()                            # корни многочлена
p.deriv()                            # производная (новый Polynomial)
p.integ()                            # первообразная

# Базис Чебышёва — устойчив при высоких степенях
c = C.fit(xd, yd, deg=12)
c.roots()

5. Численное интегрирование

Модуль scipy.integrate. Если есть функция \(f(x)\) — используйте адаптивную квадратуру quad (высокая точность, оценка ошибки). Если есть массив значений на сетке — формулы trapezoid и simpson.

integrate.py▶ python
import numpy as np
from scipy.integrate import quad, dblquad, trapezoid, simpson, fixed_quad

# 1) Функция задана аналитически: адаптивный quad
val, err = quad(lambda x: np.exp(-x**2), 0, np.inf)
print(val, err)                      # 0.8862269...  и оценка ошибки

# 2) Двойной интеграл по прямоугольнику
v2, e2 = dblquad(lambda y, x: x*y, 0, 1, 0, 2)   # внешний x:0..1, внутр. y:0..2

# 3) Данные на сетке (функции в явном виде нет)
x = np.linspace(0, np.pi, 101)
y = np.sin(x)
trapezoid(y, x)                      # метод трапеций       -> ~2.0
simpson(y, x)                        # метод Симпсона (точнее)

# 4) Квадратура Гаусса фиксированного порядка
vg, _ = fixed_quad(np.sin, 0, np.pi, n=5)   # 5-точечная Гаусса
Про romberg. Старая функция scipy.integrate.romberg объявлена устаревшей и удаляется. Для интегрирования функции используйте quad (адаптивная, с контролем ошибки); для массива на равномерной сетке есть scipy.integrate.romb. В новом коде по умолчанию — quad.

6. Корни и оптимизация

Модуль scipy.optimize. Для скалярного уравнения \(f(x)=0\) на отрезке со сменой знака надёжнее всего brentq (гарантированная сходимость + скорость). newton быстрее, но требует хорошего начального приближения и может расходиться.

roots.py▶ python
import numpy as np
from scipy.optimize import brentq, bisect, newton, root_scalar

f = lambda x: x**3 - 2*x - 5

# Скобочные методы: нужен интервал [a,b] с f(a)*f(b) < 0
brentq(f, 2, 3)                      # 2.0945514...  — рекомендуемый
bisect(f, 2, 3)                      # бисекция (надёжно, но медленно)

# Метод Ньютона: нужна стартовая точка (и желательно f')
newton(f, x0=2.0, fprime=lambda x: 3*x**2 - 2)

# Единый интерфейс с выбором метода и диагностикой
sol = root_scalar(f, bracket=[2, 3], method='brentq')
print(sol.root, sol.converged, sol.iterations)

Минимизация. Для функции одной переменной — minimize_scalar; для многомерной — minimize (метод BFGS при наличии гладкости, Nelder-Mead без производных). Для задач линейного программирования — linprog.

optimize.py▶ python
from scipy.optimize import minimize_scalar, minimize, linprog

# Скаляр: золотое сечение / метод Брента
r = minimize_scalar(lambda x: (x - 2)**2 + 1, method='brent')
print(r.x, r.fun)                    # 2.0  1.0

# Многомерная: функция Розенброка
rosen = lambda v: (1 - v[0])**2 + 100*(v[1] - v[0]**2)**2
m = minimize(rosen, x0=[0.0, 0.0], method='BFGS')        # градиентный
m = minimize(rosen, x0=[0.0, 0.0], method='Nelder-Mead') # без производных
print(m.x, m.fun, m.success)

# Линейное программирование: min c@x при A_ub x <= b_ub
c   = [-1, -2]                        # максимизируем x+2y -> min -(x+2y)
Aub = [[1, 1], [1, 0]]
bub = [4, 2]
lp = linprog(c, A_ub=Aub, b_ub=bub, bounds=[(0, None), (0, None)])
print(lp.x, -lp.fun)

7. Разреженные матрицы

Когда матрица почти вся из нулей (например, трёхдиагональная из конечных разностей), её хранят в разреженном формате scipy.sparse — экономия памяти и времени на порядки. Собирают матрицу удобно через diags, решают через spsolve.

sparse.py▶ python
import numpy as np
from scipy.sparse import diags, csr_array
from scipy.sparse.linalg import spsolve, cg

n = 1000
# Трёхдиагональная матрица: -1 на побочных, 2 на главной диагонали
main = 2.0*np.ones(n)
off  = -1.0*np.ones(n - 1)
A = diags([off, main, off], offsets=[-1, 0, 1], format='csr')

b = np.ones(n)
x = spsolve(A, b)                    # прямой разреженный решатель

# Итерационный решатель (сопряжённые градиенты) для симм. пол.-опр. A
x_it, info = cg(A, b, rtol=1e-10)
print('cg сошёлся' if info == 0 else 'cg не сошёлся')

# Явная сборка в формате CSR из строк/столбцов/значений:
rows = np.array([0, 1, 2])
cols = np.array([0, 1, 2])
vals = np.array([1.0, 2.0, 3.0])
D = csr_array((vals, (rows, cols)), shape=(3, 3))
Форматы. Для сборки матрицы поэлементно удобны lil/dok; для арифметики и решения — csr/csc. Конвертация: A.tocsr(). diags сразу строит нужный формат через аргумент format='csr'. Это тот самый случай, когда трёхдиагональную систему из метода прогонки решают «по-взрослому» одним вызовом spsolve.

8. Построение графиков

Matplotlib. Для графиков ошибок и порядков сходимости почти всегда нужны логарифмические оси: semilogy (степенное убывание ошибки превращается в прямую при логарифмической оси Y) и loglog (наклон прямой равен порядку метода \(p\)).

plot_basic.py▶ python
import numpy as np
import matplotlib.pyplot as plt

x = np.linspace(0, 2*np.pi, 200)

fig, ax = plt.subplots(figsize=(6, 4))
ax.plot(x, np.sin(x), label='sin x')
ax.plot(x, np.cos(x), '--', label='cos x')
ax.set_xlabel('x'); ax.set_ylabel('f(x)')
ax.set_title('Тригонометрические функции')
ax.legend(); ax.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()                           # в скрипте; в Jupyter не обязательно

Типичная задача курса — показать, как ошибка метода \(E(h)\) убывает при измельчении шага \(h\). На графике loglog зависимость \(E\sim C h^p\) выглядит прямой с наклоном \(p\). Для контроля рисуют референсные наклоны \(h^1, h^2\).

convergence.py▶ python
import numpy as np
import matplotlib.pyplot as plt

# Пример: ошибка центральной разности для f'(x), f = sin
hs = np.logspace(-1, -6, 12)         # шаги от 1e-1 до 1e-6
x0 = 1.0
err = np.abs((np.sin(x0 + hs) - np.sin(x0 - hs))/(2*hs) - np.cos(x0))

fig, ax = plt.subplots(figsize=(6, 4))
ax.loglog(hs, err, 'o-', label='ошибка центральной разности')
ax.loglog(hs, hs**2, '--', label=r'наклон $h^2$ (эталон)')
ax.set_xlabel('шаг h'); ax.set_ylabel('абсолютная ошибка')
ax.legend(); ax.grid(True, which='both', alpha=0.3)
ax.invert_xaxis()                    # часто удобнее: h убывает слева направо
plt.tight_layout(); plt.show()

# Оценка наблюдаемого порядка по двум точкам:
p = np.log(err[-2]/err[-1]) / np.log(hs[-2]/hs[-1])
print(f'наблюдаемый порядок p ≈ {p:.2f}')
наблюдаемый порядок p ≈ 2.00

Несколько панелей в одной фигуре делают через subplots(rows, cols); для убывающей по экспоненте величины (например, невязки итераций) подходит semilogy.

subplots.py▶ python
fig, axes = plt.subplots(1, 2, figsize=(10, 4))

# Левая панель: решение
axes[0].plot(x, np.sin(x))
axes[0].set_title('решение')

# Правая панель: убывание невязки итерационного метода
resid = 0.5**np.arange(20)           # модельные невязки
axes[1].semilogy(resid, 'o-')
axes[1].set_title('сходимость (semilogy)')
axes[1].set_xlabel('итерация'); axes[1].set_ylabel('невязка')
axes[1].grid(True, which='both', alpha=0.3)

plt.tight_layout(); plt.show()
Памятка по осям ошибок. Степенная ошибка \(E=Ch^p\) — линия на loglog с наклоном \(p\). Экспоненциальное убывание \(E=Cr^k\) (итерации, спектральные методы) — прямая на semilogy. Если кривая на loglog «утыкается» в горизонтальную полку у \(10^{-16}\) при малых \(h\) — это выход на уровень ошибок округления (см. занятие 1, машинное \(\varepsilon\)), а не дефект метода.

Полная документация: NumPy, SciPy, Matplotlib. Быстрая справка прямо в сессии: ?имя_функции в IPython/Jupyter.
