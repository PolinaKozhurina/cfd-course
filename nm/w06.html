Численное дифференцирование

Лекция A · ≤ 30 мин

Производная как предел и его дискретизация

Производная по определению — это предел разностного отношения:

\[ f'(x) = \lim_{h\to 0}\frac{f(x+h)-f(x)}{h}. \tag{6.1} \]

Заманчиво просто взять «достаточно малое» \(h\) и объявить дробь приближением производной. Но компьютер не умеет переходить к пределу: он умеет лишь подставить конкретное конечное \(h\). Возникает ошибка усечения (методическая) — мы заменили предел конечной формулой. А поскольку считаем в арифметике с плавающей точкой (занятие 1), к ней добавляется ошибка округления. Сегодня мы выведем разностные формулы, оценим их порядок и увидим неожиданный вывод: уменьшать \(h\) до машинного нуля категорически вредно.

Главный инструмент анализа — ряд Тейлора (занятие 2). Будем считать \(f\) достаточно гладкой в окрестности точки \(x\) и раскладывать значения в соседних узлах.

Односторонние разности: правая и левая

Разложим \(f(x+h)\) в ряд Тейлора с остаточным членом в форме Лагранжа:

\[ f(x+h) = f(x) + h\,f'(x) + \frac{h^2}{2}f''(\xi),\qquad \xi\in(x,\,x+h). \tag{6.2} \]

Выразим отсюда разностное отношение и получим правую (forward) разность:

\[ \frac{f(x+h)-f(x)}{h} = f'(x) + \underbrace{\frac{h}{2}f''(\xi)}_{\text{ошибка усечения}}. \tag{6.3} \]
Правая разность. Для дважды непрерывно дифференцируемой \(f\) \[ f'(x)\approx \frac{f(x+h)-f(x)}{h},\qquad \text{ошибка усечения } = -\frac{h}{2}f''(\xi) = O(h). \] Схема имеет первый порядок точности: при уменьшении \(h\) вдвое ошибка усечения падает примерно вдвое.

Симметрично, используя узел слева, разложим \(f(x-h)=f(x)-h f'(x)+\tfrac{h^2}{2}f''(\xi)\) и получим левую (backward) разность того же первого порядка:

\[ f'(x)\approx \frac{f(x)-f(x-h)}{h},\qquad \text{ошибка } = +\frac{h}{2}f''(\xi) = O(h). \tag{6.4} \]

Знаки ведущих членов ошибки у правой и левой разностей противоположны. Это наводит на мысль: если их усреднить, главные члены сократятся. Так и рождается центральная разность.

Центральная разность

Возьмём два разложения с членами до третьего порядка:

\[ \begin{aligned} f(x+h) &= f(x) + h f'(x) + \tfrac{h^2}{2}f''(x) + \tfrac{h^3}{6}f'''(x) + O(h^4),\\ f(x-h) &= f(x) - h f'(x) + \tfrac{h^2}{2}f''(x) - \tfrac{h^3}{6}f'''(x) + O(h^4). \end{aligned} \tag{6.5} \]

Вычитая второе из первого, члены с \(f(x)\) и \(f''(x)\) уничтожаются, остаётся нечётная часть:

\[ f(x+h)-f(x-h) = 2h\,f'(x) + \frac{h^3}{3}f'''(x) + O(h^5). \tag{6.6} \]
Центральная разность. Для трижды непрерывно дифференцируемой \(f\) \[ f'(x)\approx \frac{f(x+h)-f(x-h)}{2h},\qquad \text{ошибка усечения } = -\frac{h^2}{6}f'''(\xi) = O(h^2). \] Схема имеет второй порядок: при уменьшении \(h\) вдвое ошибка падает вчетверо. Та же стоимость (два вычисления \(f\)), что у разности «правая + левая», но точность качественно выше.
Рис. 1. Разностные формулы как секущие. Правая и левая секущие наклонены в разные стороны относительно истинной касательной; центральная секущая (через \(x-h\) и \(x+h\)) почти параллельна касательной — отсюда её второй порядок.

Вторая производная

Если в (6.5) разложения сложить, уничтожатся нечётные члены (с \(f'\) и \(f'''\)):

\[ f(x+h)+f(x-h) = 2f(x) + h^2 f''(x) + \frac{h^4}{12}f^{(4)}(x) + O(h^6). \tag{6.7} \]

Выражая \(f''(x)\), получаем симметричную трёхточечную формулу:

Вторая производная (трёхточечная). \[ f''(x)\approx \frac{f(x+h)-2f(x)+f(x-h)}{h^2},\qquad \text{ошибка усечения } = -\frac{h^2}{12}f^{(4)}(\xi) = O(h^2). \] Эта формула — рабочая лошадка для дискретизации уравнения теплопроводности и Пуассона (оператор Лапласа в МКР).

Метод неопределённых коэффициентов

Выводить каждую формулу через индивидуальные манипуляции с рядами утомительно. Универсальный приём: зафиксировать шаблон узлов \(x_i = x + s_i h\) и искать коэффициенты \(c_i\) так, чтобы линейная комбинация приближала нужную производную:

\[ f^{(k)}(x)\approx \frac{1}{h^k}\sum_{i} c_i\, f(x+s_i h). \tag{6.8} \]

Подставляем разложения Тейлора каждого узла и приравниваем коэффициенты при \(f(x),\,f'(x),\,f''(x),\dots\) тем, что должны получиться у \(f^{(k)}\). Получается линейная система на \(c_i\). Покажем на шаблоне \(\{x,\,x+h,\,x+2h\}\) для первой производной (правая разность второго порядка).

Пусть \(f'(x)\approx \tfrac1h\big(c_0 f(x)+c_1 f(x+h)+c_2 f(x+2h)\big)\). Разложения: \(f(x+h)=f+hf'+\tfrac{h^2}{2}f''+\dots\), \(f(x+2h)=f+2hf'+2h^2 f''+\dots\). Приравнивая коэффициенты при \(f\) (нуль), при \(hf'\) (единица) и при \(h^2 f''\) (нуль), получаем систему:

\[ \begin{cases} c_0 + c_1 + c_2 = 0,\\ c_1 + 2c_2 = 1,\\ \tfrac12 c_1 + 2 c_2 = 0, \end{cases} \qquad\Longrightarrow\qquad c_0=-\tfrac32,\ c_1=2,\ c_2=-\tfrac12. \tag{6.9} \]
Односторонняя разность 2-го порядка. \[ f'(x)\approx \frac{-3f(x)+4f(x+h)-f(x+2h)}{2h},\qquad \text{ошибка } = O(h^2). \] Полезна на границе области, где центральный шаблон недоступен (нет узла слева).
Общее правило. Шаблон из \(n+1\) узлов позволяет точно воспроизвести производные до порядка \(n\) включительно, то есть аппроксимировать \(f^{(k)}\) со схемой порядка не ниже \(n+1-k\). Симметричные шаблоны за счёт сокращения чётных/нечётных членов дают «бесплатный» дополнительный порядок — поэтому центральные формулы выгоднее односторонних той же ширины.

Цель практикума — увидеть две конкурирующие ошибки численной производной и подтвердить теорию измерением: порядок схем по наклону log-log графика, U-образную кривую полной ошибки и ускорение сходимости экстраполяцией Ричардсона. Справочник по инструментам — в памятке Python/NumPy/SciPy.

Практикум 1. Порядок схем и наклон log-log

Возьмём \(f(x)=\sin x\) в точке \(x_0=1\), где \(f'(x_0)=\cos 1\) известно точно. Сравним правую и центральную разности на «среднем» диапазоне шагов, где доминирует ошибка усечения.

schemes.py▶ python
import numpy as np

f      = np.sin
fprime = np.cos
x0     = 1.0
exact  = fprime(x0)

def forward(f, x, h):
    return (f(x + h) - f(x)) / h

def central(f, x, h):
    return (f(x + h) - f(x - h)) / (2 * h)

hs = np.logspace(-1, -6, 6)          # 1e-1 ... 1e-6 — зона усечения
print(f"{'h':>10} {'ошибка fwd':>14} {'ошибка cen':>14}")
for h in hs:
    ef = abs(forward(f, x0, h) - exact)
    ec = abs(central(f, x0, h) - exact)
    print(f"{h:10.1e} {ef:14.3e} {ec:14.3e}")

# Оценка порядка: наклон log(ошибка) по log(h)
ef = np.array([abs(forward(f, x0, h) - exact) for h in hs])
ec = np.array([abs(central(f, x0, h) - exact) for h in hs])
p_fwd = np.polyfit(np.log(hs), np.log(ef), 1)[0]
p_cen = np.polyfit(np.log(hs), np.log(ec), 1)[0]
print(f"\nнаклон (порядок) forward  p ≈ {p_fwd:.3f}")   # ≈ 1
print(f"наклон (порядок) central  p ≈ {p_cen:.3f}")     # ≈ 2
h ошибка fwd ошибка cen 1.0e-01 2.345e-02 9.001e-04 1.0e-02 2.323e-03 9.002e-06 1.0e-03 2.321e-04 9.002e-08 1.0e-04 2.321e-05 9.003e-10 1.0e-05 2.321e-06 8.987e-12 1.0e-06 2.321e-07 8.890e-12 наклон (порядок) forward p ≈ 1.000 наклон (порядок) central p ≈ 1.954

Наклон правой разности равен \(1\) (каждый шаг \(\times 10\) уменьшает ошибку в \(10\) раз), центральной — близок к \(2\) (ошибка падает в \(100\) раз). У центральной схемы при \(h=10^{-6}\) сходимость уже «ломается» — это вторгается округление, которым мы займёмся в практикуме 2.

Практикум 2. Полная ошибка и оптимальный шаг

Теперь пройдём шаги вплоть до \(10^{-16}\) и построим полную ошибку. Появится U-образная кривая: слева ошибка падает (усечение), справа — растёт (округление от вычитания близких значений \(f\)).

optimal_h.py▶ python
import numpy as np
import matplotlib.pyplot as plt

f, fprime, x0 = np.sin, np.cos, 1.0
exact = fprime(x0)
u = np.finfo(float).eps / 2          # единица округления ≈ 1.1e-16

hs = np.logspace(-1, -16, 200)
err_fwd = np.abs((f(x0 + hs) - f(x0)) / hs - exact)
err_cen = np.abs((f(x0 + hs) - f(x0 - hs)) / (2 * hs) - exact)

# Эмпирический оптимум
h_fwd = hs[np.argmin(err_fwd)]
h_cen = hs[np.argmin(err_cen)]
print(f"эмпирич. h* forward ≈ {h_fwd:.2e}  (теория ~ sqrt(u) = {np.sqrt(u):.2e})")
print(f"эмпирич. h* central ≈ {h_cen:.2e}  (теория ~ u**(1/3) = {u**(1/3):.2e})")
print(f"мин. ошибка forward ≈ {err_fwd.min():.2e}  (теория ~ sqrt(u))")
print(f"мин. ошибка central ≈ {err_cen.min():.2e}  (теория ~ u**(2/3))")

plt.loglog(hs, err_fwd, label="правая (p=1)")
plt.loglog(hs, err_cen, label="центральная (p=2)")
plt.gca().invert_xaxis()
plt.xlabel("h"); plt.ylabel("полная ошибка |D(h) - f'|")
plt.legend(); plt.grid(True, which="both", alpha=.3); plt.show()

Эмпирические оптимумы совпали с теоретическими по порядку величины. Вывод, важнейший для всего курса: чрезмерно малый \(h\) вреден. После \(h^*\) уменьшение шага не улучшает, а ухудшает результат — это та же потеря значимости из занятия 1, только теперь в числителе \(f(x+h)-f(x)\).

Практикум 3. Экстраполяция Ричардсона

Центральная разность имеет разложение ошибки только по чётным степеням: \(D(h)=f'(x)+c_2 h^2 + c_4 h^4 + \dots\). Это позволяет одним комбинированием двух приближений поднять порядок с \(2\) до \(4\).

richardson.py▶ python
import numpy as np

f, fprime, x0 = np.sin, np.cos, 1.0
exact = fprime(x0)

def central(h):
    return (f(x0 + h) - f(x0 - h)) / (2 * h)

def richardson(h):
    # D(h) = f' + c2 h^2 + O(h^4)  ⟹  (4 D(h/2) - D(h)) / 3 = f' + O(h^4)
    return (4 * central(h / 2) - central(h)) / 3

for h in [1e-1, 1e-2, 1e-3]:
    e2 = abs(central(h)    - exact)
    e4 = abs(richardson(h) - exact)
    print(f"h={h:.0e}:  central O(h^2)={e2:.3e}   Richardson O(h^4)={e4:.3e}")
h=1e-01: central O(h^2)=9.001e-04 Richardson O(h^4)=2.700e-07 h=1e-02: central O(h^2)=9.002e-06 Richardson O(h^4)=2.700e-11 h=1e-03: central O(h^2)=9.002e-08 Richardson O(h^4)=4.7e-14

При уменьшении \(h\) в \(10\) раз ошибка Ричардсона падает в \(\sim 10^4\) раз — это и есть четвёртый порядок. Метод почти даровой: всего одно дополнительное вычисление центральной разности.

Примените экстраполяцию повторно к результатам Ричардсона — какой порядок получится и до какого \(h\) она полезна, прежде чем вмешается округление?

Лекция B · 30 мин

Полная ошибка: усечение против округления

Практикум показал U-образную кривую. Объясним её аналитически. Полная ошибка численной производной складывается из двух слагаемых разной природы.

Ошибка усечения. Для схемы порядка \(p\) она равна \(C\,h^{p}\) с константой \(C\), зависящей от старшей производной \(f\). Она убывает с уменьшением \(h\).

Ошибка округления. Значения \(f\) вычислены с относительной точностью \(\sim u\): вместо \(f(x)\) машина даёт \(f(x)(1+\delta)\), \(|\delta|\le u\). В числителе разностной формулы стоит разность близких величин порядка \(|f|\), и абсолютная ошибка числителя \(\sim u\,|f|\). После деления на \(h^k\) (для первой производной \(k=1\)) ошибка округления результата ведёт себя как

\[ \varepsilon_{\text{окр}}(h)\ \sim\ \frac{u\,|f|}{h}. \tag{6.10} \]

Она растёт при уменьшении \(h\). Полная ошибка — сумма двух конкурирующих вкладов:

Модель полной ошибки. Для разностной схемы первой производной порядка \(p\) \[ E(h)\ \approx\ \underbrace{C\,h^{p}}_{\text{усечение}\,\downarrow} \;+\; \underbrace{\frac{u\,M}{h}}_{\text{округление}\,\uparrow}, \qquad M\sim|f(x)|. \] Первое слагаемое тянет \(h\) вниз, второе — вверх. Минимум суммы и даёт оптимальный шаг.

Оптимальный шаг и почему «слишком мало» — плохо

Найдём минимум \(E(h)=C h^{p}+ uM/h\), приравняв производную нулю:

\[ E'(h)=pC\,h^{p-1}-\frac{uM}{h^2}=0 \quad\Longrightarrow\quad h^{*}\ \sim\ \Big(\frac{uM}{pC}\Big)^{\!1/(p+1)}\ \sim\ u^{1/(p+1)}. \tag{6.11} \]

Подставив \(h^*\) обратно, получаем порядок минимально достижимой ошибки \(E(h^*)\sim u^{p/(p+1)}\). Конкретизируем:

Оптимальный шаг по схемам (double, \(u\approx1.1\times10^{-16}\)):
  1. Правая разность (\(p=1\)): \(h^*\sim \sqrt{u}\approx 10^{-8}\), минимальная ошибка \(\sim\sqrt{u}\approx 10^{-8}\). Половина значащих цифр потеряна!
  2. Центральная разность (\(p=2\)): \(h^*\sim u^{1/3}\approx 10^{-5}\), минимальная ошибка \(\sim u^{2/3}\approx 10^{-11}\). Заметно точнее и при большем шаге.
  3. Вторая производная (деление на \(h^2\)): округление \(\sim uM/h^2\) ещё агрессивнее, оптимум сдвигается к \(h^*\sim u^{1/4}\approx 10^{-4}\), достижимая точность \(\sim u^{1/2}\). Вторые производные численно «дорогие».
Ключевой вывод (связь с занятием 1). Уменьшать \(h\) ниже \(h^*\) не просто бесполезно — вредно: ошибка начинает расти. Виновата потеря значимости при вычитании \(f(x+h)-f(x)\): чем меньше \(h\), тем ближе вычитаемые значения и тем меньше верных цифр в разности. «Возьму \(h=10^{-15}\), чтобы было точнее» — классическая и дорогая ошибка новичка.

Экстраполяция Ричардсона

Можно ли получить высокий порядок, не уменьшая \(h\) до опасной зоны округления? Да — комбинируя приближения с разными шагами. Пусть метод имеет известное разложение ошибки

\[ D(h)=f'(x)+c_2\,h^2 + c_4\,h^4 + O(h^6) \tag{6.12} \]

(как у центральной разности — только чётные степени). Запишем то же для шага \(h/2\):

\[ D(h/2)=f'(x)+c_2\frac{h^2}{4}+c_4\frac{h^4}{16}+O(h^6). \tag{6.13} \]

Подберём линейную комбинацию, уничтожающую член \(c_2 h^2\). Умножим (6.13) на \(4\), вычтем (6.12) и разделим на \(3\):

Экстраполяция Ричардсона. Если \(D(h)=f'+c_2h^2+O(h^4)\), то \[ D_R(h)=\frac{4\,D(h/2)-D(h)}{3}=f'(x)+O(h^4). \] Ведущий член ошибки уничтожен — порядок поднят с \(2\) до \(4\). Общая формула для метода порядка \(p\) с шагом, делённым на \(t\): \[ D_R=\frac{t^{p}\,D(h/t)-D(h)}{t^{p}-1}. \] Повторяя приём по столбцам (\(O(h^4)\to O(h^6)\to\dots\)), получают таблицу Ромберга — её мы применим к интегралам в занятии 7.

Комплексный шаг — производная без вычитания

Источник всех бед — вычитание близких чисел в числителе. Существует элегантный обход для аналитических функций, которые можно вычислить от комплексного аргумента. Разложим \(f\) по мнимому приращению \(ih\):

\[ f(x+ih)=f(x)+ih\,f'(x)-\frac{h^2}{2}f''(x)-i\frac{h^3}{6}f'''(x)+\dots \tag{6.14} \]

Взяв мнимую часть и поделив на \(h\):

Метод комплексного шага. Для аналитической \(f\) \[ f'(x)\approx \frac{\operatorname{Im}\,f(x+ih)}{h},\qquad \text{ошибка } = O(h^2), \] причём без вычитания в числителе. Поэтому \(h\) можно брать сколь угодно малым (вплоть до \(10^{-200}\)) без потери значимости — ошибка округления не растёт, и достигается полная машинная точность \(\sim u\).
Ограничение. Метод требует, чтобы \(f\) была задана выражением, допускающим подстановку комплексного аргумента (как в Python через complex или в numpy). Для «чёрного ящика», доступного только в вещественных точках, он неприменим — там остаются разностные формулы с оптимальным \(h\) и экстраполяция Ричардсона.
Задача 6.4Комплексный шаг (доп.). Реализуйте fprime = lambda f,x,h: (f(x + 1j*h)).imag / h и постройте полную ошибку для \(f(x)=\sin x\) в \(x_0=1\) на шагах от \(10^{-1}\) до \(10^{-200}\). Убедитесь, что в отличие от центральной разности кривая не заворачивает вверх при малых \(h\), а выходит на плато \(\sim u\). Объясните, почему правую/центральную разность так нельзя.
Подсказка Используйте cmath или просто 1j в чистом Python; np.sin принимает комплексный аргумент. Числитель \(\operatorname{Im} f(x+ih)\) сам по себе \(\sim h f'\), деление на \(h\) даёт \(f'\) — нигде не вычитаются близкие большие числа, поэтому округление не усиливается. В обычных разностях вычитание \(f(x+h)-f(x)\) близких величин неизбежно.

Итоги занятия

Что нужно унести с занятия 6:
  1. Разностные формулы выводятся из ряда Тейлора. Правая/левая разность — \(O(h)\); центральная — \(O(h^2)\); трёхточечная для \(f''\) — \(O(h^2)\). Симметрия даёт лишний порядок.
  2. Метод неопределённых коэффициентов систематически строит схему нужного порядка на любом шаблоне узлов через линейную систему на коэффициенты.
  3. Полная ошибка \(E(h)\approx C h^p + uM/h\) — сумма убывающего усечения и растущего округления. Кривая U-образная.
  4. Оптимальный шаг \(h^*\sim u^{1/(p+1)}\): для правой \(\sim10^{-8}\), для центральной \(\sim10^{-5}\). Минимальная ошибка \(\sim u^{p/(p+1)}\). Уменьшать \(h\) ниже \(h^*\) — вредно (потеря значимости).
  5. Экстраполяция Ричардсона \(D_R=(4D(h/2)-D(h))/3\) поднимает порядок без опасно малого \(h\). Комплексный шаг \(\operatorname{Im}f(x+ih)/h\) даёт \(O(h^2)\) вовсе без округления — но лишь для аналитических \(f\).
Классная работа. Доделать задачи 6.1–6.3 (6.4 — по желанию). Сравните её оптимальный \(h^*\) с центральной трёхточечной.
