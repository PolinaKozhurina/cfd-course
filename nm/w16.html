Занятие 16 · Блок VII. Итоговый контроль

Зачётное занятие: итоговый проект и чек-лист

Цель занятия

Курс пройден: от машинной арифметики и анализа ошибок до решения систем, квадратур, итераций, разложений матриц и оптимизации. Зачётное занятие не вводит нового материала — его цель продемонстрировать владение методами курса на сквозном мини-проекте на Python и пройти чек-лист освоения, проверяющий ключевые умения и контрольные значения.

Зачёт состоит из двух частей. Первая — итоговый мини-проект: вы выбираете один из трёх вариантов, реализуете описанные методы своими руками (без обращения к готовым решателям там, где требуется собственная реализация), верифицируете результат на контрольных значениях и оформляете краткий отчёт с графиками. Вторая — сводный задачник: короткие задачи по каждой крупной теме курса для быстрой самопроверки. Чек-лист служит картой: незакрытые пункты подскажут, к какому занятию вернуться.

Что разрешено. Стандартная библиотека, NumPy, Matplotlib. numpy.linalg и scipy используйте только как эталон для сравнения (np.linalg.solve, np.linalg.cond, scipy.integrate.quad), а не вместо собственной реализации. Справочник по инструментам — в памятке Python/NumPy/SciPy.

1. Итоговый мини-проект

Выберите один из трёх вариантов. Каждый опирается на 3–4 занятия курса, имеет чёткую постановку, разбит на этапы и снабжён контрольными значениями для верификации. Объём — один скрипт на 150–300 строк плюс отчёт: код, 1–2 графика, таблица и краткий вывод.

ВАРИАНТ A

Собственный линейный решатель и анализ обусловленности

Соберите свой набор прямых методов решения СЛАУ и исследуйте, как обусловленность матрицы ограничивает достижимую точность. Темы: LU с частичным выбором (зан. 9), разложение Холецкого (зан. 10), число обусловленности \(\kappa(A)\) (зан. 11).

Этапы AЧто реализовать.
  1. LU с частичным выбором ведущего элемента. Функция lu_solve(A, b), возвращающая решение и факторы \(P, L, U\). Проверка тождества \(PA = LU\) по норме невязки.
  2. Разложение Холецкого \(A = LL^\top\) для симметричных положительно определённых матриц; решение через две треугольные подстановки.
  3. Число обусловленности \(\kappa_2(A) = \sigma_{\max}/\sigma_{\min}\) (через сингулярные числа) и/или \(\kappa_\infty(A) = \|A\|_\infty\|A^{-1}\|_\infty\). Сравнить со np.linalg.cond.
  4. Серия систем с известным точным решением \(x^\ast=(1,1,\dots,1)^\top\), \(b = Ax^\ast\): случайные хорошо обусловленные матрицы и матрицы Гильберта \(H_{ij}=1/(i+j-1)\) размеров \(n=2\dots12\).
  5. График потери точности. По оси \(x\) — \(\log_{10}\kappa(A)\), по оси \(y\) — число потерянных верных цифр \(-\log_{10}\big(\|\hat x-x^\ast\|_\infty/\|x^\ast\|_\infty\big)\). Нанести теоретическую прямую «потеря \(\approx\log_{10}\kappa\) цифр».
Эталонные контрольные значения
  • Матрица Гильберта \(H_5\): \(\kappa_2(H_5)\approx 4.8\times10^{5}\); \(H_8\): \(\kappa_2\approx 1.5\times10^{10}\); \(H_{10}\): \(\kappa_2\approx 1.6\times10^{13}\).
  • Правило большого пальца: при решении в double (\(u\approx10^{-16}\)) относительная ошибка \(\delta\lesssim \kappa(A)\cdot u\), то есть теряется примерно \(\log_{10}\kappa(A)\) верных цифр. Для \(H_5\): \(\log_{10}(4.8\!\times\!10^5)\approx 5.7\) цифр потеряно — из ~16 остаётся ~10.
  • Проверка факторизации: \(\|PA-LU\|_\infty/\|A\|_\infty \sim 10^{-16}\) (порядка машинной точности).
  • Холецкий должен падать (отрицательное подкоренное) для несимметричных или незнакоопределённых матриц — используйте это как тест положительной определённости.
variant_a_check.py
import numpy as np
from scipy.linalg import hilbert        # только как эталон

for n in (2, 5, 8, 10):
    H = hilbert(n)
    xstar = np.ones(n)
    b = H @ xstar
    xhat = lu_solve(H, b)               # ваша реализация
    rel = np.linalg.norm(xhat - xstar, np.inf) / np.linalg.norm(xstar, np.inf)
    kappa = np.linalg.cond(H, 2)
    lost = -np.log10(rel) if rel > 0 else 16
    print(f"n={n:2d}  cond={kappa:.2e}  rel.err={rel:.2e}  "
          f"верных цифр≈{lost:4.1f}  (предсказано≈{16-np.log10(kappa):4.1f})")
ВАРИАНТ B

Адаптивная квадратура

Постройте набор квадратур и адаптивную схему, которая сама сгущает разбиение там, где функция «трудная». Темы: составной метод Симпсона (зан. 7), квадратуры Гаусса–Лежандра (зан. 8), оценка ошибки и адаптивное деление.

Этапы BЧто реализовать.
  1. Составной Симпсон simpson(f, a, b, n) (\(n\) чётно) и проверка порядка \(O(h^4)\) на гладкой функции.
  2. Гаусс–Лежандр на \(2,3,4\) узла (узлы и веса — табличные либо через numpy.polynomial.legendre.leggauss) с отображением \([-1,1]\to[a,b]\).
  3. Адаптивное деление: на отрезке сравнить квадратуру с её разбиением пополам; если оценка локальной ошибки превышает допуск \(\varepsilon\,\frac{|h|}{b-a}\) — делить рекурсивно, иначе принять. Считать число вычислений \(f\).
  4. Тесты: гладкий интеграл и интеграл с пиком/особенностью. Сравнить число вычислений \(f\) у равномерного и адаптивного метода при одинаковой достигнутой точности.
Эталонные контрольные значения
  • Гладкий: \(\displaystyle\int_0^1 \frac{4}{1+x^2}\,dx = \pi = 3.14159265358979\). Составной Симпсон с \(n=8\) даёт уже ~6 верных цифр; при удвоении \(n\) ошибка падает в \(\approx16\) раз (наклон \(-4\) на log–log).
  • Особенность: \(\displaystyle\int_0^1 \frac{1}{\sqrt{x}}\,dx = 2\). У \(x=0\) подынтегральная функция неограничена — равномерный Симпсон сходится медленно (порядок падает), адаптивный сгущает узлы у нуля и достигает той же точности на порядок меньшим числом вычислений \(f\).
  • Гаусс–Лежандр на \(n\) узлов точен для многочленов степени \(\le 2n-1\); на \(\int_{-1}^1(3x^4-2x^2+1)dx=28/15\) двухточечный даёт \(4/3\) (неточно), трёхточечный — точно.
  • Контроль пика: \(\displaystyle\int_{-1}^1 \frac{1}{1+25x^2}\,dx = \tfrac{2}{5}\arctan 5 \approx 0.549360\) (функция Рунге) — адаптив сгущает у \(x=0\).

import numpy as np

f_smooth = lambda x: 4.0 / (1.0 + x*x)      # интеграл = pi на [0,1]
f_sing   = lambda x: 1.0 / np.sqrt(x)       # интеграл = 2  на [0,1], особ. в 0

I1, n1 = adaptive_quad(f_smooth, 0, 1, tol=1e-10)
I2, n2 = adaptive_quad(f_sing,   1e-14, 1, tol=1e-8)   # обход x=0
print(f"гладкий : I={I1:.12f}  err={abs(I1-np.pi):.2e}  вычислений f={n1}")
print(f"особенн.: I={I2:.12f}  err={abs(I2-2.0):.2e}  вычислений f={n2}")

ВАРИАНТ C

Интерполяция и сходимость

Сравните способы интерполяции и продемонстрируйте феномен Рунге и его подавление. Темы: интерполяция Ньютона по разделённым разностям (зан. 3), узлы Чебышёва (зан. 4), кубический сплайн.

Этапы CЧто реализовать.
  1. Полином Ньютона по разделённым разностям: divided_diff(x, y) и newton_eval(coef, x, t).
  2. Узлы Чебышёва \(x_k=\tfrac{a+b}{2}+\tfrac{b-a}{2}\cos\!\big(\tfrac{2k+1}{2n+2}\pi\big)\) и тот же интерполянт на них.
  3. Кубический сплайн (естественный): прогонка трёхдиагональной системы на вторые производные (можно взять scipy.interpolate.CubicSpline как эталон).
  4. Функция Рунге \(f(x)=\dfrac{1}{1+25x^2}\) на \([-1,1]\): для равномерных узлов, узлов Чебышёва и сплайна построить max-ошибку \(\max_x|f-p_n|\) на густой сетке как функцию числа узлов \(n\).
  5. График max-ошибки vs \(n\) (полулог по \(y\)): показать расходимость равномерной интерполяции (феномен Рунге) и сходимость Чебышёва/сплайна.
Эталонные контрольные значения
  • Равномерные узлы на функции Рунге: max-ошибка растёт с \(n\) (у краёв возникают всё более сильные осцилляции). При \(n=10\) ошибка \(\approx 1.9\), при \(n=20\) — порядка десятков; рост \(\sim 2^n\). Это и есть феномен Рунге.
  • Узлы Чебышёва: max-ошибка убывает экспоненциально; при \(n=20\) — порядка \(10^{-2}\), при \(n=40\) — \(\sim10^{-3}\) и ниже. Осцилляции у краёв подавлены.
  • Естественный кубический сплайн на равномерной сетке: ошибка убывает как \(O(h^4)\sim O(n^{-4})\), монотонно и устойчиво (Рунге не возникает).
  • Контроль: \(f(0)=1\), \(f(\pm1)=1/26\approx0.03846\). Любой интерполянт обязан проходить через узлы точно — проверьте \(p_n(x_k)=f(x_k)\) до машинной точности.
variant_c_check.py
import numpy as np

runge = lambda x: 1.0 / (1.0 + 25.0*x*x)
xx = np.linspace(-1, 1, 2001)            # густая сетка для оценки max-ошибки

for n in (10, 20, 40):
    xu = np.linspace(-1, 1, n+1)                         # равномерные
    k  = np.arange(n+1)
    xc = np.cos((2*k+1)*np.pi/(2*n+2))                   # Чебышёв на [-1,1]
    eu = np.max(np.abs(runge(xx) - newton_interp(xu, runge(xu), xx)))
    ec = np.max(np.abs(runge(xx) - newton_interp(xc, runge(xc), xx)))
    print(f"n={n:2d}  равномерн.max-err={eu:.3e}  Чебышёв max-err={ec:.3e}")
# ожидание: eu растёт (Рунге), ec убывает

def run_tests():
    """Сверка с эталоном; печать таблицы 'значение / эталон / ошибка'."""

2. Сводный задачник

Восемь коротких задач — по одной на крупную тему курса. Решайте устно или в пару строк кода; в каждой свёрнут ответ с числовым контрольным значением. Эти задачи — экспресс-проверка перед чек-листом.

Задача 1Округление и обусловленность (зан. 1, 11). (а) Чему равно машинное \(\varepsilon\) для double и единица округления \(u\)? (б) Матрица \(A=\begin{pmatrix}1 & 1\\ 1 & 1.0001\end{pmatrix}\): оцените порядок \(\kappa_\infty(A)\) и скажите, сколько верных цифр останется при решении \(Ax=b\) в double.
Ответ (а) \(\varepsilon_{\text{маш}}=2^{-52}\approx 2.22\times10^{-16}\); единица округления \(u=2^{-53}\approx1.11\times10^{-16}\). (б) \(A^{-1}=\frac{1}{0.0001}\begin{pmatrix}1.0001 & -1\\ -1 & 1\end{pmatrix}\), \(\|A\|_\infty\approx2.0001\), \(\|A^{-1}\|_\infty\approx 2.0001/10^{-4}\approx2\times10^{4}\), так что \(\kappa_\infty\approx 4\times10^{4}\). Потеря \(\approx\log_{10}(4\times10^4)\approx4.6\) цифры — останется ~11 верных из 16.
Задача 2Ряд Тейлора (зан. 2). Сколько членов ряда \(e^{x}=\sum x^k/k!\) нужно для вычисления \(e^{0.5}\) с абсолютной ошибкой \(<10^{-8}\)? Используйте остаток в форме Лагранжа \(R_n=e^{\xi}x^{n+1}/(n+1)!\), \(0\le\xi\le0.5\).
Ответ Оценка остатка \(R_n\le e^{0.5}\,0.5^{n+1}/(n+1)!\le 1.65\cdot0.5^{n+1}/(n+1)!\). Перебор: \(n=6\) даёт \(1.65\cdot0.5^{7}/5040\approx 2.6\times10^{-6}\) — мало; \(n=8\) даёт \(\approx 7\times10^{-9}<10^{-8}\). Достаточно \(n=8\) (члены до \(x^8/8!\)). Контроль: \(e^{0.5}=1.6487212707\).
Задача 3Интерполяция (зан. 3–4). По узлам \((0,1),(1,2),(2,5)\) постройте интерполяционный многочлен Ньютона и вычислите \(p(0.5)\). Почему узлы Чебышёва предпочтительнее равномерных при большом \(n\)?
Ответ Разделённые разности: \(f[x_0]=1\), \(f[x_0,x_1]=(2-1)/1=1\), \(f[x_1,x_2]=(5-2)/1=3\), \(f[x_0,x_1,x_2]=(3-1)/2=1\). Многочлен \(p(x)=1+1\cdot x+1\cdot x(x-1)=x^2+1\). Тогда \(p(0.5)=1.25\). Узлы Чебышёва минимизируют норму узлового многочлена \(\prod(x-x_k)\), подавляя осцилляции у краёв (феномен Рунге) и обеспечивая сходимость для гладких функций.
Задача 4МНК (зан. 5). Методом наименьших квадратов проведите прямую \(y=a+bx\) через точки \((0,1),(1,3),(2,4),(3,6)\). Найдите \(a,b\) из нормальных уравнений. Почему \(\kappa(A^\top A)=\kappa(A)^2\)?
Ответ \(\sum x=6,\ \sum x^2=14,\ \sum y=14,\ \sum xy=29,\ N=4\). Нормальная система \(\begin{pmatrix}4 & 6\\ 6 & 14\end{pmatrix}\!\binom{a}{b}=\binom{14}{29}\), определитель \(56-36=20\). \(b=(4\cdot29-6\cdot14)/20=(116-84)/20=1.6\), \(a=(14-6\cdot1.6)/4=(14-9.6)/4=1.1\). Прямая \(y=1.1+1.6x\). Сингулярные числа \(A^\top A\) суть квадраты сингулярных чисел \(A\), поэтому \(\kappa_2(A^\top A)=\sigma_{\max}^2/\sigma_{\min}^2=\kappa_2(A)^2\) — нормальные уравнения квадратично хуже обусловлены (отсюда предпочтение QR/SVD).
Задача 5Дифференцирование (зан. 6). Для \(f(x)=\sin x\) в точке \(x=1\) центральная разность \(f'\approx\frac{f(x+h)-f(x-h)}{2h}\). Каков порядок ошибки усечения по \(h\) и почему слишком малый \(h\) ухудшает результат?
Ответ Ошибка усечения центральной разности \(=-\frac{h^2}{6}f'''(\xi)=O(h^2)\). Но в числителе вычитаются близкие значения, и ошибка округления растёт как \(O(u/h)\). Суммарная ошибка \(\sim c_1 h^2 + c_2 u/h\) минимальна при \(h_{\text{opt}}\sim u^{1/3}\approx 6\times10^{-6}\), давая минимальную ошибку \(\sim u^{2/3}\approx10^{-11}\). Контроль: \(f'(1)=\cos1=0.5403023\).
Задача 6Интегрирование (зан. 7–8). Вычислите \(\int_0^1 x^2\,dx\) составным методом трапеций с \(n=2\) и Симпсона с \(n=2\). Сравните с точным \(1/3\). Каков порядок каждого метода?
Ответ Узлы \(0,\,0.5,\,1\), \(h=0.5\). Трапеции: \(\frac{h}{2}[f_0+2f_1+f_2]=\frac{0.5}{2}[0+2\cdot0.25+1]=0.25\cdot1.5=0.375\) (ошибка \(0.0417\), порядок \(O(h^2)\)). Симпсон: \(\frac{h}{3}[f_0+4f_1+f_2]=\frac{0.5}{3}[0+1+1]=0.3333\overline{3}\) — точно, так как Симпсон точен для многочленов до 3-й степени (порядок \(O(h^4)\)).
Задача 7СЛАУ и итерации (зан. 9–10, 12). Для системы с \(A=\begin{pmatrix}4 & 1\\ 1 & 3\end{pmatrix}\), \(b=\binom{1}{2}\) сделайте одну итерацию Якоби из \(x^{(0)}=0\). Сойдётся ли метод и почему?
Ответ Якоби: \(x_i^{(1)}=(b_i-\sum_{j\ne i}a_{ij}x_j^{(0)})/a_{ii}\). При \(x^{(0)}=0\): \(x_1^{(1)}=1/4=0.25\), \(x_2^{(1)}=2/3\approx0.667\). Метод сходится: матрица строго диагонально доминантна (\(|4|>|1|\), \(|3|>|1|\)) — достаточное условие сходимости Якоби и Зейделя. Точное решение \(x=(1/11,\,7/11)\approx(0.0909,\,0.6364)\).
Задача 8Собственные числа, SVD и оптимизация (зан. 13–15). (а) Степенной метод для \(A=\begin{pmatrix}2 & 1\\ 1 & 2\end{pmatrix}\): к какому \(\lambda\) сойдётся? (б) Минимум \(f(x)=x^2-4x+5\) — найдите его и один шаг градиентного спуска из \(x_0=0\) с шагом \(\alpha=0.1\).
Ответ (а) Собственные числа \(2\pm1=\{3,1\}\); степенной метод сходится к доминирующему \(\lambda_{\max}=3\) с вектором \((1,1)/\sqrt2\). (б) \(f'(x)=2x-4=0\Rightarrow x^\ast=2\), \(f(2)=1\). Шаг спуска: \(x_1=x_0-\alpha f'(x_0)=0-0.1\cdot(-4)=0.4\) — движемся к минимуму \(x=2\).

Интерактив: калькулятор κ(A) для 2×2 матрицы

Введите элементы матрицы \(A=\begin{pmatrix}a & b\\ c & d\end{pmatrix}\) — калькулятор вычислит сингулярные числа \(\sigma_{\max},\sigma_{\min}\), число обусловленности \(\kappa_2(A)=\sigma_{\max}/\sigma_{\min}\) и оценит, сколько верных десятичных цифр теряется при решении \(Ax=b\) в double. Это контрольное значение для самопроверки варианта A: задайте \(\begin{pmatrix}1 & 0.5\\ 0.5 & 1/3\end{pmatrix}\) (матрица Гильберта \(H_2\)) и сравните с \(\kappa(H_2)\approx 19.3\).
