Занятие 1 · Блок I. Основы вычислений

Машинная арифметика и анализ ошибок

Лекция A · ≤ 30 мин

Зачем нужны численные методы

Численные методы — это алгоритмы, которые дают приближённое решение математических задач за конечное число арифметических операций. Большинство задач анализа — вычисление интеграла, корня уравнения, решение системы — не имеют замкнутого ответа в элементарных функциях либо требуют бесконечного процесса (предела, ряда, итерации). Численный метод заменяет такой процесс конечной вычислительной схемой.

Платой за это служит погрешность. Грамотный вычислитель не стремится её обнулить — это невозможно, — а умеет оценить и контролировать её. Поэтому курс открывается не рецептами, а языком, на котором об ошибках говорят: абсолютная и относительная погрешность, обусловленность задачи и устойчивость алгоритма.

Определение. Пусть \(\tilde a\) — приближение точного значения \(a\). Абсолютная погрешность — это \(\Delta = |\tilde a - a|\), относительная погрешность — \(\delta = |\tilde a - a| / |a|\) (при \(a \neq 0\)). Относительная погрешность безразмерна и измеряется в «верных значащих цифрах»: \(\delta \approx 10^{-k}\) означает примерно \(k\) верных цифр.

Источники ошибок

В вычислительной цепочке погрешность накапливается из трёх независимых источников:

  • Ошибки в данных — неточность входных величин (измерения, ранее вычисленные значения). От алгоритма не зависят, но им усиливаются.
  • Ошибка усечения (метода) — следствие замены бесконечного процесса конечным: обрыв ряда Тейлора, замена производной разностью, конечное число итераций. Существует даже при идеальной арифметике.
  • Ошибка округления — следствие того, что компьютер хранит вещественные числа конечным набором битов. Возникает в каждой арифметической операции.

Первые два источника мы разберём в последующих занятиях (ряды, квадратуры, итерации). Сегодня — фундамент: как именно машина представляет числа и к каким сюрпризам это приводит.

Числа с плавающей точкой (IEEE 754)

Вещественное число хранится в нормализованном виде, аналогичном экспоненциальной записи, но в двоичной системе:

\[ x = \pm\, (1.b_1 b_2 \dots b_{p-1})_2 \times 2^{e}, \tag{1.1} \]

где мантисса (значащая часть) содержит \(p\) двоичных разрядов, а порядок \(e\) лежит в ограниченном диапазоне. Стандарт IEEE 754 определяет два основных формата:

Рис. 1. Разрядная структура числа двойной точности. Знак, смещённый порядок и мантисса. Старший бит мантиссы в нормализованном виде всегда равен 1 и не хранится («неявная единица»).

Множество представимых чисел конечно и распределено по оси неравномерно: внутри каждого двоичного интервала \([2^e, 2^{e+1})\) находится \(2^{p-1}\) равноотстоящих чисел, а при переходе к следующему интервалу шаг удваивается. Иначе говоря, плотность представимых чисел постоянна в относительном, а не в абсолютном смысле.

Рис. 2. Представимые числа разрежаются по мере удаления от нуля. Относительный зазор между соседними числами постоянен — это и есть машинное ε.
Модель округления. Если \(x\) лежит в нормализованном диапазоне, то результат его записи в формат с плавающей точкой удовлетворяет \[ \mathrm{fl}(x) = x\,(1 + \varepsilon),\qquad |\varepsilon| \le u, \] а для каждой арифметической операции \(\circ \in \{+,-,\times,/\}\) выполняется \(\mathrm{fl}(a \circ b) = (a \circ b)(1 + \varepsilon)\), \(|\varepsilon|\le u\). Величина \(u = 2^{-p}\) называется единицей округления. Для double \(u = 2^{-53} \approx 1.11\times 10^{-16}\).

Машинное эпсилон

Машинное эпсилон \(\varepsilon_{\text{маш}}\) — это расстояние от \(1\) до следующего представимого числа: \(\varepsilon_{\text{маш}} = 2^{-(p-1)} = 2^{-52} \approx 2.22\times 10^{-16}\) для double. Это наибольшее число, которое «теряется» при сложении с единицей: при \(\delta < \varepsilon_{\text{маш}}/2\) результат \(\mathrm{fl}(1+\delta)\) снова равен \(1\). Эпсилон задаёт относительную точность арифметики и фигурирует во всех оценках ошибок округления.

Не путать. Единица округления \(u = 2^{-53}\) (граница относительной ошибки одной операции) и машинное эпсилон \(\varepsilon_{\text{маш}} = 2^{-52}\) (зазор у единицы) отличаются ровно вдвое. В литературе обоим иногда говорят «machine epsilon»; важно понимать, какая именно величина имеется в виду.

Потеря значимости (катастрофическое вычитание)

Самый опасный эффект конечной арифметики — катастрофическая потеря значимости при вычитании двух близких чисел. Пусть \(a\) и \(b\) известны с относительной точностью \(\sim u\), но \(a \approx b\). Их разность мала, а абсолютная ошибка остаётся прежней — значит, относительная ошибка разности резко возрастает: верные старшие цифры сокращаются, и обнажаются «мусорные» младшие.

\[ \text{если } a = 1.23456789,\ b = 1.23456700,\quad a-b = 0.00000089 \]

— из девяти значащих цифр осталось две. Числа округления при этом не появилось новой ошибки: вычитание точно. Беда в том, что относительная ошибка уже присутствовавшая в \(a,b\) после сокращения старших разрядов стала огромной. Лекарство — переписать формулу так, чтобы избежать вычитания близких величин. Это мы проверим на практике прямо сейчас.

Практикум 1. Эпсилон, округление и порядок суммирования

machine_eps.py▶ python
import numpy as np

# 1. Машинное эпсилон «вручную»: наименьшее eps, при котором 1 + eps != 1
eps = 1.0
while 1.0 + eps / 2.0 != 1.0:
    eps /= 2.0
print("eps вручную      :", eps)
print("numpy float64 eps:", np.finfo(np.float64).eps)
print("единица округл. u:", np.finfo(np.float64).eps / 2)

# 2. Классика: 0.1 + 0.2 не равно 0.3
print(0.1 + 0.2)                 # 0.30000000000000004
print(0.1 + 0.2 == 0.3)          # False
print(abs((0.1 + 0.2) - 0.3))    # ~5.5e-17 — порядка u

# 3. Сложение единицы и малого числа
print(1.0 + 1e-16 == 1.0)        # True  — 1e-16 потерялось
print(1.0 + 1e-15 == 1.0)        # False

Просуммируйте \(N = 10^7\) одинаковых слагаемых \(0.1\) двумя способами: (а) наивным циклом слева направо в обычном float; (б) функцией math.fsum (точное суммирование) либо суммированием по Кэхэну. Сравните с точным значением \(10^6\). Объясните, откуда берётся расхождение в наивной сумме.
Подсказка и эталон
kahan.py
import math

N = 10**7
naive = 0.0
for _ in range(N):
    naive += 0.1
exact = N * 0.1                      # = 1_000_000.0

print("наивная сумма :", naive)
print("math.fsum     :", math.fsum([0.1] * N))
print("ошибка наивной:", abs(naive - exact))   # порядка 1e-3
Каждое сложение вносит относительную ошибку \(\le u\); при \(N\) операциях ошибки накапливаются и в худшем случае растут как \(N u\). Точное суммирование (fsum, Кэхэн) удерживает «потерянные» младшие разряды и даёт почти точный результат.

Практикум 2. Катастрофическое вычитание и его лечение

Рассмотрим функцию \(f(x) = \dfrac{1 - \cos x}{x^2}\). Её точный предел при \(x \to 0\) равен \(\tfrac12\). Но прямое вычисление при малых \(x\) катастрофически теряет точность: \(\cos x \approx 1\), и в числителе вычитаются близкие числа.

cancellation.py▶ python
import numpy as np

def f_naive(x):
    return (1.0 - np.cos(x)) / x**2          # теряет значимость при малых x

def f_stable(x):
    # 1 - cos x = 2 sin^2(x/2)  ⟹  деления близких чисел нет
    return 0.5 * (np.sin(x / 2) / (x / 2))**2

for k in range(1, 9):
    x = 10.0**(-k)
    print(f"x=1e-{k}:  наивно={f_naive(x):.16f}   устойчиво={f_stable(x):.16f}")
x=1e-1: наивно=0.4995834721974... устойчиво=0.4995834721974... x=1e-4: наивно=0.4999999969612... устойчиво=0.4999999995833... x=1e-7: наивно=0.0000000000000... устойчиво=0.4999999999999... x=1e-8: наивно=0.0000000000000... устойчиво=0.5000000000000...

При \(x = 10^{-8}\) имеем \(1 - \cos x \approx 5\times10^{-17}\) — это меньше зазора \(\varepsilon_{\text{маш}}\) у единицы, поэтому fl(1 - cos x) = 0 и наивная формула даёт ноль вместо \(\tfrac12\). Устойчивый вариант через \(\sin\) считает то же выражение без вычитания близких чисел и сохраняет все цифры.

Задача 1.2Квадратное уравнение. Корни \(x^2 + bx + c = 0\) по школьной формуле \(x = \dfrac{-b \pm \sqrt{b^2 - 4c}}{2}\) при \(b = 10^{8},\ c = 1\) дают для «малого» корня катастрофическую ошибку. Вычислите оба корня наивно и через устойчивую формулу Виета \(x_1 x_2 = c\). Сравните с точным \(x_2 \approx -10^{-8}\).
Подсказка Большой по модулю корень \(x_1 = \dfrac{-b - \operatorname{sign}(b)\sqrt{b^2-4c}}{2}\) считается без вычитания близких чисел; малый получаем из \(x_2 = c / x_1\). Наивная формула (-b + sqrt(b**2 - 4*c)) / 2 вычитает \(b\) из почти равного ему корня и теряет почти все цифры.

Предложите устойчивый вариант (подсказка: в NumPy есть np.expm1). На каком \(x\) ошибка наивной формулы превышает \(1\%\)?

Обусловленность задачи

Практикум показал: одни вычисления устойчивы, другие — нет. Чтобы разделить вину между задачей и алгоритмом, вводят два независимых понятия. Первое относится к самой задаче, безотносительно способа её решения.

Определение. Число обусловленности задачи \(y = f(x)\) — это коэффициент усиления относительной погрешности входа в относительную погрешность выхода: \[ \kappa = \lim_{\delta \to 0}\,\sup_{|\Delta x| \le \delta|x|}\frac{|f(x+\Delta x)-f(x)|/|f(x)|}{|\Delta x|/|x|}. \] Для гладкой скалярной функции это даёт \(\displaystyle \kappa(x) = \left|\frac{x\,f'(x)}{f(x)}\right|\). Задача хорошо обусловлена, если \(\kappa\) умеренно (порядка единиц), и плохо обусловлена, если \(\kappa \gg 1\).

Число обусловленности — свойство задачи и точки, а не программы. Никакой алгоритм не вычислит плохо обусловленную задачу точнее, чем позволяет \(\kappa\): если вход известен с относительной ошибкой \(u\), то выход в принципе нельзя получить точнее, чем \(\kappa\cdot u\). Примеры:

  • \(f(x) = \sqrt{x}\): \(\kappa = \tfrac12\) — всюду прекрасно обусловлена.
  • \(f(x) = e^x\): \(\kappa = |x|\) — теряет точность при больших \(|x|\).
  • \(f(x) = \ln x\): \(\kappa = 1/|\ln x|\) — взрывается у \(x = 1\) (там \(\ln x \to 0\)).
  • Вычитание \(f(a,b) = a - b\) при \(a \approx b\): \(\kappa\) велико — это и есть теоретическая причина катастрофического вычитания из практикума.

Устойчивость алгоритма

Второе понятие относится к конкретному способу вычисления. Один и тот же \(f\) можно считать по наивной формуле или по устойчивой — задача одна, алгоритмы разные.

Определение. Алгоритм \(\tilde f\), вычисляющий \(f\), называется обратно устойчивым (backward stable), если его результат есть точное решение задачи для слегка возмущённого входа: \[ \tilde f(x) = f(\tilde x)\quad\text{для некоторого } \tilde x \text{ с } \frac{|\tilde x - x|}{|x|} = O(u). \] То есть «мы решили почти ту задачу, что хотели». Обратная устойчивость — золотой стандарт численных алгоритмов (метод Гаусса с выбором главного элемента, QR-разложение и др. обратно устойчивы).

Прямая и обратная ошибка. Главное правило

Различают две ошибки результата \(\tilde y = \tilde f(x)\):

  • Прямая ошибка — \(|\tilde y - y|/|y|\): насколько ответ отличается от истинного. Это то, что нас в конечном счёте интересует.
  • Обратная ошибка — наименьшее возмущение входа \(|\tilde x - x|/|x|\), при котором \(\tilde y = f(\tilde x)\): «для какой задачи наш ответ точен».
Рис. 3. Прямая и обратная ошибка. Обратно устойчивый алгоритм даёт малую обратную ошибку \(O(u)\); прямая ошибка тогда ограничена числом обусловленности задачи.
Главное правило вычислительной математики. \[ \boxed{\ \text{прямая ошибка} \;\lesssim\; \kappa \cdot \text{обратная ошибка}\ } \] Если алгоритм обратно устойчив (обратная ошибка \(O(u)\)), то прямая ошибка ограничена \(\kappa\cdot u\). Плохой результат означает либо плохую обусловленность задачи (\(\kappa\) велико — ничего не поделать, нужна другая постановка), либо неустойчивый алгоритм (большая обратная ошибка — нужно переписать вычисление).

Интерактив: число обусловленности функции

Слева — график \(\kappa(x) = |x f'(x)/f(x)|\) для выбранной функции; красная зона \(\kappa > 10\) отмечает плохую обусловленность. Сдвигая точку \(x\), наблюдайте справа, как фиксированное относительное возмущение входа \(\delta_{\text{вх}} = 10^{-6}\) усиливается в относительную ошибку выхода \(\delta_{\text{вых}} \approx \kappa\,\delta_{\text{вх}}\).

Итоги занятия

Что нужно унести с занятия 1:
  1. Числа в компьютере представлены конечным набором битов (IEEE 754); множество представимых чисел конечно и разрежается вдали от нуля. Относительная точность задаётся машинным \(\varepsilon\approx2.2\times10^{-16}\) (double).
  2. Каждая операция вносит относительную ошибку \(\le u\). Опасность — не отдельная операция, а накопление и особенно вычитание близких чисел (потеря значимости).
  3. Обусловленность \(\kappa\) — свойство задачи: во сколько раз усиливается ошибка входа. \(\kappa=|xf'/f|\) для скалярной функции.
  4. Устойчивость — свойство алгоритма: обратно устойчивый даёт точный ответ для слегка возмущённого входа.
  5. Главное правило: прямая ошибка ≲ κ × обратная ошибка. Диагностируйте: велик \(\kappa\) (меняйте постановку) или велика обратная ошибка (переписывайте формулу).
