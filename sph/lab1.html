Реализация ядра и SPH-аппроксимация функции

Цель: реализовать кубический B-сплайн Монагана и его производные, проверить условия нормировки аналитически, убедиться что SPH-сумма корректно приближает гладкую функцию и её производную.

Среда: Maple в режиме Document Mode. Все графики и вычисления выполняются внутри Maple-файла.

К занятию освежите: формулу кубического B-сплайна (1.7), производные (1.9)–(1.10), основную SPH-формулу (1.4).

Задание 1.1. Ядро и производные

ЗАДАНИЕ 1.1

Реализация кубического B-сплайна

Написать в Maple две процедуры:

  • W_kernel(q) — значение ядра (1.7),
  • dW_kernel(q) — значение производной (1.9).

Построить на одном рисунке три графика: \(W(q),\ W'(q),\ W''(q)\) для \(q \in [0, 3]\). При построении положить \(\sigma = \sigma_{1D} = 2/3\), \(h = 1\) — то есть строить безразмерную функцию \(W(q)\cdot h\).

Контрольные значения для проверки:

  • \(W(0) = \sigma_{1D} = 2/3 \approx 0.667\) (максимум в центре),
  • \(W(1) = \sigma_{1D}\cdot 0.25 = 1/6 \approx 0.167\) (склейка полиномов),
  • \(W(2) = 0\) (граница носителя),
  • \(W'(0) = 0\) (экстремум в нуле),
  • \(W'(1)\) непрерывна: \(W'(1-) = W'(1+) = -1/2\).
# --- Задание 1.1: Кубический B-сплайн Монагана ---
restart;

# Константа нормировки для 1D (см. лекцию, §Нормировка)
sigma1D := 2/3;

W_kernel := proc(q)
  local qq;
  qq := abs(q);  # ядро чётное
  if qq >= 2 then return 0
  elif qq >= 1 then return sigma1D * (2 - qq)^3 / 4
  else return sigma1D * (1 - 3*qq^2/2 + 3*qq^3/4)
  end if;
end proc:

dW_kernel := proc(q)
  local qq, sgn;
  qq := abs(q);
  sgn := sign(q);
  if qq >= 2 then return 0
  elif qq >= 1 then return -sigma1D * 3*(2-qq)^2/4 * sgn
  else return sigma1D * (-3*qq + 9*qq^2/4) * sgn
  end if;
end proc:

# Вторая производная (для построения)
d2W_kernel := proc(q)
  local qq;
  qq := abs(q);
  if qq >= 2 then return 0
  elif qq >= 1 then return sigma1D * 3*(2-qq)/2
  else return sigma1D * (-3 + 9*qq/2)
  end if;
end proc:

# Проверка контрольных значений
printf("W(0) = %a,  ожидается 2/3 = %a\n", W_kernel(0), evalf(2/3));
printf("W(1) = %a,  ожидается 1/6 = %a\n", W_kernel(1), evalf(1/6));
printf("W(2) = %a,  ожидается 0\n", W_kernel(2));
printf("W'(0) = %a, ожидается 0\n", dW_kernel(0));
printf("W'(1-) = %a = W'(1+) = %a\n", dW_kernel(0.999), dW_kernel(1.001));

# Графики
with(plots):
p1 := plot(W_kernel(q), q=0..3, color=blue, legend="W(q)", thickness=2);
p2 := plot(dW_kernel(q), q=0..3, color=red, legend="W'(q)", thickness=2);
p3 := plot(d2W_kernel(q), q=0..3, color="DarkGreen", legend="W''(q)", thickness=2);
display(p1, p2, p3, title="Кубический B-сплайн и производные",
        labels=["q", "значение"]);
Подсказка. В Maple функции, определённые через if, нельзя напрямую передать в plot как выражение — нужно использовать стрелку или создать процедуру с помощью proc, что и сделано выше. Альтернативно можно записать ядро через piecewise: 
W_sym := (q) -> piecewise(
  abs(q) < 1, sigma1D*(1 - 3*q^2/2 + 3*abs(q)^3/4),
  abs(q) < 2, sigma1D*(2 - abs(q))^3/4,
  0);
Такую форму удобнее для символьных вычислений, но в численных циклах быстрее работает proc с if.

Задание 1.2. Проверка нормировки

ЗАДАНИЕ 1.2

Аналитическая проверка нормировки в 1D и 2D

С помощью команды Maple int() вычислить аналитически следующие интегралы:

1D:

\[ I_{1D} = 2\int_0^2 W(q)\,dq = 2\int_0^1 \Bigl(1 - \tfrac{3}{2}q^2 + \tfrac{3}{4}q^3\Bigr)dq + \int_1^2 \tfrac{1}{4}(2-q)^3\,dq \]

Подставить \(\sigma_{1D} = 2/3\) и убедиться, что \(\sigma_{1D}\cdot I_{1D} = 1\).

2D (радиальная часть \(r\,dr\) вместо просто \(dr\)):

\[ I_{2D} = 2\pi\int_0^2 W(q)\,q\,dq \]

Проверить, что \(\sigma_{2D}\cdot I_{2D} = 1\) при \(\sigma_{2D} = 10/(7\pi)\).

# --- Задание 1.2: Проверка нормировки ---

# Символьная форма ядра (без σ) — коэффициент добавим в конце
Wbar := (q) -> piecewise(q < 1, 1 - 3*q^2/2 + 3*q^3/4,
                         q < 2, (2-q)^3/4, 0);

# 1D
I1D := 2*int(Wbar(q), q=0..2);
printf("I_1D = %a\n", I1D);  # ожидается 3/2

sigma1D := 2/3;
printf("σ₁D · I₁D = %a\n", sigma1D * I1D);  # должно быть 1

# 2D
I2D := 2*Pi*int(Wbar(q) * q, q=0..2);
printf("I_2D = %a\n", I2D);

sigma2D := 10/(7*Pi);
printf("σ₂D · I₂D = %a\n", simplify(sigma2D * I2D));  # должно быть 1

# 3D (дополнительно)
I3D := 4*Pi*int(Wbar(q) * q^2, q=0..2);
sigma3D := 1/Pi;
printf("σ₃D · I₃D = %a\n", simplify(sigma3D * I3D));  # должно быть 1
Что должно получиться. I1D = 3/2, 2/3 · 3/2 = 1. I2D = 7π/10, 10/(7π) · 7π/10 = 1. Если Maple вернул другое — проверьте пределы интегрирования и коэффициенты в полиномах.

Задание 1.3. SPH-аппроксимация функции и её производной

ЗАДАНИЕ 1.3

Написать SPH-аппроксимацию sin(x) и cos(x)

Написать код, создающий \(N = 200\) равноотстоящих частиц на \([0, 2\pi]\):

  • координаты \(x_j = 2\pi(j-1)/N\), \(\Delta x = 2\pi/N\),
  • массы \(m_j = \rho_0 \Delta x\) при \(\rho_0 = 1\),
  • значения функции \(f_j = \sin(x_j)\).

В цикле по частицам \(i\) вычислить:

  • SPH-аппроксимацию функции по (1.4): \(f_i^{\text{SPH}} = \sum_j \frac{m_j}{\rho_j}f_j W_{ij}\),
  • SPH-аппроксимацию производной по (1.6): \((f')_i^{\text{SPH}} = \sum_j \frac{m_j}{\rho_j}f_j \,\partial_i W_{ij}\).

Построить два графика: (а) \(f_i^{\text{SPH}}\) и \(\sin(x)\); (б) \((f')_i^{\text{SPH}}\) и \(\cos(x)\). Использовать \(h = \Delta x\).

# --- Задание 1.3: SPH-аппроксимация sin и cos ---

# Параметры
N := 200;
Lx := 2*Pi;
dx := Lx/N;
rho0 := 1;
h := dx;  # длина сглаживания = шаг сетки

# Инициализация массивов (используем Array для скорости)
x := Array(1..N):
f := Array(1..N):
m := Array(1..N):
for j from 1 to N do
  x[j] := (j-1) * dx + dx/2;  # центры ячеек
  f[j] := sin(x[j]);
  m[j] := rho0 * dx;
end do:

# Плотность тут постоянна, но для общности посчитаем её суммированием
# (для лаб 1 можно положить ρ = 1 без пересчёта)
rho := Array(1..N):
for i from 1 to N do
  s := 0;
  for j from 1 to N do
    qij := abs(x[i]-x[j])/h;
    s := s + m[j] * W_kernel(qij)/h;
  end do;
  rho[i] := s;
end do:

# SPH-аппроксимация функции и производной
fSph := Array(1..N):
dfSph := Array(1..N):
for i from 1 to N do
  sumF := 0; sumDf := 0;
  for j from 1 to N do
    dxij := x[i] - x[j];
    qij := abs(dxij)/h;
    if qij < 2 then
      Wij := W_kernel(qij)/h;
      dWij := dW_kernel(qij) * sign(dxij) / h^2;
      # Примечание: 1/h от цепного правила умножается на 1/h² из-за σ/h
      sumF  := sumF  + m[j]/rho[j] * f[j] * Wij;
      sumDf := sumDf + m[j]/rho[j] * f[j] * dWij;
    end if;
  end do;
  fSph[i] := sumF;
  dfSph[i] := sumDf;
end do:

# Визуализация
with(plots):
p_f := plot([seq([x[i], fSph[i]], i=1..N)], style=point, symbol=circle,
            symbolsize=8, color=red, legend="SPH"):
p_fex := plot(sin(x), x=0..Lx, color=black, thickness=2, legend="sin(x)"):
display(p_f, p_fex, title="SPH-аппроксимация функции", labels=["x","f"]);

p_df := plot([seq([x[i], dfSph[i]], i=1..N)], style=point, symbol=circle,
             symbolsize=8, color=red, legend="SPH"):
p_dfex := plot(cos(x), x=0..Lx, color=black, thickness=2, legend="cos(x)"):
display(p_df, p_dfex, title="SPH-аппроксимация производной", labels=["x","f'"]);
Замечание о нормировке \(1/h\). В определении ядра (1.7) константа \(\sigma/h^d\) уже включает множитель \(1/h^d\). В моей реализации W_kernel возвращает sigma * polynomial(q) без \(1/h\) — поэтому в SPH-сумме я делю на \(h\) отдельно. Проверьте себя: в 1D итоговая формула \(\rho_i = \sum_j m_j\,W(q_{ij})/h\), единица измерения — [масса/длина].
Периодические граничные условия. Частицы на краях \([0, 2\pi]\) имеют соседей с противоположного края. Для аккуратного моделирования периодичности нужно искать соседей не только в основной области, но и через «image»-частицы. В этом задании для упрощения оставим свободные границы — возле краёв \(x=0\) и \(x=2\pi\) точность будет ниже. В задании 1.4 посмотрим, как это проявляется.

Задание 1.4. Влияние длины сглаживания h

ЗАДАНИЕ 1.4

Параметрический анализ по h

Запустить код задания 1.3 три раза при разных значениях длины сглаживания: \(h = 0.5\,\Delta x\), \(h = \Delta x\), \(h = 2\,\Delta x\). Для каждого случая вычислить \(L_2\)-ошибку:

\[ \varepsilon_{L_2} = \sqrt{\frac{1}{N}\sum_{i=1}^N \bigl(f_i^{\text{SPH}} - \sin(x_i)\bigr)^2}. \]

Построить все три аппроксимации на одном графике. Объяснить: почему при \(h < \Delta x\) ядро «не видит» соседей (SPH даёт \(f_i \approx 0\)), а при \(h \gg \Delta x\) теряется локальность (сглаженная функция заметно отклоняется от истинной).

# --- Задание 1.4: Анализ влияния h ---

compute_sph := proc(h_val)
  local rho, fSph, i, j, qij, dxij, sumF, Wij, s, dWij, err;
  rho := Array(1..N);
  fSph := Array(1..N);
  for i from 1 to N do
    s := 0;
    for j from 1 to N do
      qij := abs(x[i]-x[j])/h_val;
      s := s + m[j] * W_kernel(qij)/h_val;
    end do;
    rho[i] := max(s, 1e-12);
  end do;
  for i from 1 to N do
    sumF := 0;
    for j from 1 to N do
      qij := abs(x[i]-x[j])/h_val;
      if qij < 2 then
        Wij := W_kernel(qij)/h_val;
        sumF := sumF + m[j]/rho[j] * f[j] * Wij;
      end if;
    end do;
    fSph[i] := sumF;
  end do;
  err := sqrt(add((fSph[i] - sin(x[i]))^2, i=1..N) / N);
  return fSph, err;
end proc:

f_h05, err_h05 := compute_sph(0.5*dx):
f_h10, err_h10 := compute_sph(1.0*dx):
f_h20, err_h20 := compute_sph(2.0*dx):

printf("h = 0.5·Δx: L₂ = %.4e\n", err_h05);
printf("h = 1.0·Δx: L₂ = %.4e\n", err_h10);
printf("h = 2.0·Δx: L₂ = %.4e\n", err_h20);

with(plots):
P_ex := plot(sin(x), x=0..Lx, color=black, thickness=2, legend="sin(x)"):
P_h05 := plot([seq([x[i], f_h05[i]], i=1..N)], style=point, symbol=box,
              symbolsize=6, color="DarkOrange", legend="h=0.5Δx"):
P_h10 := plot([seq([x[i], f_h10[i]], i=1..N)], style=point, symbol=circle,
              symbolsize=6, color=red, legend="h=Δx"):
P_h20 := plot([seq([x[i], f_h20[i]], i=1..N)], style=point, symbol=cross,
              symbolsize=6, color=blue, legend="h=2Δx"):
display(P_ex, P_h05, P_h10, P_h20,
        title="Влияние длины сглаживания h",
        labels=["x","f_SPH"]);
Ожидаемое наблюдение.
  • h = 0.5Δx — ядро «не достаёт» до соседей (носитель ядра \(2h = \Delta x\) покрывает только саму частицу и её ближайшего соседа). Плотность получается недооценённой \(\rho_i < 1\), SPH-аппроксимация функции не работает. Видно «рваное» точечное распределение.
  • h = Δx — компромисс. Ядро покрывает 3-5 соседей, SPH-аппроксимация плотности и функции работает хорошо.
  • h = 2Δx — сильное сглаживание. Каждая частица видит \(\sim 9\) соседей, аппроксимация очень стабильная, но амплитуда синуса немного занижена (видно на пиках \(\pm 1\)) — это проявление \(O(h^2)\)-ошибки из теоремы 1.
