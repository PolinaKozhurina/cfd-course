Занятие 11 · Блок IV. Линейная алгебра: решение СЛАУ

Нормы, число обусловленности и анализ ошибок

Лекция A · ≤ 30 мин

На прошлых занятиях мы научились решать систему \(Ax = b\) — методом Гаусса, через \(LU\)- и \(LL^\top\)-разложения, прогонкой. Теперь зададим неизбежный вопрос: насколько можно доверять полученному решению? Входные данные \(A\) и \(b\) приходят из измерений или предыдущих вычислений и уже содержат ошибку \(\sim u\) (машинное эпсилон, занятие 1). Сегодня мы построим аппарат, который связывает ошибку входа с ошибкой решения. Этот аппарат — нормы и число обусловленности — позволяет заранее оценить точность ещё до запуска счёта.

Векторные нормы

Чтобы говорить о «величине» ошибки вектора, нужна мера длины. Норма обобщает понятие длины на произвольное векторное пространство.

Определение. Нормой на \(\mathbb{R}^n\) называется функция \(\|\cdot\|:\mathbb{R}^n\to\mathbb{R}\), удовлетворяющая трём аксиомам для любых \(x,y\in\mathbb{R}^n\) и \(\alpha\in\mathbb{R}\):
  1. положительность: \(\|x\|\ge 0\), причём \(\|x\| = 0 \iff x = 0\);
  2. однородность: \(\|\alpha x\| = |\alpha|\,\|x\|\);
  3. неравенство треугольника: \(\|x + y\| \le \|x\| + \|y\|\).

На практике используют три нормы из семейства \(p\)-норм \(\|x\|_p = \left(\sum_i |x_i|^p\right)^{1/p}\):

\[ \|x\|_1 = \sum_{i=1}^n |x_i|,\qquad \|x\|_2 = \sqrt{\sum_{i=1}^n x_i^2},\qquad \|x\|_\infty = \max_{1\le i\le n} |x_i|. \tag{11.1} \]

Это, соответственно, манхэттенская норма (сумма модулей), привычная евклидова длина и чебышёвская (максимум-) норма. Геометрически «единичный шар» \(\{x : \|x\|\le 1\}\) для них — ромб, круг и квадрат соответственно.

Рис. 1. Единичные шары трёх стандартных норм на плоскости: ромб (\(\ell_1\)), круг (\(\ell_2\)) и квадрат (\(\ell_\infty\)). Видно вложение шаров — основа эквивалентности норм.

Эквивалентность норм в конечномерном пространстве

Теорема (эквивалентность норм). В конечномерном пространстве \(\mathbb{R}^n\) любые две нормы \(\|\cdot\|_a\) и \(\|\cdot\|_b\) эквивалентны: существуют константы \(0 < c \le C < \infty\), такие что для всех \(x\) \[ c\,\|x\|_b \;\le\; \|x\|_a \;\le\; C\,\|x\|_b. \]

Практический смысл: если ошибка мала в одной норме, она мала и в любой другой — выбор нормы не меняет качественного вывода о сходимости или точности. Для трёх стандартных норм константы явные:

\[ \|x\|_\infty \le \|x\|_2 \le \|x\|_1 \le \sqrt{n}\,\|x\|_2 \le n\,\|x\|_\infty. \tag{11.2} \]

Например, \(\|x\|_2 \le \sqrt{n}\,\|x\|_\infty\): евклидова длина не превосходит максимальной координаты, умноженной на \(\sqrt n\). При этом константы растут с размерностью \(n\) — в очень больших задачах разница между нормами может быть существенной численно, хотя качественно эквивалентность сохраняется.

Матричные (индуцированные) нормы

Чтобы оценивать, как матрица \(A\) усиливает вектор, вводят матричную норму, согласованную с векторной. Самый естественный способ — измерить максимальный коэффициент растяжения.

Определение. Операторная (индуцированная) норма матрицы \(A\in\mathbb{R}^{m\times n}\), порождённая векторной нормой \(\|\cdot\|\), есть \[ \|A\| = \max_{x\neq 0}\frac{\|Ax\|}{\|x\|} = \max_{\|x\|=1}\|Ax\|. \] Это наибольшее, во сколько раз \(A\) удлиняет вектор.

Для трёх стандартных норм индуцированные нормы вычисляются явно:

\[ \|A\|_1 = \max_{1\le j\le n}\sum_{i=1}^m |a_{ij}| \quad(\text{макс. сумма по столбцу}),\qquad \|A\|_\infty = \max_{1\le i\le m}\sum_{j=1}^n |a_{ij}| \quad(\text{макс. сумма по строке}). \tag{11.3} \]
\[ \|A\|_2 = \sigma_{\max}(A) \quad(\text{максимальное сингулярное число}). \tag{11.4} \]
Мнемоника. Единичка «стоит» вертикально (столбцы) — \(\|A\|_1\) суммирует по столбцам; бесконечность «лежит» горизонтально (строки) — \(\|A\|_\infty\) суммирует по строкам. Спектральная норма \(\|A\|_2\) равна максимальному сингулярному числу \(\sigma_{\max}\) — этот объект мы подробно разберём на занятии 13 (SVD).

Отдельно стоит норма Фробениуса — она не индуцирована, а определяется как евклидова длина матрицы, «растянутой» в вектор:

\[ \|A\|_F = \sqrt{\sum_{i,j} a_{ij}^2} = \sqrt{\operatorname{tr}(A^\top A)} = \sqrt{\textstyle\sum_k \sigma_k^2}. \tag{11.5} \]

Норму Фробениуса легко считать, и она удовлетворяет \(\|A\|_2 \le \|A\|_F \le \sqrt{r}\,\|A\|_2\), где \(r\) — ранг. Поэтому её часто используют как дешёвую оценку спектральной нормы.

Ключевые свойства

Согласованность и субмультипликативность. Индуцированная норма по построению удовлетворяет \[ \|Ax\| \le \|A\|\,\|x\|,\qquad \|AB\| \le \|A\|\,\|B\|. \]
Первое неравенство — прямое следствие определения: для любого \(x\neq 0\) по максимальности \(\dfrac{\|Ax\|}{\|x\|}\le\|A\|\), откуда \(\|Ax\|\le\|A\|\,\|x\|\). Второе получаем дважды применив первое: для любого \(x\) имеем \(\|ABx\| \le \|A\|\,\|Bx\| \le \|A\|\,\|B\|\,\|x\|\); деля на \(\|x\|\) и беря максимум, получаем \(\|AB\|\le\|A\|\,\|B\|\).

Эти два неравенства — рабочая лошадка всего анализа ошибок: они позволяют «протаскивать» норму сквозь умножение на матрицу. Именно из них мы выведем главную оценку точности решения СЛАУ.

Практикум · Python (NumPy/SciPy) · ≈ 60 мин

Цель практикума — научиться измерять нормы и число обусловленности средствами NumPy и увидеть собственными руками, как плохая обусловленность разрушает точность. Главный инструмент — np.linalg.norm (с параметром ord) и np.linalg.cond. Справочник — в памятке Python/NumPy/SciPy.

Практикум 1. Нормы вектора и матрицы, число κ

norms.py▶ python
import numpy as np

x = np.array([3.0, -4.0, 12.0])
print("‖x‖₁  =", np.linalg.norm(x, 1))     # 19.0  = 3+4+12
print("‖x‖₂  =", np.linalg.norm(x, 2))     # 13.0  = sqrt(9+16+144)
print("‖x‖∞  =", np.linalg.norm(x, np.inf))# 12.0  = max|x_i|

A = np.array([[1.0, 2.0],
              [3.0, 4.0]])
print("‖A‖₁  =", np.linalg.norm(A, 1))     # 6.0  = max столбц. сумма (2+4)
print("‖A‖∞  =", np.linalg.norm(A, np.inf))# 7.0  = max строч. сумма (3+4)
print("‖A‖₂  =", np.linalg.norm(A, 2))     # 5.464 = σ_max
print("‖A‖_F =", np.linalg.norm(A, 'fro')) # 5.477 = sqrt(1+4+9+16)

# Число обусловленности в разных нормах
print("κ₂(A) =", np.linalg.cond(A, 2))     # σ_max/σ_min ≈ 14.93
print("κ∞(A) =", np.linalg.cond(A, np.inf))

Для вектора \(x = (1,\,-2,\,2)^\top\) и матрицы \(A = \left(\begin{smallmatrix} 2 & 0 & 1\\ -1 & 3 & 0\\ 0 & 1 & -2 \end{smallmatrix}\right)\) вычислите на бумаге \(\|x\|_1,\|x\|_2,\|x\|_\infty\) и \(\|A\|_1,\|A\|_\infty,\|A\|_F\). Затем проверьте все шесть значений вызовами np.linalg.norm.
Ответы для самопроверки \(\|x\|_1 = 5,\ \|x\|_2 = 3,\ \|x\|_\infty = 2\). Для матрицы: \(\|A\|_1 = \max(3,4,3) = 4\) (столбцовые суммы модулей), \(\|A\|_\infty = \max(3,4,3) = 4\) (строчные суммы), \(\|A\|_F = \sqrt{4+0+1+1+9+0+0+1+4} = \sqrt{20} \approx 4.472\). Практикум 2. Матрица Гильберта — катастрофа обусловленности

Матрица Гильберта \(H_{ij} = 1/(i+j-1)\) — классический пример системы, которая выглядит безобидно, но обусловлена ужасающе плохо: \(\kappa(H_n)\) растёт с \(n\) экспоненциально. Решим \(Hx = b\) с заведомо известным точным решением \(x = (1,\dots,1)^\top\) (для этого берём \(b = Hx\)) и измерим, сколько цифр потерялось.

hilbert.py▶ python
import numpy as np
from scipy.linalg import hilbert   # H[i,j] = 1/(i+j-1)

print(f"{'n':>3} {'κ₂(H)':>12} {'отн.ошибка':>12} {'log10 κ':>8}")
for n in range(2, 14):
    H = hilbert(n)
    x_true = np.ones(n)
    b = H @ x_true
    x_num = np.linalg.solve(H, b)        # численное решение
    err = np.linalg.norm(x_num - x_true) / np.linalg.norm(x_true)
    kappa = np.linalg.cond(H, 2)
    print(f"{n:>3} {kappa:>12.2e} {err:>12.2e} {np.log10(kappa):>8.1f}")
n κ₂(H) отн.ошибка log10 κ 2 1.93e+01 5.66e-16 1.3 4 1.55e+04 5.42e-13 4.2 6 1.50e+07 1.16e-10 7.2 8 1.53e+10 6.12e-08 10.2 10 1.60e+13 5.24e-04 13.2 12 1.75e+16 8.78e-01 16.2 13 3.0e+18 ~1 18.5

Закономерность бьёт в глаза: относительная ошибка решения примерно равна \(\kappa(H)\cdot u\), где \(u\approx10^{-16}\). При \(n=8\) число \(\kappa\approx10^{10}\), и ошибка \(\sim10^{10}\cdot10^{-16}=10^{-6}\) — потеряно около 10 значащих цифр. При \(n=13\) уже \(\kappa\sim10^{18} > 1/u\): решение полностью разрушено, хотя сама матрица невырождена и алгоритм Гаусса безупречен. Виновата не программа — виновата обусловленность задачи.

Задача 11.2Рост κ Гильберта. Постройте график \(\log_{10}\kappa_2(H_n)\) против \(n\) для \(n=1,\dots,15\). Убедитесь, что зависимость близка к линейной (то есть \(\kappa\) растёт экспоненциально по \(n\)). Оцените угловой коэффициент: на сколько порядков растёт \(\kappa\) при увеличении \(n\) на единицу?
Подсказка По точкам из практикума 2 наклон \(\approx 1.5\) порядка на единицу \(n\): известна асимптотика \(\kappa(H_n)\sim e^{3.5 n}\), то есть \(\log_{10}\kappa \approx 1.5\,n\). Это и есть «экспоненциальный рост».

Практикум 3. Малая невязка ≠ малая ошибка; усиление возмущения

Опасная иллюзия: «если невязка \(r = b - A\tilde x\) мала, решение хорошее». На плохо обусловленной матрице это неверно. Проверим, а заодно возмутим правую часть на \(10^{-10}\) и измерим, во сколько раз усилилась ошибка.

residual_vs_error.py▶ python
import numpy as np
from scipy.linalg import hilbert

n = 10
H = hilbert(n)
x_true = np.ones(n)
b = H @ x_true
kappa = np.linalg.cond(H, 2)

# --- решение и его невязка/ошибка ---
x_num = np.linalg.solve(H, b)
resid = np.linalg.norm(b - H @ x_num) / np.linalg.norm(b)
error = np.linalg.norm(x_num - x_true) / np.linalg.norm(x_true)
print(f"κ₂(H)        = {kappa:.2e}")
print(f"невязка r/b  = {resid:.2e}")   # КРОШЕЧНАЯ ~1e-16
print(f"ошибка Δx/x  = {error:.2e}")   # большая ~1e-4
print(f"κ × невязка  = {kappa*resid:.2e}  (оценка сверху для ошибки)")

# --- возмущение правой части на 1e-10 ---
db = 1e-10 * np.random.default_rng(0).standard_normal(n)
x_pert = np.linalg.solve(H, b + db)
amp = (np.linalg.norm(x_pert - x_num)/np.linalg.norm(x_num)) \
      / (np.linalg.norm(db)/np.linalg.norm(b))
print(f"усиление Δb→Δx = {amp:.2e}   (теория: ≤ κ = {kappa:.2e})")
κ₂(H) = 1.60e+13 невязка r/b = 4.4e-16 ошибка Δx/x = 5.2e-04 κ × невязка = 7.0e-03 (оценка сверху для ошибки) усиление Δb→Δx = 3.1e+12 (теория: ≤ κ = 1.60e+13)

Невязка порядка \(10^{-16}\) — машинная точность, идеал! — а ошибка решения \(5\cdot10^{-4}\): двенадцать порядков разницы. Возмущение правой части на \(10^{-10}\) раздувается в ошибку решения в \(\sim10^{12}\) раз — ровно как предсказывает \(\kappa\). Невязка обманывает; число обусловленности — нет.

Задача 11.3Невязка против ошибки. Возьмите \(2\times2\) матрицу \(A = \left(\begin{smallmatrix}1 & 1\\ 1 & 1.0001\end{smallmatrix}\right)\) и \(b=(2,\,2.0001)^\top\) (точное решение \(x=(1,1)^\top\)). Подсуньте «решение» \(\tilde x = (2,\,0)^\top\). Вычислите невязку \(\|b-A\tilde x\|\) и ошибку \(\|\tilde x-x\|\). Убедитесь, что невязка мала, а ошибка огромна, и сопоставьте отношение с \(\kappa(A)\).
Подсказка \(A\tilde x = (2,\,2)^\top\), значит \(r = b - A\tilde x = (0,\,0.0001)^\top\), \(\|r\|_2 = 10^{-4}\) — мало. Но \(\|\tilde x - x\|_2 = \|(1,-1)\|_2 = \sqrt2 \approx 1.41\) — огромно. Здесь \(\kappa_2(A)\approx 4\times10^4\), и неравенство \(\|\tilde x-x\|/\|x\| \le \kappa\cdot\|r\|/\|b\|\) выполняется с запасом.
Задача 11.4Потеря цифр ≈ log₁₀κ. Для \(n=2,4,6,8,10\) (матрица Гильберта) посчитайте число «верных цифр» решения как \(-\log_{10}(\text{отн.ошибка})\) и сравните с \(16 - \log_{10}\kappa(H_n)\) — теоретическим прогнозом числа уцелевших цифр в double. Постройте обе величины на одном графике.
Подсказка Правило большого пальца: в double изначально \(\approx 16\) верных цифр, при решении теряется \(\approx\log_{10}\kappa\) из них, остаётся \(\approx 16-\log_{10}\kappa\). Для \(H_8\): \(\log_{10}\kappa\approx10\), значит остаётся \(\approx 6\) цифр — что и наблюдается (ошибка \(\sim10^{-6}\)).

Лекция B · 30 мин

Число обусловленности матрицы

Практикум показал эмпирически: ошибка \(\approx \kappa\cdot u\). Теперь введём \(\kappa\) строго и докажем эту оценку.

Определение. Число обусловленности невырожденной матрицы \(A\) относительно нормы \(\|\cdot\|\) есть \[ \kappa(A) = \|A\|\cdot\|A^{-1}\|. \] Всегда \(\kappa(A)\ge 1\), поскольку \(1 = \|I\| = \|A A^{-1}\| \le \|A\|\,\|A^{-1}\| = \kappa(A)\). Для спектральной нормы \[ \kappa_2(A) = \frac{\sigma_{\max}(A)}{\sigma_{\min}(A)} \] — отношение наибольшего и наименьшего сингулярных чисел (вывод — на занятии 13, SVD).

Чем ближе \(\kappa\) к единице, тем «лучше» матрица; \(\kappa\gg1\) означает почти вырожденную матрицу. Для ортогональной матрицы \(Q\) (поворот/отражение) \(\kappa_2(Q)=1\) — она не искажает длины и потому идеальна численно.

Анализ ошибок СЛАУ

Рассмотрим систему \(Ax = b\). Пусть входные данные возмущены; как это сказывается на решении?

Возмущение правой части. Если \(Ax = b\) и \(A(x+\Delta x) = b + \Delta b\), то \[ \frac{\|\Delta x\|}{\|x\|} \;\le\; \kappa(A)\,\frac{\|\Delta b\|}{\|b\|}. \]
Вычитая \(Ax=b\) из \(A(x+\Delta x)=b+\Delta b\), получаем \(A\,\Delta x = \Delta b\), откуда \(\Delta x = A^{-1}\Delta b\), и по согласованности \[ \|\Delta x\| \le \|A^{-1}\|\,\|\Delta b\|. \] С другой стороны, из \(b = Ax\) следует \(\|b\| \le \|A\|\,\|x\|\), то есть \(\dfrac{1}{\|x\|} \le \dfrac{\|A\|}{\|b\|}\). Перемножая два неравенства: \[ \frac{\|\Delta x\|}{\|x\|} \le \|A^{-1}\|\,\|\Delta b\|\cdot\frac{\|A\|}{\|b\|} = \underbrace{\|A\|\,\|A^{-1}\|}_{\kappa(A)}\,\frac{\|\Delta b\|}{\|b\|}. \]

Относительная ошибка решения может быть в \(\kappa(A)\) раз больше относительной ошибки данных. Аналогичный (с точностью до первого порядка) результат для возмущения матрицы:

Возмущение матрицы. Если \(Ax=b\) и \((A+\Delta A)(x+\Delta x)=b\), то в первом порядке по \(\|\Delta A\|\) \[ \frac{\|\Delta x\|}{\|x\|} \;\lesssim\; \kappa(A)\,\frac{\|\Delta A\|}{\|A\|}. \]
Раскрывая скобки и используя \(Ax=b\), получаем \(A\,\Delta x + \Delta A\,(x+\Delta x) = 0\), откуда \(\Delta x = -A^{-1}\Delta A\,(x+\Delta x)\). В первом порядке \(x+\Delta x\approx x\), поэтому \(\|\Delta x\| \lesssim \|A^{-1}\|\,\|\Delta A\|\,\|x\|\). Деля на \(\|x\|\) и домножая числитель и знаменатель на \(\|A\|\): \[ \frac{\|\Delta x\|}{\|x\|} \lesssim \|A^{-1}\|\,\|\Delta A\| = \|A\|\,\|A^{-1}\|\,\frac{\|\Delta A\|}{\|A\|} = \kappa(A)\,\frac{\|\Delta A\|}{\|A\|}. \]

Оба результата объединяются одним лозунгом: относительная ошибка решения \(\lesssim \kappa(A)\times\) (относительная ошибка данных). Поскольку округление вносит в данные ошибку \(\sim u\), получаем оценку точности решения \(\sim\kappa(A)\cdot u\) — ровно то, что мы видели на матрице Гильберта.

Геометрический смысл числа обусловленности

Откуда берётся \(\kappa_2 = \sigma_{\max}/\sigma_{\min}\)? Рассмотрим действие \(A\) на единичную сферу \(\{x:\|x\|_2 = 1\}\). Образ — эллипсоид, полуоси которого равны сингулярным числам \(\sigma_1\ge\sigma_2\ge\dots\ge\sigma_n\), а направлены вдоль левых сингулярных векторов.

  • \(\sigma_{\max}\) — наибольшее растяжение (\(=\|A\|_2\)); \(\sigma_{\min}\) — наименьшее.
  • \(\kappa_2 = \sigma_{\max}/\sigma_{\min}\) — отношение макс. и мин. полуоси, мера «вытянутости» эллипсоида.
  • \(\kappa_2 = 1\): сфера переходит в сферу (ортогональная матрица с точностью до масштаба) — идеал.
  • \(\kappa_2 \gg 1\): эллипсоид сильно вытянут, почти «схлопнут» в одном направлении — матрица близка к вырожденной (\(\sigma_{\min}\to0\)).

Геометрически плохая обусловленность означает: вдоль «тонкого» направления эллипсоида крошечное смещение выхода соответствует огромному смещению входа — обратное отображение \(A^{-1}\) растягивает это направление в \(1/\sigma_{\min}\) раз. Поиграйте с этим ниже.

Интерактив: эллипс числа обусловленности

Матрица \(A=\left(\begin{smallmatrix} s & k\\ 0 & 1 \end{smallmatrix}\right)\) задаётся двумя слайдерами: \(s\) — растяжение, \(k\) — сдвиг (shear). Слева — единичная окружность (входные векторы) и её образ-эллипс \(A\cdot(\text{окружность})\); красным и синим показаны главные полуоси \(\sigma_{\max}u_1\) и \(\sigma_{\min}u_2\). Справа — те же сингулярные числа столбиками. Увеличивая вытянутость эллипса, наблюдайте рост \(\kappa_2=\sigma_{\max}/\sigma_{\min}\).

Невязка против ошибки. Главное правило

Практикум 3 продемонстрировал ловушку: малая невязка не гарантирует малой ошибки. Свяжем их строго. Пусть \(\tilde x\) — приближённое решение, \(r = b - A\tilde x\) — невязка. Тогда \(A(x-\tilde x) = b - A\tilde x = r\), то есть \(x-\tilde x = A^{-1}r\), и повторяя выкладку доказательства выше:

Оценка ошибки через невязку. \[ \frac{\|\tilde x - x\|}{\|x\|} \;\le\; \kappa(A)\,\frac{\|r\|}{\|b\|}. \] Малая невязка \(\|r\|\) даёт малую ошибку только если \(\kappa(A)\) умеренно. При большом \(\kappa\) даже машинно-точная невязка совместима с грубо неверным решением.

Это утверждение — частный случай главного правила вычислительной математики из занятия 1: прямая ошибка \(\lesssim\) обусловленность \(\times\) обратная ошибка. Невязка играет роль обратной ошибки (метод Гаусса с выбором главного элемента обратно устойчив: \(\|r\|/\|b\|\sim u\)), а \(\kappa(A)\) — роль обусловленности задачи. Отсюда и практическое правило большого пальца:

Правило большого пальца. При решении СЛАУ в формате double (\(\approx16\) значащих цифр) теряется примерно \(\log_{10}\kappa(A)\) значащих цифр. Уцелевает \(\approx 16 - \log_{10}\kappa(A)\) цифр. Если \(\kappa\sim10^{16}\), не уцелевает ни одной — решение бессмысленно. Матрица Гильберта \(H_{ij}=1/(i+j-1)\) достигает этого уже при \(n\approx13\).

Итоги занятия

Что нужно унести с занятия 11:
  1. Нормы измеряют величину векторов и матриц. Векторные: \(\|x\|_1=\sum|x_i|\), \(\|x\|_2=\sqrt{\sum x_i^2}\), \(\|x\|_\infty=\max|x_i|\). В конечномерье все нормы эквивалентны.
  2. Индуцированные матричные нормы: \(\|A\|_1\) — макс. столбцовая сумма, \(\|A\|_\infty\) — макс. строчная, \(\|A\|_2=\sigma_{\max}\). Свойства: \(\|Ax\|\le\|A\|\|x\|\) и \(\|AB\|\le\|A\|\|B\|\).
  3. Число обусловленности \(\kappa(A)=\|A\|\,\|A^{-1}\|\ge1\); для 2-нормы \(\kappa_2=\sigma_{\max}/\sigma_{\min}\). Геометрически — вытянутость эллипсоида-образа единичной сферы.
  4. Анализ ошибок: \(\dfrac{\|\Delta x\|}{\|x\|}\le\kappa(A)\dfrac{\|\Delta b\|}{\|b\|}\) и аналогично для \(\Delta A\). Относительная ошибка решения \(\lesssim\kappa\cdot u\).
  5. Невязка \(\neq\) ошибка: \(\dfrac{\|\tilde x-x\|}{\|x\|}\le\kappa(A)\dfrac{\|r\|}{\|b\|}\). Малая невязка обманчива при большом \(\kappa\).
  6. Правило большого пальца: теряется \(\approx\log_{10}\kappa\) значащих цифр. Матрица Гильберта — эталон катастрофической обусловленности.
